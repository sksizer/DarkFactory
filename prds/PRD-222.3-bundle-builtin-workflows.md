# Bundle built-in workflows inside the package

## Summary

Move the three first-party workflows (`default`, `extraction`, `planning`) from the top-level `workflows/` directory into `src/darkfactory/workflows/` so they ship inside the installed wheel. Update the workflow loader to discover built-in workflows via Python module imports rather than filesystem scanning of an external directory. Delete the old top-level `workflows/` directory once the loader is switched over.

## Requirements

1. Move `workflows/default/` to `src/darkfactory/workflows/default/`.
2. Move `workflows/extraction/` to `src/darkfactory/workflows/extraction/`.
3. Move `workflows/planning/` to `src/darkfactory/workflows/planning/`.
4. Create or update `src/darkfactory/workflows/__init__.py` to expose a function `get_builtin_workflows() -> dict[str, Workflow]` that imports and returns all three built-in workflows.
5. Update `loader.py`'s `load_workflows()` to accept a source specifier or add a new `load_builtin_workflows()` function that discovers workflows from the package rather than a filesystem directory.
6. Delete the old top-level `workflows/` directory (and its `__init__.py`).
7. All existing tests that reference workflows continue to pass after the move.
8. Built-in workflow modules must be importable as `darkfactory.workflows.default`, `darkfactory.workflows.extraction`, `darkfactory.workflows.planning`.

## Technical Approach

### Moving workflow files

```
workflows/default/    → src/darkfactory/workflows/default/
workflows/extraction/ → src/darkfactory/workflows/extraction/
workflows/planning/   → src/darkfactory/workflows/planning/
```

Each subdirectory keeps its `__init__.py` and `workflow.py`. The top-level `workflows/__init__.py` is replaced by `src/darkfactory/workflows/__init__.py`.

### `src/darkfactory/workflows/__init__.py`

```python
from pathlib import Path
from darkfactory.loader import _load_workflow_module
from darkfactory.models import Workflow

BUILTIN_WORKFLOW_NAMES = ["default", "extraction", "planning"]

def get_builtin_workflows() -> dict[str, Workflow]:
    """Load all built-in workflows shipped with the package."""
    result = {}
    base = Path(__file__).parent
    for name in BUILTIN_WORKFLOW_NAMES:
        subdir = base / name
        wf_file = subdir / "workflow.py"
        if wf_file.exists():
            wf = _load_workflow_module(wf_file, subdir)
            result[name] = wf
    return result
```

### Changes to `src/darkfactory/loader.py`

- Extract `_load_workflow_module` as a semi-public helper (rename to `load_workflow_module` or keep private but importable within the package).
- `load_workflows(workflows_dir)` continues to work for filesystem-based workflow loading (used by user-level and project-level layers).
- Add or expose `load_builtin_workflows()` that delegates to `get_builtin_workflows()`.

### Cleanup

- Delete `workflows/` at the repo top level after confirming all references point to the new location.
- Update any test fixtures or conftest that reference the old `workflows/` path.

## Acceptance Criteria

- [ ] AC-1: `default`, `extraction`, and `planning` workflows exist under `src/darkfactory/workflows/` and are importable as Python modules.
- [ ] AC-2: `get_builtin_workflows()` returns all three built-in workflows.
- [ ] AC-3: The old top-level `workflows/` directory is deleted.
- [ ] AC-4: Existing tests pass after the move.
- [ ] AC-5: A `pip install -e .` or `uv tool install` includes the workflow files in the installed package.
