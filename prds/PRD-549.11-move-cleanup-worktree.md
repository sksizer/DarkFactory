# Move cleanup_worktree to its own submodule with colocated tests

## Summary

Extract the `cleanup_worktree` builtin function from `builtins/__init__.py` into `src/darkfactory/builtins/cleanup_worktree.py`. Create a colocated `cleanup_worktree_test.py` with unit tests. This builtin has no private helpers.

## Requirements

1. Create `src/darkfactory/builtins/cleanup_worktree.py` containing the `cleanup_worktree` function decorated with `@builtin("cleanup_worktree")`.
2. Create `src/darkfactory/builtins/cleanup_worktree_test.py` with unit tests covering: no worktree path set (skip), worktree already gone (skip), dry-run path, successful removal.
3. Remove `cleanup_worktree` from `builtins/__init__.py`.
4. Add submodule import in `__init__.py` to trigger registration.
5. All existing tests pass.

## Technical Approach

### `src/darkfactory/builtins/cleanup_worktree.py`

```python
from __future__ import annotations
import logging
import subprocess
from darkfactory.builtins._registry import builtin
from darkfactory.workflow import ExecutionContext

_log = logging.getLogger(__name__)

@builtin("cleanup_worktree")
def cleanup_worktree(ctx: ExecutionContext) -> None: ...
```

### `src/darkfactory/builtins/cleanup_worktree_test.py`

- `worktree_path is None` → logs and returns
- `worktree_path` does not exist → logs and returns
- Dry-run logs the `git worktree remove` command without executing
- Successful removal calls `git worktree remove`

## Acceptance Criteria

- [ ] AC-1: `src/darkfactory/builtins/cleanup_worktree.py` contains the `cleanup_worktree` function.
- [ ] AC-2: `cleanup_worktree_test.py` covers no-path, already-gone, dry-run, and successful removal.
- [ ] AC-3: `cleanup_worktree` is no longer defined in `builtins/__init__.py`.
- [ ] AC-4: `from darkfactory.builtins import cleanup_worktree` still works.
- [ ] AC-5: All tests pass. `just test`, `just lint`, `just typecheck`, `just format-check` all clean.
