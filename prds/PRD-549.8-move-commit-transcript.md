# Move commit_transcript to its own submodule with colocated tests

## Summary

Extract the `commit_transcript` builtin function from `builtins/__init__.py` into `src/darkfactory/builtins/commit_transcript.py`. Create a colocated `commit_transcript_test.py` with unit tests. This builtin has no private helpers — it uses `shutil.move` and `subprocess.run` directly.

## Requirements

1. Create `src/darkfactory/builtins/commit_transcript.py` containing the `commit_transcript` function decorated with `@builtin("commit_transcript")`.
2. Create `src/darkfactory/builtins/commit_transcript_test.py` with unit tests covering: no transcript file (skip), dry-run path, successful move and stage.
3. Remove `commit_transcript` from `builtins/__init__.py`.
4. Add submodule import in `__init__.py` to trigger registration.
5. All existing tests pass.

## Technical Approach

### `src/darkfactory/builtins/commit_transcript.py`

```python
from __future__ import annotations
import logging
import shutil
import subprocess
from datetime import datetime
from darkfactory.builtins._registry import builtin
from darkfactory.workflow import ExecutionContext

_log = logging.getLogger(__name__)

@builtin("commit_transcript")
def commit_transcript(ctx: ExecutionContext) -> None: ...
```

### `src/darkfactory/builtins/commit_transcript_test.py`

- No transcript file → logs and returns (no-op)
- Dry-run with transcript file → logs intended move without executing
- Successful run → creates transcript dir, moves file, calls `git add`
