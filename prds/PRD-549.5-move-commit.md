# Move commit to its own submodule with colocated tests

## Summary

Extract the `commit` builtin function from `builtins/__init__.py` into `src/darkfactory/builtins/commit.py`. Create a colocated `commit_test.py` with unit tests. The `commit` builtin uses `_scan_for_forbidden_attribution` from `_shared.py`.

## Requirements

1. Create `src/darkfactory/builtins/commit.py` containing the `commit` function decorated with `@builtin("commit")`.
2. Import `_scan_for_forbidden_attribution` from `darkfactory.builtins._shared`.
3. Create `src/darkfactory/builtins/commit_test.py` with unit tests covering: dry-run path, empty diff (no changes to commit), successful commit, forbidden attribution in message raises `RuntimeError`.
4. Remove `commit` from `builtins/__init__.py`.
5. Add submodule import in `__init__.py` to trigger registration.
6. All existing tests pass.

## Technical Approach

### `src/darkfactory/builtins/commit.py`

```python
from __future__ import annotations
import subprocess
from darkfactory.builtins._registry import builtin
from darkfactory.builtins._shared import _scan_for_forbidden_attribution
from darkfactory.workflow import ExecutionContext

@builtin("commit")
def commit(ctx: ExecutionContext, *, message: str) -> None: ...
```

### `src/darkfactory/builtins/commit_test.py`

- Dry-run logs the formatted message without subprocess calls
- Empty diff (returncode 0 from `git diff --cached --quiet`) skips commit
- Successful commit calls `git add -A` then `git commit -m`
- Forbidden attribution pattern in message raises `RuntimeError`

## Acceptance Criteria

- [ ] AC-1: `src/darkfactory/builtins/commit.py` contains the `commit` function.
- [ ] AC-2: `commit_test.py` covers dry-run, empty diff, successful commit, and forbidden attribution.
- [ ] AC-3: `commit` is no longer defined in `builtins/__init__.py`.
- [ ] AC-4: `from darkfactory.builtins import commit` still works.
- [ ] AC-5: All tests pass. `just test`, `just lint`, `just typecheck`, `just format-check` all clean.
