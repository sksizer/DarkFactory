# Move set_status to its own submodule with colocated tests

## Summary

Extract the `set_status` builtin function from `builtins/__init__.py` into `src/darkfactory/builtins/set_status.py`. Create a colocated `set_status_test.py` with unit tests. This builtin has no private helpers — it delegates to `prd_module.set_status_at`.

## Requirements

1. Create `src/darkfactory/builtins/set_status.py` containing the `set_status` function decorated with `@builtin("set_status")`.
2. Create `src/darkfactory/builtins/set_status_test.py` with unit tests covering: dry-run path, missing worktree raises `RuntimeError`, successful status update calls `prd_module.set_status_at` and updates `ctx.prd.status`.
3. Remove `set_status` from `builtins/__init__.py`.
4. Add submodule import in `__init__.py` to trigger registration.
5. All existing tests pass.

## Technical Approach

### `src/darkfactory/builtins/set_status.py`

```python
from __future__ import annotations
from darkfactory.builtins._registry import builtin
from darkfactory import prd as prd_module
from darkfactory.workflow import ExecutionContext, Status

@builtin("set_status")
def set_status(ctx: ExecutionContext, *, to: Status) -> None: ...
```

### `src/darkfactory/builtins/set_status_test.py`

- Dry-run logs but does not call `set_status_at`
- `worktree_path is None` raises `RuntimeError`
- Successful call updates the file and mirrors status onto `ctx.prd`

## Acceptance Criteria

- [ ] AC-1: `src/darkfactory/builtins/set_status.py` contains the `set_status` function.
- [ ] AC-2: `set_status_test.py` covers dry-run, missing-worktree error, and successful update.
- [ ] AC-3: `set_status` is no longer defined in `builtins/__init__.py`.

- [ ] AC-5: All tests pass. `just test`, `just lint`, `just typecheck`, `just format-check` all clean.
