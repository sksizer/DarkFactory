# Cascade resolver for config

## Summary

Generalize the style-only config resolution from `style.py` into a shared `darkfactory.config` module that merges settings across all layers: built-in defaults < user-level `config.toml` < project-level `config.toml` < env vars < CLI flags. The existing style config resolver (`resolve_style_config` in `style.py`) is refactored to consume the resolved `Config` object instead of resolving its own cascade. This establishes the single config resolution path used by all features.

## Requirements

1. Create `src/darkfactory/config.py` with a `resolve_config()` function that performs the full cascade merge.
2. Config cascade order (later wins, key-by-key merge):
   - Built-in defaults (hardcoded in the module)
   - User-level: `~/.config/darkfactory/config.toml`
   - Project-level: `<project>/.darkfactory/config.toml`
   - Environment variables (prefixed with `DARKFACTORY_`)
   - CLI flags
3. Define the config schema as a dataclass or typed dict covering:
   - `[model]` — model defaults per capability tier (trivial, simple, moderate, complex)
   - `[timeouts]` — effort-based timeout overrides (from PRD-542)
   - `[style]` — theme, icon_set, no_color (from PRD-541)
   - Extensible for future sections without breaking existing configs
4. TOML parsing uses Python stdlib `tomllib` (3.11+).
5. Refactor `style.py`'s `resolve_style_config()` to read from the resolved `Config` object rather than doing its own file loading. Remove the duplicate `_merge_toml_file` logic from `style.py`.
6. Env var mapping convention: `DARKFACTORY_MODEL_TRIVIAL` maps to `config.model.trivial`, `DARKFACTORY_STYLE_THEME` maps to `config.style.theme`, etc.

## Technical Approach

### `src/darkfactory/config.py`

```python
import tomllib
from dataclasses import dataclass, field
from pathlib import Path
from darkfactory.paths import user_config_file

@dataclass
class ModelConfig:
    trivial: str = "haiku"
    simple: str = "sonnet"
    moderate: str = "sonnet"
    complex: str = "opus"

@dataclass
class StyleConfig:
    theme: str = "dark"
    icon_set: str = "ascii"
    no_color: bool = False

@dataclass
class Config:
    model: ModelConfig = field(default_factory=ModelConfig)
    style: StyleConfig = field(default_factory=StyleConfig)
    # Extensible: add timeouts, etc.

def _load_toml(path: Path) -> dict:
    """Load a TOML file, returning {} if absent."""
    if not path.is_file():
        return {}
    with open(path, "rb") as f:
        return tomllib.load(f)

def _merge_section(target, section_data: dict) -> None:
    """Merge a dict into a dataclass, key-by-key."""
    for key, value in section_data.items():
        if hasattr(target, key):
            setattr(target, key, value)

def resolve_config(
    project_dir: Path | None = None,
    env: dict[str, str] | None = None,
    cli_overrides: dict | None = None,
) -> Config:
    """Resolve config through the full cascade."""
    config = Config()

    # Layer 1: user config
    user_file = user_config_file()
    if user_file:
        user_data = _load_toml(user_file)
        _merge_section(config.model, user_data.get("model", {}))
        _merge_section(config.style, user_data.get("style", {}))

    # Layer 2: project config
    if project_dir:
        proj_file = project_dir / "config.toml"
        proj_data = _load_toml(proj_file)
        _merge_section(config.model, proj_data.get("model", {}))
        _merge_section(config.style, proj_data.get("style", {}))

    # Layer 3: env vars
    # Layer 4: CLI flags
    # (applied by caller or via helper)

    return config
```

### Changes to `src/darkfactory/style.py`

- Remove `_merge_toml_file` and the file-loading logic from `resolve_style_config()` (around line 380).
- `resolve_style_config()` accepts a `Config` object and reads `config.style.*` instead of loading TOML files itself.

### Changes to `src/darkfactory/cli.py`

- Call `resolve_config(darkfactory_dir, cli_overrides=...)` early in `main()`.
- Pass the resolved config to `resolve_style_config()` and other consumers.

### `tests/test_config.py`

- Test built-in defaults when no files exist.
- Test user config overrides built-in defaults.
- Test project config overrides user config.
- Test env vars override project config.
- Test CLI flags override everything.
- Test key-by-key merge (partial override doesn't wipe other keys).
- Test absent config files are handled gracefully.

## Acceptance Criteria

- [ ] AC-1: `resolve_config()` merges layers in the correct order.
- [ ] AC-2: A key set in project config overrides the same key in user config.
- [ ] AC-3: Env vars (e.g. `DARKFACTORY_MODEL_TRIVIAL`) override config file values.
- [ ] AC-4: CLI flags override everything.
- [ ] AC-5: Partial overrides merge key-by-key without wiping other keys in the same section.
- [ ] AC-6: `style.py` consumes the resolved `Config` object instead of loading files itself.
- [ ] AC-7: Absent config files produce no errors (empty merge).
