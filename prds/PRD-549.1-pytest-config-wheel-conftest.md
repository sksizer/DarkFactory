# Pytest config, wheel exclusion, and conftest split

## Summary

Configure pytest to discover colocated `*_test.py` files alongside standard `test_*.py` files, exclude test files from the built wheel, and split `conftest.py` so shared fixtures are available to both `tests/` and colocated unit tests.

This is the foundational infrastructure task — all per-builtin migration tasks depend on the test discovery and conftest changes landing first.

## Requirements

1. Update `[tool.pytest.ini_options]` in `pyproject.toml`:
   - `testpaths = ["src", "tests", "workflows"]`
   - `python_files = ["test_*.py", "*_test.py"]`
   - `addopts = "--import-mode=importlib"`
2. Update the build backend config to exclude `**/*_test.py` and `**/conftest.py` from the built wheel.
3. Split `tests/conftest.py`: move truly project-wide fixtures to a repo-root `conftest.py`, keep integration-specific fixtures in `tests/conftest.py`.
4. Add a throwaway colocated test file (e.g. `src/darkfactory/prd_test.py` with a trivial assertion) to verify discovery works. Remove or replace before merge.
5. Build the wheel and verify no `*_test.py` files are inside it.

## Technical Approach

### `pyproject.toml` changes

```toml
[tool.pytest.ini_options]
testpaths = ["src", "tests", "workflows"]
python_files = ["test_*.py", "*_test.py"]
addopts = "--import-mode=importlib"
```

### Wheel exclusion

Check the build backend in `pyproject.toml`. If Hatchling:

```toml
[tool.hatch.build.targets.wheel]
exclude = ["**/*_test.py", "**/conftest.py"]
```

If setuptools, use the equivalent `exclude` pattern.

### conftest.py split

- Read `tests/conftest.py` and identify which fixtures are used by tests across both `tests/` and `src/`.
- Move shared fixtures to a new repo-root `conftest.py`.
- Keep integration-only fixtures in `tests/conftest.py`.

### Verification

```bash
uv build
# Unzip the wheel and assert no *_test.py files
python -c "import zipfile; z=zipfile.ZipFile('dist/*.whl'); assert not [f for f in z.namelist() if f.endswith('_test.py')]"
```

## Acceptance Criteria

- [ ] AC-1: `pyproject.toml` configures pytest to discover both `test_*.py` and `*_test.py` across `src/`, `tests/`, and `workflows/` with `--import-mode=importlib`.
- [ ] AC-2: All existing tests pass under the new config (`just test` green).
- [ ] AC-3: A colocated `*_test.py` file under `src/darkfactory/` is discovered and run by pytest.
- [ ] AC-4: A built wheel contains zero `*_test.py` files.
- [ ] AC-5: Repo-root `conftest.py` provides shared fixtures to both `tests/` and colocated unit tests.
- [ ] AC-6: `just lint`, `just typecheck`, and `just format-check` all pass.
