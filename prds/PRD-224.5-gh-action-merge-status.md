# GitHub Action: flip review to done on PR merge

## Summary

Add a GitHub Action at `.github/workflows/prd-status-on-merge.yml` that fires when a PR targeting `main` is merged. If the PR's head branch matches `prd/PRD-*`, the action extracts the PRD ID, finds the corresponding PRD file, flips `status: review` to `status: done`, updates the `updated:` field, commits to main, and pushes.

This is a `[CI config]` — the structural fix for Mode 1 (status drift after merge). Once in place, the happy path is fully automated.

## Requirements

1. The action triggers on `pull_request` `closed` events on the `main` branch.
2. It only runs when the PR was actually merged (not just closed) AND the head branch starts with `prd/PRD-`.
3. It extracts the PRD ID from the branch name (e.g. `prd/PRD-216-foo` -> `PRD-216`).
4. It finds the PRD file matching `prds/${PRD_ID}-*.md`.
5. It flips `status: review` to `status: done` using sed (surgical, single-line replacement).
6. It updates the `updated:` field to today's date.
7. It commits with message `chore(prd): mark PRD-X done (auto-reconciled by merge action) [skip ci]` using a bot identity.
8. If no PRD file matches or the status is not `review`, the action exits 0 gracefully.
9. The action uses `[skip ci]` in the commit message to avoid triggering further CI runs.

## Technical Approach

### `.github/workflows/prd-status-on-merge.yml`

```yaml
name: Reconcile PRD status on merge
on:
  pull_request:
    types: [closed]
    branches: [main]
jobs:
  flip-status:
    if: github.event.pull_request.merged == true && startsWith(github.event.pull_request.head.ref, 'prd/PRD-')
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
        with:
          ref: main
          fetch-depth: 0
      - name: Flip status to done
        run: |
          BRANCH="${{ github.event.pull_request.head.ref }}"
          PRD_ID=$(echo "$BRANCH" | sed -E 's|^prd/(PRD-[0-9.]+).*|\1|')
          FILE=$(ls prds/${PRD_ID}-*.md 2>/dev/null || true)
          if [ -z "$FILE" ]; then
            echo "No PRD file matching $PRD_ID -- skipping"
            exit 0
          fi
          sed -i 's|^status: review$|status: done|' "$FILE"
          sed -i "s|^updated: .*|updated: '$(date -u +%Y-%m-%d)'|" "$FILE"
          if ! git diff --quiet "$FILE"; then
            git config user.name "darkfactory-bot"
            git config user.email "darkfactory-bot@users.noreply.github.com"
            git add "$FILE"
            git commit -m "chore(prd): mark $PRD_ID done (auto-reconciled by merge action) [skip ci]"
            git push
          fi
```

## Open Questions

- What about PRDs whose status field uses a different YAML quoting style than the sed pattern expects? The `update_frontmatter_field_at` function from PRD-214 is the robust answer, but the sed pattern is good enough for now since it matches the shape every other status mutation produces.
- What if the PRD file is missing on main? Action exits 0 with a log message.
