# Timeout resolution module with effort-based mapping

## Summary

Create a new `darkfactory/timeouts.py` module exposing a pure function `resolve_timeout(prd, config, cli_override) -> int` that computes the agent task timeout in seconds. The function implements a clear resolution order: CLI flag > PRD frontmatter `timeout_minutes` > config table lookup by effort × capability multiplier > built-in defaults. This task also adds the config schema for the timeout table in `.darkfactory/config.toml` and comprehensive unit tests.

## Requirements

1. Create `src/darkfactory/timeouts.py` with a `resolve_timeout()` function.
2. Built-in default effort → timeout mapping:
   - `xs` → 300s (5 min)
   - `s` → 600s (10 min)
   - `m` → 1200s (20 min)
   - `l` → 2400s (40 min)
   - `xl` → 4500s (75 min)
3. Capability multiplier applied on top of effort base: `simple` ×1.0, `moderate` ×1.25, `complex` ×1.5. When `capability` is `trivial` or missing, use ×1.0.
4. Resolution order (highest priority wins):
   - `cli_override` (integer minutes, if provided)
   - PRD frontmatter `timeout_minutes` field (integer minutes)
   - Config table lookup (`[timeouts]` section in `.darkfactory/config.toml`) by effort × capability multiplier
   - Built-in defaults × capability multiplier
5. The function returns an integer (seconds) and a string describing the source (e.g. `"cli"`, `"prd_frontmatter"`, `"config"`, `"default"`).
6. Add a `[timeouts]` section to `.darkfactory/config.toml` schema with keys for each effort tier (values in minutes) and capability multipliers.
7. If the PRD has no `effort` field, fall back to `s` (10 min) as the default.

## Technical Approach

### `src/darkfactory/timeouts.py`

```python
DEFAULT_EFFORT_TIMEOUTS = {
    "xs": 300,
    "s": 600,
    "m": 1200,
    "l": 2400,
    "xl": 4500,
}

CAPABILITY_MULTIPLIERS = {
    "trivial": 1.0,
    "simple": 1.0,
    "moderate": 1.25,
    "complex": 1.5,
}

def resolve_timeout(
    effort: str | None,
    capability: str | None,
    timeout_minutes_frontmatter: int | None,
    config_timeouts: dict | None,
    cli_override: int | None,
) -> tuple[int, str]:
    """Return (timeout_seconds, source_description)."""
    ...
```

### Config schema addition (`.darkfactory/config.toml`)

```toml
[timeouts]
xs = 5       # minutes
s = 10
m = 20
l = 40
xl = 75

[timeouts.capability_multipliers]
trivial = 1.0
simple = 1.0
moderate = 1.25
complex = 1.5
```

### Unit tests (`tests/test_timeouts.py`)

Table-driven tests covering:
- Every effort tier with default mapping
- Every capability multiplier
- PRD frontmatter override takes precedence over config/defaults
- CLI override takes precedence over everything
- Config table overrides built-in defaults
- Missing effort falls back to `s`
- Missing capability uses multiplier 1.0
