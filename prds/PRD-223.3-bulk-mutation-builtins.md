# Bulk-mutation builtins for system operations

## Summary

Implement the shared builtins that system operations need but that don't exist in the current per-PRD builtin registry. The primary addition is `set_status_bulk` — update the status field of multiple PRDs in a single call. Also add `system_load_prds_by_status` (load PRDs matching a given status into `_shared_state`) and `system_check_merged` (check git history to see if each candidate's branch was merged to main).

These builtins operate on `SystemContext` and are registered in a `SYSTEM_BUILTINS` dict parallel to the existing `BUILTINS` dict.

## Requirements

1. `set_status_bulk(ctx: SystemContext, status: str)` — for each PRD id in `ctx.targets`, update its status field in the PRD file on disk. Respects `ctx.dry_run` — in dry-run mode, log what would change without writing.
2. `system_load_prds_by_status(ctx: SystemContext, status: str)` — filter `ctx.prds` by the given status, store the matching PRD ids in `ctx._shared_state['candidates']`.
3. `system_check_merged(ctx: SystemContext)` — for each candidate in `ctx._shared_state['candidates']`, check whether a branch named `prd/PRD-X-slug` was merged to main (via `git log --all --grep` or `git branch --merged`). Handles both merge-commit and squash-and-merge patterns. Populates `ctx.targets` with confirmed-merged PRD ids and `ctx.report` with human-readable lines.
4. `SYSTEM_BUILTINS: dict[str, Callable]` registry that the system runner dispatches against.
5. All builtins must handle the `dry_run` flag on `ctx`.

## Technical Approach

**New file**: `src/darkfactory/builtins/system_builtins.py`

```python
from __future__ import annotations
from typing import Callable
from ..system import SystemContext

SYSTEM_BUILTINS: dict[str, Callable] = {}

def _register(name: str):
    def decorator(fn):
        SYSTEM_BUILTINS[name] = fn
        return fn
    return decorator

@_register("set_status_bulk")
def set_status_bulk(ctx: SystemContext, *, status: str) -> None:
    ...

@_register("system_load_review_prds")
def system_load_review_prds(ctx: SystemContext) -> None:
    system_load_prds_by_status(ctx, status="review")

@_register("system_load_prds_by_status")
def system_load_prds_by_status(ctx: SystemContext, *, status: str) -> None:
    ...

@_register("system_check_merged")
def system_check_merged(ctx: SystemContext) -> None:
    ...

@_register("system_mark_done")
def system_mark_done(ctx: SystemContext) -> None:
    set_status_bulk(ctx, status="done")
```

**New file**: `tests/test_system_builtins.py` — tests covering:
- `set_status_bulk` writes status to PRD files (and skips in dry-run)
- `system_load_prds_by_status` filters correctly
- `system_check_merged` identifies squash-merged and merge-commit patterns
- Idempotency: calling `set_status_bulk` on already-done PRDs is a no-op
