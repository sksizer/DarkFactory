# Move push_branch to its own submodule with colocated tests

## Summary

Extract the `push_branch` builtin function from `builtins/__init__.py` into `src/darkfactory/builtins/push_branch.py`. Create a colocated `push_branch_test.py` with unit tests. This is one of the simplest builtins — no private helpers, just a `git push -u origin` call.

## Requirements

1. Create `src/darkfactory/builtins/push_branch.py` containing the `push_branch` function decorated with `@builtin("push_branch")`.
2. Create `src/darkfactory/builtins/push_branch_test.py` with unit tests covering: dry-run path, successful push.
3. Remove `push_branch` from `builtins/__init__.py`.
4. Add submodule import in `__init__.py` to trigger registration.
5. All existing tests pass.

## Technical Approach

### `src/darkfactory/builtins/push_branch.py`

```python
from __future__ import annotations
import subprocess
from darkfactory.builtins._registry import builtin
from darkfactory.workflow import ExecutionContext

@builtin("push_branch")
def push_branch(ctx: ExecutionContext) -> None: ...
```

### `src/darkfactory/builtins/push_branch_test.py`

- Dry-run logs the push command without subprocess calls
- Successful push calls `git push -u origin {branch}`

## Acceptance Criteria

- [ ] AC-1: `src/darkfactory/builtins/push_branch.py` contains the `push_branch` function.
- [ ] AC-2: `push_branch_test.py` covers dry-run and successful push.
- [ ] AC-3: `push_branch` is no longer defined in `builtins/__init__.py`.
- [ ] AC-4: `from darkfactory.builtins import push_branch` still works.
- [ ] AC-5: All tests pass. `just test`, `just lint`, `just typecheck`, `just format-check` all clean.
