# Public check function: validate_review_branches

## Summary

Extract a public check function `darkfactory.checks.validate_review_branches(prds, git_state) -> list[Issue]` that returns a list of "PRD is in review but its branch is gone from origin" issues. Wire it into `prd validate` so it surfaces the issues as warnings. Other commands and SystemOperations can call the same function.

This is a `[primitive]` — a reusable check function, not hard-coded logic.

## Requirements

1. Create a new module `src/darkfactory/checks.py` with a `validate_review_branches` function.
2. The function takes injected dependencies (`prds` dict, `git_state` adapter) so it's testable without a real repo.
3. The function returns a list of `Issue` objects (or dataclass) describing each PRD that is in `review` status but whose branch is gone from origin.
4. Wire the function into the existing `prd validate` command so it runs as part of validation and surfaces issues as warnings (not errors — the PRD file itself is still valid).
5. Unit tests covering: no issues found, one stale PRD, multiple stale PRDs, PRD in review with branch still present.

## Technical Approach

### `src/darkfactory/checks.py`

```python
@dataclass
class Issue:
    prd_id: str
    message: str
    severity: str  # "warning" | "error"

def validate_review_branches(
    prds: dict[str, PRDMeta],
    git_state: GitStateAdapter,
) -> list[Issue]:
    """Return issues for PRDs in 'review' whose branch is gone from origin."""
    issues = []
    for prd_id, meta in prds.items():
        if meta.status != "review":
            continue
        branch = f"prd/{prd_id}-{meta.slug}"
        if not git_state.remote_branch_exists(branch):
            issues.append(Issue(
                prd_id=prd_id,
                message=f"{prd_id} is in 'review' but branch '{branch}' is gone from origin",
                severity="warning",
            ))
    return issues
```

### `src/darkfactory/cli.py`

In `cmd_validate`, after existing validation logic:

```python
from darkfactory.checks import validate_review_branches
issues = validate_review_branches(prds, git_state)
for issue in issues:
    console.print(f"[yellow]WARNING:[/yellow] {issue.message}")
```

### `tests/test_checks.py`

Test with mock `prds` dict and mock `git_state` adapter. Cover:
- Empty prds dict -> no issues
- PRD in review, branch exists -> no issues
- PRD in review, branch gone -> one issue
- Multiple PRDs in review, mixed -> correct subset
