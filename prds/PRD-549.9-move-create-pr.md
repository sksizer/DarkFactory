# Move create_pr to its own submodule with colocated tests

## Summary

Extract the `create_pr` builtin function and its exclusively-used private helpers (`_extract_acceptance_criteria`, `_pr_body`, `_AC_LINE_RE`) from `builtins/__init__.py` into `src/darkfactory/builtins/create_pr.py`. Create a colocated `create_pr_test.py` with unit tests. This builtin imports `_scan_for_forbidden_attribution` from `_shared.py`.

## Requirements

1. Create `src/darkfactory/builtins/create_pr.py` containing the `create_pr` function decorated with `@builtin("create_pr")`, plus its private helpers: `_AC_LINE_RE`, `_extract_acceptance_criteria`, `_pr_body`.
2. Import `_scan_for_forbidden_attribution` from `darkfactory.builtins._shared`.
3. Create `src/darkfactory/builtins/create_pr_test.py` with unit tests covering: dry-run path (sets `ctx.pr_url` to dummy URL), successful PR creation, PR body generation with acceptance criteria, forbidden attribution in title or body raises `RuntimeError`, `_extract_acceptance_criteria` parsing.
4. Remove `create_pr`, `_extract_acceptance_criteria`, `_pr_body`, and `_AC_LINE_RE` from `builtins/__init__.py`.
5. Add submodule import in `__init__.py` to trigger registration.
6. All existing tests pass.

### `src/darkfactory/builtins/create_pr_test.py`

- Dry-run sets `ctx.pr_url` to placeholder URL
- `_extract_acceptance_criteria` parses AC lines from markdown
- `_pr_body` generates body with PRD link and AC checklist
- Forbidden attribution in title raises `RuntimeError`
- Successful creation calls `gh pr create` and parses URL from stdout
