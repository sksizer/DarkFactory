# reconcile-status SystemOperation + prd reconcile CLI

## Summary

A `reconcile-status` SystemOperation (PRD-223 abstraction) plus a thin CLI wrapper `prd reconcile` that finds PRDs in `review` status whose PRs are already merged and flips them to `done`. Dry-run by default; `--execute` performs the updates; `--commit-to-main` commits directly to main for trivial status-only changes.

This supersedes and absorbs PRD-223.4 (`reconcile-status` SystemOperation). They are the same thing — tracked here to avoid duplication.

This is a `[primitive + CLI]`.

## Requirements

1. Use `gh pr list --state merged --json headRefName,mergedAt,number` to find merged PRs whose head branch matches `prd/PRD-*`.
2. For each merged PR, find the corresponding PRD file with `status: review` and flip it to `done`.
3. Update the `updated:` field to today's date.
4. **Dry-run by default**: `prd reconcile` prints what would change but makes no modifications.
5. **`--execute`**: performs the status updates and creates a PR with the changes.
6. **`--commit-to-main`**: for trivial status-only changes, commit directly to main instead of opening a PR. Explicit opt-in.
7. Commit message format: `chore(prd): mark PRD-X done (auto-reconciled from merged PR #N) [skip ci]`.
8. When multiple PRDs need reconciling, batch them in a single commit: `chore(prd): reconcile N merged PRD statuses [skip ci]`.
9. Use `update_frontmatter_field_at` (PRD-214) for robust YAML frontmatter updates.

## Technical Approach

### `src/darkfactory/cli.py`

```python
@app.command()
def reconcile(
    execute: bool = False,
    commit_to_main: bool = typer.Option(False, "--commit-to-main"),
):
    """Find merged-but-not-flipped PRDs and reconcile their status."""
    # 1. Get merged PRs with prd/* branches
    merged_prs = _get_merged_prd_prs()

    # 2. Find corresponding PRD files still in 'review'
    candidates = []
    for pr in merged_prs:
        prd_file = _find_prd_file_for_branch(pr["headRefName"])
        if prd_file and _get_status(prd_file) == "review":
            candidates.append((prd_file, pr))

    if not candidates:
        console.print("All PRD statuses are up to date.")
        return

    # 3. Print what would change (dry-run)
    for prd_file, pr in candidates:
        prd_id = _extract_prd_id(prd_file)
        console.print(f"  {prd_id}: review -> done (from merged PR #{pr['number']})")

    if not execute:
        console.print("\nDry run. Use --execute to apply changes.")
        return

    # 4. Apply changes
    for prd_file, pr in candidates:
        update_frontmatter_field_at(prd_file, "status", "done")
        update_frontmatter_field_at(prd_file, "updated", date.today().isoformat())

    # 5. Commit
    if commit_to_main:
        _commit_to_main(candidates)
    else:
        _create_reconcile_pr(candidates)
```

### Tests (`tests/test_cli_reconcile.py`)

- Dry-run with one candidate -> prints the change, makes no modifications.
- `--execute` with one candidate -> flips status and updated field.
- `--execute --commit-to-main` -> commits directly to main.
- No candidates -> prints "up to date" message.
- Multiple candidates -> batched in a single commit.

## Acceptance Criteria

- [ ] AC-3: `prd reconcile --execute --commit-to-main` commits directly to main with the correct commit message format.

## Open Questions

- Should reconcile also clean up local worktrees as a side effect? Recommendation: no — stay focused on status. Chain with `prd cleanup --merged` for full cleanup.
- Direct-to-main without a PR is unusual. Default to creating a PR (`--execute`), with `--commit-to-main` as an explicit shortcut.
