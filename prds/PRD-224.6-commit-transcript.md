# commit_transcript BuiltIn + transcripts layout

## Summary

A new `commit_transcript` BuiltIn that moves the agent transcript from the temporary location to `.darkfactory/transcripts/PRD-X-{ISO8601}.log`, stages it, and ensures the next `commit` BuiltIn picks it up. One file per agent invocation, timestamped, never overwritten.

This is a `[primitive]` — composable into any workflow's task list.

## Requirements

1. Create a `commit_transcript` BuiltIn that:
   - Reads the transcript from the temporary location written by `runner.py` (currently `.harness-agent-output.log` or similar).
   - Moves it to `.darkfactory/transcripts/PRD-X-{ISO8601}.log` where ISO8601 is the invocation timestamp.
   - Stages the file with `git add`.
   - One file per agent invocation — never overwrite existing transcripts.
2. Update the default workflow to insert `commit_transcript` before the final `commit` step.
3. Ensure `.darkfactory/transcripts/` is NOT in `.gitignore`.
4. Add `.gitattributes` entry marking `*.log` under `.darkfactory/transcripts/` as `linguist-generated=true` so GitHub collapses them in PR diffs.
5. If no transcript exists (e.g. dry-run or the runner didn't produce one), the BuiltIn is a no-op.
6. Custom workflows can omit `commit_transcript` if they don't want transcript persistence.

## Technical Approach

### `src/darkfactory/builtins.py`

```python
def commit_transcript(ctx: ExecutionContext) -> None:
    """Move agent transcript to .darkfactory/transcripts/ and stage it."""
    src = ctx.worktree_root / ".harness-agent-output.log"
    if not src.exists():
        logger.info("No transcript found; skipping")
        return

    transcript_dir = ctx.worktree_root / ".darkfactory" / "transcripts"
    transcript_dir.mkdir(parents=True, exist_ok=True)

    timestamp = ctx.invocation_timestamp.strftime("%Y-%m-%dT%H-%M-%S")
    dest = transcript_dir / f"{ctx.prd_id}-{timestamp}.log"

    shutil.move(str(src), str(dest))

    subprocess.run(["git", "add", str(dest)], cwd=ctx.worktree_root, check=True)
    logger.info(f"Transcript staged: {dest.relative_to(ctx.worktree_root)}")
```

### `src/darkfactory/runner.py`

Ensure the transcript is written to a known location that `commit_transcript` can find. If the transcript path needs adjustment, update it here.

### `.gitattributes`

```
.darkfactory/transcripts/*.log linguist-generated=true
```

### Default workflow

Insert `commit_transcript` in the task list before the final `commit` step.

### Tests

- Test `commit_transcript` with a mock transcript file -> verify it's moved and staged.
- Test `commit_transcript` with no transcript file -> verify no-op.
- Test that multiple invocations produce separate timestamped files.

## Acceptance Criteria

- [ ] AC-1: `commit_transcript` BuiltIn exists and is registered.
- [ ] AC-2: An agent invocation creates a file at `.darkfactory/transcripts/PRD-X-{timestamp}.log`.
- [ ] AC-3: The transcript file is staged by `commit_transcript` and committed by the subsequent `commit` step.
- [ ] AC-4: `.gitattributes` marks transcript logs as `linguist-generated=true`.
- [ ] AC-5: If no transcript exists, `commit_transcript` is a no-op (no error).
- [ ] AC-6: Multiple invocations produce separate files, never overwriting.
- [ ] AC-7: Unit tests pass for all described scenarios.

## Notes

- Bloat: ~100KB per transcript x ~100 PRDs/year = ~10MB/year. Acceptable for now.
- Security: transcripts may contain file contents. A future `prd validate --check-transcripts` can scan for token-shaped strings.
- Layout assumes `.darkfactory/` discovery (PRD-222.1). Until then, use `<repo_root>/.darkfactory/transcripts/` directly.
