DarkFactory stores all configuration, PRDs, workflows, operations, and runtime artifacts in a `.darkfactory/` directory at the root of your repository. This page documents every file and subdirectory, including what is tracked in git and what is generated at runtime.

## Directory Structure

```
<repo-root>/
  .worktrees/                          # Runtime: git worktrees (repo root, not .darkfactory/)
    PRD-042-add-retry-logic/
    PRD-045-dead-letter-queue/
    PRD-042.lock                       # Advisory per-PRD file locks
  .harness-transcripts/                # Runtime: raw agent transcripts (JSONL)
    PRD-042.jsonl
    PRD-045.jsonl
  .darkfactory/
    config.toml                        # Project configuration
    data/                              # All PRD data lives under here
      prds/                            # Active PRD documents
        PRD-001-auth-refactor.md
        PRD-002-add-logging.md
        PRD-010-epic-runner-v2.md
      archive/                         # Completed/cancelled PRDs (terminal states)
        PRD-001-auth-refactor.md
    workflows/                         # Custom workflow definitions
      frontend/
        workflow.py
        prompts/
          implement.md
          verify.md
      rust-service/
        workflow.py
        prompts/
          implement.md
    operations/                        # Custom system operations
      upgrade-deps/
        operation.py
      rotate-secrets/
        operation.py
    transcripts/                       # Runtime: committed transcript copies
      PRD-042-20260101T120000.jsonl
      PRD-045-20260101T130000.jsonl
    events/                            # Runtime: JSONL event logs
      PRD-042_session-abc123.jsonl
      PRD-045_session-abc123.jsonl
```

## Git Tracking Status

| Path | Tracked | Notes |
|------|---------|-------|
| `.darkfactory/config.toml` | Yes (optional) | Project configuration. Committed to share settings across team. |
| `.darkfactory/data/prds/` | Yes | Active PRD documents. These are the source of truth for all work items. |
| `.darkfactory/data/archive/` | Yes (optional) | Archived PRDs (terminal states: `done`, `superseded`, `cancelled`). |
| `.darkfactory/workflows/` | Yes (optional) | Custom workflow definitions. Committed to share workflow logic. |
| `.darkfactory/operations/` | Yes (optional) | Custom system operations. Committed to share operational scripts. |
| `.darkfactory/transcripts/` | Yes (optional) | Committed transcript copies produced by `commit_transcript`. |
| `.darkfactory/events/` | No (git-ignored) | Runtime artifact. JSONL event logs for post-mortem analysis. |
| `.worktrees/` | No (git-ignored) | Runtime artifact at repo root. Created by `ensure_worktree`. |
| `.harness-transcripts/` | No (git-ignored) | Runtime artifact at repo root. Raw agent transcripts written by `pnpm dlx @anthropic-ai/claude-code --print`. |

The `.gitignore` entries for runtime directories:

```gitignore
# .gitignore (repo root)
.worktrees/
.harness-transcripts/

# .darkfactory/.gitignore
events/
```

## `config.toml`

Project configuration file. Controls model selection, timeout tiers, capability multipliers, and output styling. See the [Configuration Reference](/reference/config/) for the complete schema.

This file is optional. DarkFactory uses sensible defaults for all settings. Create it only when you need to override defaults.

## `data/`

All PRD data lives under `.darkfactory/data/`. The harness auto-migrates older layouts (where PRDs were stored directly under `.darkfactory/prds/`) on startup via `ensure_data_layout()`.

## `data/prds/`

Contains all active PRD documents as Markdown files with YAML frontmatter. This is the primary data store for DarkFactory -- the harness reads PRDs from this directory on every invocation.

**File naming**: `PRD-NNN-slug.md` (e.g., `PRD-042-add-retry-logic.md`). The frontmatter `id` field is authoritative; the filename is for human convenience.

**Subdirectories**: Not supported. All PRDs must be direct children of `data/prds/`. The containment hierarchy is expressed via the `parent` frontmatter field, not via filesystem nesting.

See the [PRD Frontmatter Schema](/reference/frontmatter/) for field definitions.

<Aside type="caution">
Do not nest PRDs in subdirectories. The harness scans `data/prds/*.md` (not recursive). PRDs in subdirectories will not be discovered.
</Aside>

## `data/archive/`

Contains PRDs that have reached a terminal state (`done`, `superseded`, or `cancelled`) and been moved out of the active queue via `prd archive <PRD-ID>`. Archived PRDs are still visible to `prd tree` and other read-only commands but are excluded from execution planning.

## `workflows/`

Custom workflow definitions. Each workflow lives in its own directory containing a `workflow.py` and optional `prompts/` subdirectory.

```
workflows/
  <name>/
    workflow.py          # Exports `workflow` attribute (Workflow dataclass)
    prompts/             # Prompt templates for AgentTasks
      implement.md
      verify.md
```

The harness discovers workflows by scanning for `workflows/*/workflow.py` files at startup. See [Custom Workflows](/guides/custom-workflows/) for authoring details and [Workflow API](/reference/workflow-api/) for the dataclass schema.

## `operations/`

Custom system operations. Each operation lives in its own directory containing an `operation.py`.

```
operations/
  <name>/
    operation.py         # Exports `operation` attribute (Operation dataclass)
```

Operations are discovered by scanning `operations/*/operation.py` at startup. See [System Operations](/guides/system-operations/) for authoring details.

## `.worktrees/` (repo root)

Git worktrees created during PRD execution. Each PRD gets its own worktree directory. Created by the `ensure_worktree` builtin, optionally removed by `cleanup_worktree` (opt-in, not the default).

This directory is git-ignored and can grow large during DAG execution with many PRDs. Clean up stale worktrees manually with:

```bash frame="terminal" title="Clean up stale worktrees"
git worktree prune
prd cleanup --all
```

## `.harness-transcripts/` (repo root)

Raw agent conversation transcripts in JSONL format. Written directly by `pnpm dlx @anthropic-ai/claude-code --print` during agent invocations. One file per PRD ID.

These are the source files that `commit_transcript` copies into `.darkfactory/transcripts/`. They are git-ignored and can be deleted after the committed copy exists.

<Aside type="note">
These files are written by `pnpm dlx @anthropic-ai/claude-code --print` (the Claude Code CLI), not by `claude --print`. The harness invokes Claude Code via `pnpm dlx` to avoid a global installation requirement.
</Aside>

## `.darkfactory/transcripts/`

Committed copies of agent transcripts produced by the `commit_transcript` builtin. Each file is timestamped: `<prd-id>-<timestamp>.jsonl`.

```bash frame="terminal" title="View a transcript"
cat .darkfactory/transcripts/PRD-042-20260101T120000.jsonl
```

## `events/`

JSONL event log files. One file per PRD per session. Contains structured events for every workflow and task execution: start/finish timestamps, exit codes, durations, and shell output.

See [Event Logging](/guides/event-logging/) for the event schema and analysis recipes.

<Aside type="tip">
To keep the `.darkfactory/` directory clean, periodically prune runtime artifacts. Worktrees from completed PRDs can be removed. Old transcripts and event logs can be archived or deleted. The harness does not depend on historical runtime artifacts.
</Aside>

## Initializing a New Project

```bash frame="terminal" title="Initialize DarkFactory in a project"
prd init
```

This creates the `.darkfactory/` directory with a default `config.toml` and the `data/prds/` directory. The `.gitignore` for runtime directories is created automatically.
