Workflows in DarkFactory are defined as Python dataclasses in `workflow.py` files. This page documents every dataclass, field, type, and default value in the workflow API.

## `Workflow`

The top-level container. Exported as the `workflow` attribute from a workflow module.

```python
@dataclass
class Workflow:
    name: str
    description: str
    priority: int
    applies_to: Callable[[PRD, list[PRD]], bool]
    tasks: list[Task]
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `str` | Required | Unique workflow identifier. Used in CLI output and event logs. |
| `description` | `str` | Required | Human-readable description. Shown by `prd plan`. |
| `priority` | `int` | Required | Matching priority. Higher values take precedence. Built-ins use 0-5. |
| `applies_to` | `Callable[[PRD, list[PRD]], bool]` | Required | Predicate function. Receives the candidate PRD and all PRDs in the project. |
| `tasks` | `list[Task]` | Required | Ordered task sequence. Executed sequentially by the runner. |

## Task Types

All tasks share a common base. Three specializations exist:

### `BuiltIn`

References a task from the `BUILTINS` registry by name.

```python
@dataclass
class BuiltIn(Task):
    name: str
    kwargs: dict[str, Any] = field(default_factory=dict)
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `str` | Required | Registry name (e.g., `"ensure_worktree"`, `"commit"`, `"create_pr"`). Must match a key in `BUILTINS`. |
| `kwargs` | `dict` | `{}` | Keyword arguments passed to the builtin function. Values can include `\{{PLACEHOLDER}}` tokens. |

### `AgentTask`

Invokes an LLM agent with a prompt file and scoped tool access.

```python
@dataclass
class AgentTask(Task):
    name: str
    prompt_file: str
    allowed_tools: list[str] = field(default_factory=list)
    verify_prompts: list[str] = field(default_factory=list)
    model: str | None = None
    max_tokens: int = 16384
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `str` | Required | Task identifier. Used in event logs and CLI output. |
| `prompt_file` | `str` | Required | Path to prompt template, relative to `<workflow_dir>/prompts/`. |
| `allowed_tools` | `list[str]` | `[]` | Tool allowlist. Supports glob patterns for Bash subcommands (e.g., `"Bash(cargo:*)"`). Empty list means all tools allowed. |
| `verify_prompts` | `list[str]` | `[]` | Prompt files used during `retry_agent` re-invocation. Receive `\{{CHECK_OUTPUT}}` placeholder. |
| `model` | `str \| None` | `None` | Model override. If `None`, uses the config model for the PRD's effort level. |
| `max_tokens` | `int` | `16384` | Maximum output tokens for the agent response. |

<Aside type="caution">
An empty `allowed_tools` list means ALL tools are available, not zero tools. To restrict tool access, explicitly list the permitted tools.
</Aside>

### `ShellTask`

Runs a shell command in the worktree.

```python
@dataclass
class ShellTask(Task):
    name: str
    command: str
    on_failure: str = "fail"
    timeout: int | None = None
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `str` | Required | Task identifier. Used in event logs and CLI output. |
| `command` | `str` | Required | Shell command to execute. Supports `\{{PLACEHOLDER}}` substitution. Runs via `subprocess.run(shell=True)`. |
| `on_failure` | `str` | `"fail"` | Failure policy: `"fail"` (abort), `"retry_agent"` (re-invoke last AgentTask with verify prompts), `"ignore"` (log and continue). |
| `timeout` | `int \| None` | `None` | Command timeout in seconds. If `None`, uses the PRD's effort-based timeout. |

## `ExecutionContext`

Passed to every task at runtime. Provides access to PRD metadata, project configuration, and the worktree environment.

```python
@dataclass
class ExecutionContext:
    prd: PRD
    project_root: Path
    worktree_path: Path
    branch: str
    base_branch: str
    config: Config
    dry_run: bool
    session_id: str
    event_writer: EventWriter
    agent_response: str | None = None
```

| Field | Type | Description |
|-------|------|-------------|
| `prd` | `PRD` | The parsed PRD with frontmatter and body. |
| `project_root` | `Path` | Absolute path to the repository root. |
| `worktree_path` | `Path` | Absolute path to the PRD's worktree directory. |
| `branch` | `str` | Git branch name for this PRD (e.g., `prd/042-add-retry-logic`). |
| `base_branch` | `str` | Base branch the worktree was created from. |
| `config` | `Config` | Parsed `.darkfactory/config.toml` with defaults applied. |
| `dry_run` | `bool` | `True` if `--execute` was not passed. Tasks should print but not act. |
| `session_id` | `str` | Unique session identifier. Shared across all PRDs in a DAG run. |
| `event_writer` | `EventWriter` | Handle for emitting JSONL events. |
| `agent_response` | `str \| None` | The most recent agent's output. Populated after each AgentTask completes. |

## `PRD_IMPLEMENTATION_TEMPLATE`

A helper that wraps custom tasks in the standard open/close envelope.

```python
PRD_IMPLEMENTATION_TEMPLATE.compose(
    implement: AgentTask,
    verify: list[ShellTask],
) -> list[Task]
```

Returns the full task list:

```python
[
    BuiltIn(name="ensure_worktree"),
    BuiltIn(name="set_status", kwargs={"status": "in-progress"}),
    implement,                    # Your AgentTask
    *verify,                      # Your ShellTasks
    BuiltIn(name="commit"),
    BuiltIn(name="push_branch"),
    BuiltIn(name="create_pr"),
    BuiltIn(name="cleanup_worktree"),
]
```

## Complete Workflow Example

```python
from darkfactory.workflow import (
    Workflow, AgentTask, ShellTask, BuiltIn,
    PRD_IMPLEMENTATION_TEMPLATE,
)

workflow = Workflow(
    name="python-library",
    description="Standard workflow for Python library PRDs",
    priority=15,
    applies_to=lambda prd, prds: (
        prd.kind == "task"
        and any(i.endswith(".py") for i in prd.impacts)
    ),
    tasks=PRD_IMPLEMENTATION_TEMPLATE.compose(
        implement=AgentTask(
            name="implement",
            prompt_file="implement.md",
            allowed_tools=[
                "Read", "Edit", "Write", "Glob", "Grep",
                "Bash(python:*)", "Bash(pytest:*)", "Bash(mypy:*)",
            ],
            verify_prompts=["fix-errors.md"],
        ),
        verify=[
            ShellTask(name="typecheck", command="mypy src/", on_failure="retry_agent"),
            ShellTask(name="lint", command="ruff check . --fix", on_failure="retry_agent"),
            ShellTask(name="test", command="pytest tests/ -x", on_failure="fail"),
        ],
    ),
)
```

<Aside type="tip">
When writing workflows, put cheap verification steps first (typecheck, lint) and expensive ones last (integration tests). The runner stops on the first `on_failure="fail"` failure, so ordering by cost saves time.
</Aside>
