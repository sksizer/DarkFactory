Workflows in DarkFactory are defined as Python dataclasses in `workflow.py` files. This page documents every dataclass, field, type, and default value in the workflow API.

## `Workflow`

The top-level container. Exported as the `workflow` attribute from a workflow module.

```python
@dataclass
class Workflow:
    name: str
    description: str = ""
    applies_to: AppliesToPredicate = _default_applies_to
    priority: int = 0
    tasks: list[Task] = field(default_factory=list)
    workflow_dir: Path | None = None
    template_name: str | None = None
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `str` | Required | Unique workflow identifier. Used in CLI output and event logs. |
| `description` | `str` | `""` | Human-readable description. Shown by `prd plan`. |
| `applies_to` | `Callable[[PRD, dict[str, PRD]], bool]` | matches nothing | Predicate function. Receives the candidate PRD and all PRDs as a dict. |
| `priority` | `int` | `0` | Matching priority. Higher values take precedence. Built-ins use 0-5. |
| `tasks` | `list[Task]` | `[]` | Ordered task sequence. Executed sequentially by the runner. |
| `workflow_dir` | `Path \| None` | `None` | Set by the loader to the workflow's directory. AgentTask prompt paths resolve relative to this. |
| `template_name` | `str \| None` | `None` | Template name if this workflow was composed from a `WorkflowTemplate`. |

## Task Types

All tasks share a common base. Three specializations exist:

### `BuiltIn`

References a task from the `BUILTINS` registry by name.

```python
@dataclass
class BuiltIn(Task):
    name: str
    kwargs: dict[str, Any] = field(default_factory=dict)
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `str` | Required | Registry name (e.g., `"ensure_worktree"`, `"commit"`, `"create_pr"`). Must match a key in `BUILTINS`. |
| `kwargs` | `dict` | `{}` | Keyword arguments passed to the builtin function. Values can include `\{{PLACEHOLDER}}` tokens. |

### `AgentTask`

Invokes an LLM agent with composed prompts and a scoped tool allowlist.

```python
@dataclass
class AgentTask(Task):
    name: str = "implement"
    prompts: list[str] = field(default_factory=list)
    tools: list[str] = field(default_factory=list)
    model: str | None = None
    model_from_capability: bool = True
    retries: int = 1
    verify_prompts: list[str] = field(default_factory=list)
    sentinel_success: str = "PRD_EXECUTE_OK"
    sentinel_failure: str = "PRD_EXECUTE_FAILED"
    effort_level: EffortLevel | None = None
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `str` | `"implement"` | Task identifier. Used in event logs and CLI output. |
| `prompts` | `list[str]` | `[]` | Paths to prompt template files, relative to `workflow_dir`. Concatenated in order. |
| `tools` | `list[str]` | `[]` | Tool allowlist passed to `--allowed-tools`. Supports glob patterns (e.g., `"Bash(cargo:*)"`). Empty list means all tools allowed. |
| `model` | `str \| None` | `None` | Explicit model override. Takes precedence over capability mapping. |
| `model_from_capability` | `bool` | `True` | When `True` and `model` is `None`, selects model from the PRD's `capability` field. |
| `retries` | `int` | `1` | Maximum retry attempts when a `ShellTask` triggers `retry_agent`. |
| `verify_prompts` | `list[str]` | `[]` | Prompt files used during `retry_agent` re-invocation. Receive `{CHECK_OUTPUT}` bound to failure output. |
| `sentinel_success` | `str` | `"PRD_EXECUTE_OK"` | String the agent must emit to signal success. |
| `sentinel_failure` | `str` | `"PRD_EXECUTE_FAILED"` | String the agent emits to signal failure. |
| `effort_level` | `EffortLevel \| None` | `None` | Session-scoped adaptive-reasoning budget passed to `--effort`. One of `"low"`, `"medium"`, `"high"`, `"max"`. `"max"` requires Opus 4.6. |

<Aside type="caution">
An empty `tools` list means ALL tools are available, not zero tools. To restrict tool access, explicitly list the permitted tools.
</Aside>

### `ShellTask`

Runs a shell command in the worktree.

```python
@dataclass
class ShellTask(Task):
    name: str
    cmd: str
    on_failure: OnFailure = "fail"
    env: dict[str, str] = field(default_factory=dict)
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `str` | Required | Task identifier. Used in event logs and CLI output. |
| `cmd` | `str` | Required | Shell command to execute. Supports `{placeholder}` substitution from `ExecutionContext`. |
| `on_failure` | `str` | `"fail"` | Failure policy: `"fail"` (abort), `"retry_agent"` (re-invoke last AgentTask with verify prompts), `"ignore"` (log and continue). |
| `env` | `dict[str, str]` | `{}` | Environment variable overrides merged into the subprocess environment. |

## `ExecutionContext`

Passed to every task at runtime. State is mutated in place as the workflow progresses.

```python
@dataclass
class ExecutionContext:
    prd: PRD
    repo_root: Path
    workflow: Workflow
    base_ref: str
    branch_name: str
    worktree_path: Path | None = None
    cwd: Path = field(default_factory=Path.cwd)
    agent_output: str | None = None
    agent_success: bool = False
    pr_url: str | None = None
    model: str | None = None
    invoke_count: int = 0
    dry_run: bool = True
    event_writer: EventWriter | None = None
```

| Field | Type | Description |
|-------|------|-------------|
| `prd` | `PRD` | The parsed PRD with frontmatter and body. |
| `repo_root` | `Path` | Absolute path to the repository root. |
| `workflow` | `Workflow` | The executing workflow object. |
| `base_ref` | `str` | Base branch the worktree was created from (e.g., `main`). |
| `branch_name` | `str` | Git branch name for this PRD (e.g., `prd/PRD-042-add-retry-logic`). |
| `worktree_path` | `Path \| None` | Absolute path to the PRD's worktree. Set by `ensure_worktree`. |
| `cwd` | `Path` | Current working directory for shell tasks. Set to `worktree_path` by `ensure_worktree`. |
| `agent_output` | `str \| None` | Captured stdout from the last agent invocation. |
| `agent_success` | `bool` | Whether the last agent emitted the success sentinel. |
| `pr_url` | `str \| None` | URL of the opened PR. Set by `create_pr`. |
| `model` | `str \| None` | Resolved model name. |
| `invoke_count` | `int` | Number of agent invocations so far. |
| `dry_run` | `bool` | `True` if `--execute` was not passed. Tasks print but do not act. |
| `event_writer` | `EventWriter \| None` | Handle for emitting JSONL events. |

## `PRD_IMPLEMENTATION_TEMPLATE`

A `WorkflowTemplate` that wraps custom tasks in the standard open/close envelope.

```python
PRD_IMPLEMENTATION_TEMPLATE.compose(
    implement: AgentTask,
    verify: list[ShellTask],
) -> list[Task]
```

Returns the full task list (open + your middle tasks + close):

```python
[
    # open
    BuiltIn("ensure_worktree"),
    BuiltIn("set_status", kwargs={"to": "in-progress"}),
    BuiltIn("commit", kwargs={"message": "chore(prd): {prd_id} start work"}),
    # middle (your tasks)
    implement,
    *verify,
    # close
    BuiltIn("set_status", kwargs={"to": "review"}),
    BuiltIn("commit_events"),
    BuiltIn("analyze_transcript"),
    BuiltIn("commit", kwargs={"message": "chore(prd): {prd_id} ready for review"}),
    BuiltIn("summarize_agent_run"),
    BuiltIn("lint_attribution"),
    BuiltIn("push_branch"),
    BuiltIn("create_pr"),
]
```

## Complete Workflow Example

```python
from darkfactory.workflow import (
    Workflow, AgentTask, ShellTask, BuiltIn,
)
from darkfactory.templates_builtin import PRD_IMPLEMENTATION_TEMPLATE

workflow = Workflow(
    name="python-library",
    description="Standard workflow for Python library PRDs",
    priority=15,
    applies_to=lambda prd, prds: (
        prd.kind == "task"
        and any(i.endswith(".py") for i in prd.impacts)
    ),
    tasks=PRD_IMPLEMENTATION_TEMPLATE.compose(
        implement=AgentTask(
            name="implement",
            prompts=["implement.md"],
            tools=[
                "Read", "Edit", "Write", "Glob", "Grep",
                "Bash(python:*)", "Bash(pytest:*)", "Bash(mypy:*)",
            ],
            verify_prompts=["fix-errors.md"],
        ),
        verify=[
            ShellTask(name="typecheck", cmd="mypy src/", on_failure="retry_agent"),
            ShellTask(name="lint", cmd="ruff check . --fix", on_failure="retry_agent"),
            ShellTask(name="test", cmd="pytest tests/ -x", on_failure="fail"),
        ],
    ),
)
```

<Aside type="tip">
When writing workflows, put cheap verification steps first (typecheck, lint) and expensive ones last (integration tests). The runner stops on the first `on_failure="fail"` failure, so ordering by cost saves time.
</Aside>
