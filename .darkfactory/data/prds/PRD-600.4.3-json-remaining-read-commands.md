# Add --json to remaining read commands

## Summary

After PRD-600.3.2 (validate) and PRD-600.3.3 (tree), the remaining read commands without `--json` support are: `children`, `orphans`, `undecomposed`, `cleanup` (dry-run output), and `reconcile` (dry-run output). Add JSON output to complete structured output coverage.

## Technical approach

For each command, add a JSON branch that emits the same data the human-readable output shows:

1. **children**: Emit JSON array of `{id, title, status, kind}` objects
2. **orphans**: Emit JSON array of `{id, title, status, kind}` objects
3. **undecomposed**: Emit JSON array of `{id, title, kind, children_count}` objects
4. **cleanup** (dry-run): Emit JSON array of `{prd_id, worktree_path, reason}` objects
5. **reconcile** (dry-run): Emit JSON array of `{prd_id, current_status, new_status, pr_url}` objects

These are all simple list commands where the data is already computed -- only the formatting branch needs to be added.

## Acceptance criteria

- [ ] AC-1: `prd children PRD-500 --json` emits JSON array
- [ ] AC-2: `prd orphans --json` emits JSON array
- [ ] AC-3: `prd undecomposed --json` emits JSON array
- [ ] AC-4: `prd cleanup --merged --json` emits JSON array (dry-run)
- [ ] AC-5: `prd reconcile --json` emits JSON array (dry-run)
- [ ] AC-6: Human-readable output unchanged for all commands
