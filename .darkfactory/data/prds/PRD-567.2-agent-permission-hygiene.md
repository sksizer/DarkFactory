# Agent permission hygiene: eliminate denial waste and harden prompts

## Summary

Every agent session produces permission denials for `git commit` — 153 total across 36 analyzed transcripts, wasting ~76K tokens per batch. The denials happen because (a) `git commit` is absent from `--allowed-tools` but not in `--disallowed-tools`, so Claude Code handles it through its interactive approval system, and (b) the prompts say "do not commit" but don't concretely explain why retrying is pointless.

Two independent fixes: make the denial instant via `--disallowed-tools`, and harden the prompts so agents don't attempt the operation in the first place.

## Acceptance criteria

- [ ] AC-1: Harness-owned git operations are in `--disallowed-tools` and rejected instantly.
- [ ] AC-2: Agent prompts explicitly list blocked commands and instruct not to retry denials.
