# PhaseState type-keyed registry module

## Summary

Create `src/darkfactory/phase_state.py` containing the `PhaseState` class — a
type-keyed registry for inter-task data. The class uses `@overload` signatures
so that `mypy` infers concrete return types from `get()` calls. This is the
foundational building block for replacing the untyped `_shared_state` dict,
the bolted-on context fields, and the `setattr` side-channel.

## Requirements

1. Create `src/darkfactory/phase_state.py` with the `PhaseState` class.
2. `PhaseState` exposes `put(value)`, `get(key, default=...)`, and `has(key)`.
3. `put` uses `type(value)` as the key. `get` is `@overload`-ed so mypy
   infers the concrete type. `get` without a default raises `KeyError` if
   the key is absent.
4. Create `src/darkfactory/phase_state_test.py` with unit tests covering:
   put/get round-trip, get with default, get missing raises KeyError, has
   for present/absent keys, overwrite semantics.
5. mypy strict passes on the new module.

## Technical Approach

### `src/darkfactory/phase_state.py`

```python
from __future__ import annotations
from typing import Any, TypeVar, overload

T = TypeVar("T")
_MISSING = object()

class PhaseState:
    """Type-keyed registry for inter-task data."""

    def __init__(self) -> None:
        self._store: dict[type, Any] = {}

    def put(self, value: object) -> None:
        """Store a value keyed by its runtime type."""
        self._store[type(value)] = value

    @overload
    def get(self, key: type[T]) -> T: ...
    @overload
    def get(self, key: type[T], default: T) -> T: ...

    def get(self, key: type[T], default: Any = _MISSING) -> T:
        if default is _MISSING:
            return self._store[key]  # type: ignore[return-value]
        return self._store.get(key, default)  # type: ignore[return-value]

    def has(self, key: type) -> bool:
        return key in self._store
```

### `src/darkfactory/phase_state_test.py`

Tests covering:
- `put` then `get` returns the same object with the correct type
- `get` with default returns default when key absent
- `get` without default raises `KeyError` when key absent
- `has` returns `True`/`False` correctly
- Second `put` of same type overwrites the first
