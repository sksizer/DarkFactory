# Add pagination or warning to reconcile PR fetching

## Summary

`_get_merged_prd_prs()` in `cli/reconcile.py` fetches up to 200 merged PRs via `gh pr list --limit 200`. For a long-lived project with hundreds of merged prd/* PRs, this silently misses any beyond the first 200. There is no pagination and no truncation warning.

## Technical approach

**Option A (minimal):** After the `gh pr list` call, check if the result count equals the limit. If so, log a warning to stderr: `"Warning: reconcile found {limit} merged PRs (at limit). Older PRs may be missed. Run 'prd reconcile' more frequently or increase --limit."`. Add `--limit` flag to reconcile subparser.

**Option B (thorough):** Implement pagination -- call `gh pr list` in a loop with `--limit 100` and cursor-based pagination until exhausted. `gh` supports `--json` output with pagination info.

Option A is recommended: the 200-PR limit is generous for incremental reconciliation, and a warning makes the behavior transparent.

### Code location

`cli/reconcile.py` -- `_get_merged_prd_prs()` function. Look for `"--limit", "200"` in the subprocess call.

## Acceptance criteria

- [ ] AC-1: A warning is emitted to stderr when the result count equals the fetch limit
- [ ] AC-2: Warning message tells the user what to do
- [ ] AC-3: Existing reconcile behavior is unchanged for small PR counts
