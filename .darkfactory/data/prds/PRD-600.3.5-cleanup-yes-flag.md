# Add --yes flag to cleanup --all

## Summary

`prd cleanup --all` calls `input("Remove all N worktree(s)? [y/N] ")` for confirmation. This blocks in non-interactive contexts (CI, scripts, pipes). Add a `--yes` / `-y` flag to skip the confirmation prompt, following the convention of `apt -y`, `rm -f`, etc.

## Technical approach

1. Add `--yes` / `-y` to the cleanup subparser in `_parser.py`:
   ```python
   sub_cleanup.add_argument("--yes", "-y", action="store_true", help="Skip confirmation prompts")
   ```
2. In `cleanup.py`, `_cleanup_all()` function (around line 182): check `args.yes` before calling `input()`. If true, skip the prompt and proceed directly.

## Acceptance criteria

- [ ] AC-1: `prd cleanup --all --yes` removes all worktrees without prompting
- [ ] AC-2: `prd cleanup --all` (without --yes) still prompts for confirmation
- [ ] AC-3: Test covers the `--yes` bypass path

## Assessment (2026-04-11)

- **Value**: 4/5 — small but concrete quality-of-life for
  scripted usage. The `input()` call currently blocks CI.
- **Effort**: xs — add the flag, one `if args.yes: skip_input`
  branch. Note that `cleanup.py` already has `--force` (for
  deleting worktrees not on main). That's a different semantic
  (filesystem override, not prompt skip). Decide whether `--yes`
  is a new flag or whether `--force` should absorb the
  prompt-skip behavior; a new `--yes` flag is cleaner and matches
  convention (`apt -y`, `gh -y`).
- **Current state**: greenfield. Grep confirms `cleanup.py` has
  `--force` but not `--yes`.
- **Gaps**: all three ACs unmet.
