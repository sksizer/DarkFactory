# Post-agent results via PhaseState

## Summary

Make the unified engine unconditionally store `AgentResult` in `PhaseState`
after every agent invocation. Remove the ad-hoc post-agent state mutations:
`ctx.agent_output`, `ctx.agent_success`, `ctx.last_invoke_result`,
`ctx.model`, `ctx.invoke_count` from `ExecutionContext`, and the
`setattr(ctx, "_last_agent_result", result)` side-channel. Migrate all
consumers to read from `PhaseState` instead.

## Requirements

1. After every `invoke_claude()` call, the engine calls
   `ctx.state.put(AgentResult(...))` with the invocation result.
2. Remove `agent_output`, `agent_success`, `last_invoke_result`, `model`,
   and `invoke_count` from `ExecutionContext`.
3. Remove the `setattr(ctx, "_last_agent_result", result)` side-channel
   from `_run_agent` in `runner.py`.
4. Migrate all builtins/code that reads the removed fields to use
   `ctx.state.get(AgentResult)` instead.
5. If `invoke_count` is needed, derive it from a simple counter on the
   engine or a list of `AgentResult` entries (consider a list-based approach
   in `PhaseState` if multiple results need tracking).
6. All existing tests pass.

## Technical Approach

### Engine integration

In the unified dispatch engine's agent handling path:

```python
result = invoke_claude(...)
ctx.state.put(AgentResult(
    output=result.output,
    success=result.success,
    model=result.model,
    cost_usd=result.cost_usd,
))
```

### Field removal from ExecutionContext

Remove these fields from `ExecutionContext` in `workflow.py`:
- `agent_output: str | None`
- `agent_success: bool`
- `last_invoke_result: InvokeResult | None`
- `model: str | None` (the per-invocation model, not config model)
- `invoke_count: int`

### Consumer migration

Search for all reads of the removed fields. Each consumer migrates to:
```python
agent_result = ctx.state.get(AgentResult)
# use agent_result.output, agent_result.success, etc.
```

Update `system_builtins.py` and any other builtins that reference these
fields.

## Acceptance Criteria

- [ ] AC-1: `AgentResult` is stored in `PhaseState` after every agent invocation.
- [ ] AC-2: `agent_output`, `agent_success`, `last_invoke_result`, `model`, `invoke_count` removed from `ExecutionContext`.
- [ ] AC-3: No `setattr` side-channel for agent results.
- [ ] AC-4: All consumers migrated to read from `PhaseState`.
- [ ] AC-5: All existing tests pass.
