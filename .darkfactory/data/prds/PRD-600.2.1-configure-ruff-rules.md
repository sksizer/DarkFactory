# Configure comprehensive ruff rule set

## Summary

The project has no `[tool.ruff]` or `[tool.ruff.lint]` configuration. Ruff runs with only default rules (E/F). Enable a broader rule set covering isort, pyupgrade, bugbear, security, and pytest style.

## Technical approach

Add to `pyproject.toml`:

```toml
[tool.ruff]
target-version = "py312"

[tool.ruff.lint]
select = ["E", "F", "I", "UP", "B", "S", "RUF", "SIM", "PT"]

[tool.ruff.lint.per-file-ignores]
"*_test.py" = ["S101"]  # assert is fine in tests
"tests/*" = ["S101"]
"src/darkfactory/builtins/__init__.py" = ["F401"]  # re-exports
"src/darkfactory/cli/__init__.py" = ["F401"]  # re-exports
```

Rule categories:
- `I` (isort): import sorting
- `UP` (pyupgrade): Python 3.12+ modernizations
- `B` (bugbear): common Python gotchas
- `S` (bandit): security checks (catches shell=True, subprocess, etc.)
- `RUF` (ruff-specific): ruff's own rules
- `SIM` (simplify): simplifiable code patterns
- `PT` (pytest-style): test best practices

After adding config, run `ruff check --fix src tests` to auto-fix safe violations, then manually address any remaining.

## Acceptance criteria

- [ ] AC-1: `[tool.ruff.lint]` section exists in `pyproject.toml` with at least `E,F,I,UP,B,S,RUF,SIM,PT`
- [ ] AC-2: `target-version = "py312"` is set
- [ ] AC-3: Per-file-ignores configured for test files and re-export modules
- [ ] AC-4: `uv run ruff check src tests` passes clean
- [ ] AC-5: `uv run ruff format --check src tests` passes clean
