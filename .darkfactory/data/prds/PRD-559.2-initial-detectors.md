# Implement initial transcript detectors

## Summary

Implement the seven initial detectors specified in PRD-559 and unit test each
one against fixture transcripts. All detectors are pure Python functions with
no subprocess or network calls. Each uses the `@detector` decorator from
PRD-559.1 to self-register.

## Requirements

1. Implement all seven initial detectors:
   - `sentinel_failure` — final assistant message lacks `PRD_EXECUTE_OK` or
     contains `PRD_EXECUTE_FAILED`.
   - `tool_denied` — events indicating the harness blocked a tool call.
   - `retry_count` — count of harness-driven retries (`task: implement-retry`
     header records).
   - `repeated_edit` — same `Edit`/`Write` to the same file path back-to-back.
   - `large_thinking_burst` — single thinking block over N tokens (configurable
     threshold, sensible default).
   - `forbidden_attribution_attempt` — patterns matching attribution strings
     (advisory, not fatal).
   - `tool_overuse` — any tool type count exceeding a configurable threshold
     (e.g. 50 Read calls).
2. Each detector must be a pure function with no side effects.
3. Unit tests must exercise each detector with fixture JSONL data covering
   both positive (finding produced) and negative (no finding) cases.
4. A test must verify that registering a fake detector via `@detector` works
   correctly (AC-3 of the parent PRD).

## Technical Approach

### File: `src/darkfactory/builtins/analyze_transcript_detectors.py`

Add the seven detector functions below the registry infrastructure from
PRD-559.1. Each follows the pattern:

```python
@detector("sentinel_failure")
def detect_sentinel_failure(events: list[dict]) -> list[Finding]:
    # scan events for final assistant message
    ...
```

Detector-specific logic:

- **sentinel_failure**: filter events for `role: assistant`, check last one
  for `PRD_EXECUTE_OK` / `PRD_EXECUTE_FAILED` in content.
- **tool_denied**: filter events with type indicating tool denial (check
  existing transcript JSONL schema for the exact field names).
- **retry_count**: count events with `task: implement-retry` in header/metadata.
  Emit `warning` if count >= 1, `error` if count >= 3.
- **repeated_edit**: track `(tool, file_path)` pairs in sequence, flag
  consecutive duplicates.
- **large_thinking_burst**: check thinking block token counts against threshold
  (default 2000 tokens). Emit `info` for large blocks.
- **forbidden_attribution_attempt**: regex scan for attribution patterns
  (e.g. "Co-Authored-By: Claude", "Generated by Claude"). Emit `warning`.
- **tool_overuse**: count tool calls by type, flag any exceeding threshold
  (default 50). Emit `warning`.

### File: `src/darkfactory/builtins/analyze_transcript_detectors_test.py`

Create fixture JSONL data as Python dicts (not files) for each detector.
Test structure:

```python
def test_detect_sentinel_failure_positive():
    events = [{"role": "assistant", "content": "PRD_EXECUTE_FAILED: reason"}]
    findings = detect_sentinel_failure(events)
    assert len(findings) == 1
    assert findings[0].severity == "error"

def test_detect_sentinel_failure_negative():
    events = [{"role": "assistant", "content": "PRD_EXECUTE_OK: PRD-123"}]
    findings = detect_sentinel_failure(events)
    assert len(findings) == 0
```

Include a test that registers a fake detector to verify the registry:

```python
def test_custom_detector_registration():
    @detector("test_custom")
    def my_detector(events):
        return []
    assert "test_custom" in DETECTORS
```
