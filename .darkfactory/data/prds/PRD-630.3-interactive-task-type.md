# InteractiveTask type in task hierarchy

## Summary

Add `InteractiveTask` as a new member of the `Task` hierarchy alongside
`AgentTask` and `BuiltIn` in `workflow.py`. This type represents tasks that
take over the terminal via `spawn_claude()` (interactive mode) rather than
running headless via `invoke_claude()`. The type is defined here; dispatch
logic is wired in the unified engine task (PRD-630.5).

## Requirements

1. Add `InteractiveTask` dataclass to `src/darkfactory/workflow.py`,
   extending from `Task`.
2. Fields: `name: str`, `prompt_file: str`, `tools: list[str]` (default
   empty), `model: str | None` (default None), `effort_level: EffortLevel | None`
   (default None).
3. Update any `Task` union types or discriminators to include
   `InteractiveTask`.
4. Add basic unit tests in the appropriate test file.
5. mypy strict passes.

## Technical Approach

### `src/darkfactory/workflow.py`

```python
@dataclass
class InteractiveTask(Task):
    name: str
    prompt_file: str
    tools: list[str] = field(default_factory=list)
    model: str | None = None
    effort_level: EffortLevel | None = None
```

If `Task` is a union type rather than a base class, add `InteractiveTask` to
the union. If there's an `isinstance` dispatch or match statement for tasks,
it does not need updating here — that belongs to the unified engine task.

### Tests

- Verify `InteractiveTask` can be instantiated with required and optional fields
- Verify it is recognized as a `Task` (isinstance check or union membership)

## Acceptance Criteria
