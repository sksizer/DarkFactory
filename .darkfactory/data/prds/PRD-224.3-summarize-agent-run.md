# summarize_agent_run BuiltIn + run-summary in create_pr

## Summary

Two pieces that compose together:

1. A new `summarize_agent_run` BuiltIn that reads the most recent `InvokeResult` from the context, aggregates tool-call counts, and writes the summary into `ctx.run_summary`.
2. The `create_pr` BuiltIn checks for `ctx.run_summary` and appends it to the PR body if present.

This is a `[primitive]` — composable into any workflow's task list.

## Requirements

1. Add a `run_summary: str | None` field to `ExecutionContext` in `workflow.py`.
2. Expose tool-call counts in `InvokeResult` (in `invoke.py`). Parse the existing `_summarize_stream_event` output or the raw tool-call events to aggregate counts by tool name.
3. Create a new `summarize_agent_run` BuiltIn that:
   - Reads the most recent `InvokeResult` from the context.
   - Aggregates: workflow name, model, agent invocation count/retries, tool-call counts, test results, lint results, sentinel value.
   - Formats the summary as a markdown section.
   - Writes it to `ctx.run_summary`.
4. Modify `create_pr` BuiltIn to append `ctx.run_summary` to the PR body if it is not None.
5. Update the default workflow's task list to insert `summarize_agent_run` before `create_pr`.
6. If `summarize_agent_run` is not in the workflow (ctx.run_summary is None), `create_pr` produces the same PR body as today — no regression.

## Technical Approach

### `python/darkfactory/workflow.py`

Add field to `ExecutionContext`:

```python
@dataclass
class ExecutionContext:
    # ... existing fields ...
    run_summary: str | None = None
```

### `python/darkfactory/invoke.py`

Expose tool-call counts on `InvokeResult`:

```python
@dataclass
class InvokeResult:
    # ... existing fields ...
    tool_counts: dict[str, int] = field(default_factory=dict)
```

Parse tool-call events during invocation to populate `tool_counts`.

### `python/darkfactory/builtins.py`

```python
def summarize_agent_run(ctx: ExecutionContext) -> None:
    """Aggregate tool-call counts and write a markdown summary to ctx.run_summary."""
    result = ctx.last_invoke_result
    if result is None:
        return

    lines = [
        "## Harness execution summary",
        "",
        f"- **Workflow:** {ctx.workflow_name}",
        f"- **Model:** {ctx.model}",
        f"- **Agent invocations:** {_format_invocations(ctx)}",
        f"- **Tools used:** {_format_tool_counts(result.tool_counts)}",
        f"- **Sentinel:** {result.sentinel or 'none'}",
    ]
    ctx.run_summary = "\n".join(lines)
```

Modify `create_pr`:

```python
def create_pr(ctx: ExecutionContext) -> None:
    body = _build_pr_body(ctx)
    if ctx.run_summary:
        body += "\n\n" + ctx.run_summary
    # ... existing gh pr create logic ...
```

### `workflows/default/workflow.py`

Insert `summarize_agent_run` in the task list just before `create_pr`.

### Tests

- Test `summarize_agent_run` with a mock context containing tool counts -> verify `ctx.run_summary` is set with correct format.
- Test `create_pr` with and without `ctx.run_summary` -> verify PR body is correct in both cases.
