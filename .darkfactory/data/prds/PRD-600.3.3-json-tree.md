# Add --json support to tree command

## Summary

`prd tree` displays the containment hierarchy with Rich-formatted tree output but has no `--json` mode. Downstream tools and scripts need the hierarchy as structured data. The `--json` global flag is silently ignored.

## Technical approach

`cmd_tree` currently uses `_format_tree_node` and `_print_tree` helpers to render Rich tree output. Add a JSON branch that emits the tree as nested objects:

```json
{
  "id": "PRD-500",
  "title": "Extract harness...",
  "status": "done",
  "kind": "epic",
  "children": [
    {"id": "PRD-501", "title": "Scaffold...", "status": "done", "kind": "task", "children": []},
    ...
  ]
}
```

For `prd tree` (full forest), emit a JSON array of root nodes. For `prd tree PRD-500` (rooted), emit a single root object.

## Acceptance criteria

- [ ] AC-1: `prd tree --json` emits JSON array of root trees
- [ ] AC-2: `prd tree PRD-500 --json` emits single root tree object
- [ ] AC-3: Each node has `id`, `title`, `status`, `kind`, `children` fields
- [ ] AC-4: Human-readable output is unchanged when `--json` is not passed
- [ ] AC-5: Test covers both full-forest and rooted JSON output
