# Extract `_run_shell_once` to shared runner utility

## Summary

`_run_shell_once` is copy-pasted verbatim between `runner.py:553-572` and `system_runner.py:356-373` (21 lines each). Any bug fix or behavior change must be applied to both files. Extract to a shared location.

## Technical approach

1. Move `_run_shell_once` from `runner.py` to a new module or keep it in `runner.py` and import it into `system_runner.py`
2. Since `system_runner.py` already imports from `runner.py` (it uses `RunResult`, `TaskStep`, `_task_kind`, `_task_name`), the simplest approach is to make `_run_shell_once` a public import from `runner.py`
3. In `system_runner.py`, replace the local `_run_shell_once` with `from .runner import _run_shell_once`

The function signature is `_run_shell_once(cmd: str, ctx: ExecutionContext | SystemContext, env: dict[str, str]) -> CompletedProcess[str]`. It only accesses `ctx.cwd` from the context, so it works with both context types.

Actually, since both `ExecutionContext` and `SystemContext` have a `cwd` attribute, the function can be typed against a `Protocol` or just use `Any` for the context and access `.cwd`. The current code uses `str(ctx.cwd)`.
