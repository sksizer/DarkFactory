# Add --version flag to CLI

## Summary

There is no `--version` flag on the argument parser. Users cannot determine which version of DarkFactory they are running, which is essential for bug reports and CI diagnostics.

## Technical approach

In `cli/_parser.py`, add after the parser construction:

```python
from darkfactory import __version__
parser.add_argument("--version", action="version", version=f"%(prog)s {__version__}")
```

The version is already defined in `src/darkfactory/__init__.py:5` as `__version__ = "0.1.0"`.

## Acceptance criteria

- [ ] AC-1: `prd --version` prints `prd 0.1.0` (or current version)
- [ ] AC-2: `darkfactory --version` prints `darkfactory 0.1.0`
- [ ] AC-3: The version string comes from `darkfactory.__version__`
