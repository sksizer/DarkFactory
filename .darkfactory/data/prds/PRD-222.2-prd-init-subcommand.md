# prd init subcommand

## Summary

Add a `prd init` subcommand that scaffolds the `.darkfactory/` directory structure in the current (or `--directory`-specified) project. This creates the standard directory layout, writes a starter `config.toml` with commented examples, and updates `.gitignore` to exclude runtime state directories. The command is idempotent — re-running on an already-initialized project reports "already initialized" and makes no changes.

## Requirements

1. `prd init` creates the following structure at the target project root:
   - `.darkfactory/prds/` (empty)
   - `.darkfactory/workflows/` (empty)
   - `.darkfactory/config.toml` (skeleton with commented examples)
   - `.darkfactory/worktrees/` (empty, will be git-ignored)
   - `.darkfactory/transcripts/` (empty, will be git-ignored)
2. Updates (or creates) `.gitignore` at the project root to include:
   - `.darkfactory/worktrees/`
   - `.darkfactory/transcripts/`
3. The command requires a `.git/` directory in the target — if absent, error with "Not a git repository. Run `git init` first."
4. Idempotent: if `.darkfactory/` already exists with the expected structure, print "Already initialized" and exit 0 without modifying anything.
5. If `.darkfactory/` partially exists (e.g. `prds/` exists but `workflows/` does not), fill in the missing pieces without overwriting existing files.
6. The `config.toml` skeleton should contain commented-out examples for common settings (model defaults, timeout overrides) so users can see what's available.

### Changes to `python/darkfactory/cli.py`

- Register `init` subparser in `build_parser()` alongside existing subcommands.
- The `init` handler does NOT require `.darkfactory/` to already exist (bypass discovery for this one subcommand).
- Wire `args.func` to call `init_project(target)`.

### `tests/test_init.py`

- Test scaffolding in a fresh git repo (tmp_path with `git init`).
- Test idempotency — run twice, second run reports "Already initialized".
- Test partial fill-in — remove one subdir, re-run, only missing piece is created.
- Test `.gitignore` creation and update (append without duplicating entries).
- Test error when no `.git/` directory exists.

- [ ] AC-4: Partial `.darkfactory/` is completed without overwriting existing files.
