# Extract reconcile domain logic from CLI handler

## Summary

`cmd_reconcile` contains full reconciliation logic: fetching merged PRs via `gh`, checking merge-commit ancestry, updating PRD frontmatter, and creating reconciliation PRs/commits -- all in the CLI handler. These are domain operations that should be callable from a server context.

## Technical approach

After PRD-600.1.1 (crash recovery) and PRD-556.17 (move to reconcile.py), extract a `reconcile` domain module:

1. Create `src/darkfactory/reconcile.py` with:
   - `find_merged_prd_prs(repo_root: Path) -> list[dict]` -- wraps `gh pr list`
   - `identify_stale_statuses(prs: list[dict], prds: dict[str, PRD]) -> list[tuple[str, str]]` -- pure logic
   - `apply_reconciliation(stale: list[tuple], prd_dir: Path, *, method: Literal["pr", "commit_to_main"]) -> str` -- git operations

2. `cmd_reconcile` becomes: parse args -> call domain functions -> format output

Depends on PRD-600.1.1 (crash recovery should be in place before extracting) and PRD-556.17 (reconcile must be in its own CLI module first).

## Acceptance criteria

- [ ] AC-1: Reconciliation logic lives in a domain module, not the CLI handler
- [ ] AC-2: Domain functions are independently testable
- [ ] AC-3: `cmd_reconcile` is a thin shell
- [ ] AC-4: Existing `tests/test_cli_reconcile.py` tests pass
