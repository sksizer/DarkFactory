# Add pytest-cov and coverage reporting to CI

## Summary

No test coverage measurement exists. Add `pytest-cov` to dev dependencies, configure coverage reporting, and add a coverage floor to prevent regressions.

## Technical approach

1. Add `pytest-cov>=6.0` to `[dependency-groups] dev` in `pyproject.toml`
2. Add coverage configuration to `pyproject.toml`:
   ```toml
   [tool.coverage.run]
   source = ["python/darkfactory"]
   omit = ["*_test.py", "*/conftest.py"]

   [tool.coverage.report]
   fail_under = 70
   show_missing = true
   skip_covered = true
   ```
3. Add a `just coverage` recipe: `uv run pytest --cov=python/darkfactory --cov-report=term-missing`
4. Update CI to run coverage: `uv run pytest --cov=python/darkfactory --cov-report=term-missing --cov-fail-under=70`

Set the initial floor conservatively at 70% -- this can be raised as `style.py` and other untested modules get coverage.

## Acceptance criteria

- [ ] AC-1: `pytest-cov` is in dev dependencies
- [ ] AC-2: `just coverage` runs tests with coverage reporting
- [ ] AC-3: CI fails if coverage drops below the configured floor
- [ ] AC-4: Coverage report shows per-file missing lines
