# Auto-recovery in ensure_worktree for merged/closed PRs

## Summary

When `ensure_worktree` detects that a branch exists but the worktree directory is gone, it currently raises `RuntimeError` with a message telling the user to run `prd cleanup`. In batch execution there is no user. This task replaces the unconditional raise with a call to `delete_branch_if_safe` — if the PR is merged or closed, delete the branch and proceed with fresh worktree creation; only raise if the PR is still open or state is unknown.

## Requirements

1. Replace the `RuntimeError` raise at `ensure_worktree.py:143-152` with a call to `delete_branch_if_safe`.
2. If the cleanup succeeds (`cleaned=True`), log the recovery and fall through to the `git worktree add` path below.
3. If the cleanup is refused (`cleaned=False`), raise the existing `RuntimeError` (preserving the current behavior for open/unknown PRs).
4. Also handle the case where the remote branch exists but local does not — fetch and delete the remote tracking ref if PR is merged/closed.
5. Unit tests covering: auto-recovery on merged PR, auto-recovery on closed PR, raise preserved on open PR.

## Technical approach

```python
# In ensure_worktree.py, replace lines 143-152:
local_exists = _branch_exists_local(ctx.repo_root, branch)
remote_exists = _branch_exists_remote(ctx.repo_root, branch)
if local_exists or remote_exists:
    # Try auto-recovery before raising.
    from darkfactory.builtins._shared import delete_branch_if_safe

    result = delete_branch_if_safe(ctx.repo_root, branch)
    if result.cleaned:
        ctx.logger.info(
            "auto-recovered stale branch %s (PR %s) — creating fresh worktree",
            branch, result.pr_state.lower(),
        )
        # Fall through to git worktree add below.
    else:
        lock.release()
        ctx._worktree_lock = None
        raise RuntimeError(
            f"branch {branch!r} already exists but worktree {worktree_path} is gone. "
            f"{result.reason}. Run `prd cleanup {ctx.prd.id}` to release it."
        )
```

For remote-only branches, `delete_branch_if_safe` should also handle deleting the remote tracking ref with `git branch -D -r origin/<branch>` if present.
