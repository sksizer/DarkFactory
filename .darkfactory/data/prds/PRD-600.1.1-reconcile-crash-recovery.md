# Add crash recovery to `_create_reconcile_pr`

## Summary

`_create_reconcile_pr` in `cli/reconcile.py` executes 6 sequential destructive git/gh subprocess calls (`git branch -D`, `git checkout -b`, `git add`, `git commit`, `git push`, `gh pr create`) with zero rollback logic. A failure at any point after `git checkout -b` leaves the user stranded on a non-main branch with partial changes and no indication of what to do. (Reconcile was extracted from `cli/__init__.py` to its own `cli/reconcile.py` module as part of the CLI split; the crash-recovery gap still exists.)

## Technical approach

1. Record the current branch at function entry: `original_branch = subprocess.run(["git", "rev-parse", "--abbrev-ref", "HEAD"], ...).stdout.strip()`
2. Wrap the entire subprocess chain in `try/finally`
3. In the `finally` block, check if we're still on the reconcile branch and `git checkout` back to `original_branch`
4. If the reconcile branch has uncommitted changes from a partial failure, reset them before switching: `git checkout -- .`
5. Log a clear message about what happened and what the user should do

### Code location

`cli/reconcile.py` -- look for `_create_reconcile_pr` function (around line 112).

The destructive sequence is:
- `git branch -D prd/reconcile-status-YYYYMMDD` via `git_check` (deletes existing branch; this one tolerates failure)
- `git checkout -b <branch>` via `git_run` (switches HEAD away from main)
- `git add`, `git commit`, `git push`, `gh pr create`

### Edge cases

- If `git branch -D` fails (branch doesn't exist) -- this is OK, the subsequent `checkout -b` will work. The current code already uses `git_check` for the delete (which tolerates failure).
- If `git push` fails (network error) -- the commit is local, user is on reconcile branch. The finally block should switch back and leave the branch for manual cleanup.
- If the user was already on a non-main branch when reconcile started -- respect that and return to it.

## Acceptance criteria

- [ ] AC-1: `_create_reconcile_pr` wraps its subprocess chain in try/finally
- [ ] AC-2: On any failure, the function returns to the original branch
- [ ] AC-3: On failure, an actionable error message is printed (which step failed, what to do)
- [ ] AC-4: `git branch -D` continues to tolerate "branch not found" gracefully
- [ ] AC-5: Existing reconcile tests (e.g. `cli/reconcile_test.py`) still pass
- [ ] AC-6: New test covers the partial-failure recovery path
