# Wire PhaseState into contexts and migrate shared state

## Summary

Add a `state: PhaseState` field to both `ExecutionContext` and `SystemContext`.
Migrate all current inter-task data passing to use `PhaseState`:

- Replace `SystemContext._shared_state` dict usage with `PhaseState.put/get`
  of `PrdContext`.
- Move rework-specific fields (`pr_number`, `review_threads`,
  `comment_filters`, `reply_to_comments`) from `ExecutionContext` into
  `ReworkContext` stored in `PhaseState`.
- Update all builtins that produce or consume these values.

## Requirements

1. Add `state: PhaseState` field to `ExecutionContext` in `workflow.py`.
2. Add `state: PhaseState` field to `SystemContext` in `system.py`.
3. `gather_prd_context` builtin writes `PrdContext` to `state` instead of
   `ctx._shared_state["prd_context"]`.
4. `discuss_prd` builtin reads `PrdContext` from `state` instead of
   `ctx._shared_state["prd_context"]`.
5. Remove `_shared_state` from `SystemContext`.
6. `resolve_rework_context` builtin writes `ReworkContext` to `state` instead
   of setting individual fields on `ExecutionContext`.
7. Rework consumers read `ReworkContext` from `state` instead of context fields.
8. Remove `pr_number`, `review_threads`, `comment_filters`,
   `reply_to_comments` from `ExecutionContext`.
9. Update all test setup that constructs these contexts.
10. All existing tests pass.

## Technical Approach

### Context changes

In `workflow.py`, add to `ExecutionContext`:
```python
state: PhaseState = field(default_factory=PhaseState)
```

In `system.py`, add to `SystemContext`:
```python
state: PhaseState = field(default_factory=PhaseState)
```

Remove `_shared_state: dict[str, Any]` from `SystemContext`.
Remove `pr_number`, `review_threads`, `comment_filters`,
`reply_to_comments` from `ExecutionContext`.

### Builtin migrations

- `gather_prd_context.py`: replace `ctx._shared_state["prd_context"] = ...`
  with `ctx.state.put(PrdContext(...))`.
- `discuss_prd.py`: replace `ctx._shared_state["prd_context"]` reads with
  `ctx.state.get(PrdContext)`.
- `resolve_rework_context.py`: replace setting `ctx.pr_number = ...` etc.
  with `ctx.state.put(ReworkContext(...))`.
- Rework template builtins: replace `ctx.pr_number` reads with
  `ctx.state.get(ReworkContext).pr_number`.

## Acceptance Criteria

- [ ] AC-1: Both `ExecutionContext` and `SystemContext` have `state: PhaseState`.
- [ ] AC-2: `_shared_state` is removed from `SystemContext`.
- [ ] AC-3: Rework fields removed from `ExecutionContext`.
- [ ] AC-4: `gather_prd_context` and `discuss_prd` use `PrdContext` via `PhaseState`.
- [ ] AC-5: Rework builtins use `ReworkContext` via `PhaseState`.
- [ ] AC-6: All existing tests pass.
