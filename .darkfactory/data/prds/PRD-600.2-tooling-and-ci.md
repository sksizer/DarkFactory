# Tooling and CI hardening

## Summary

Strengthen the development toolchain and CI pipeline. Configures ruff with a comprehensive rule set, adds test coverage measurement, provides `--version` support, single-sources the version number, tests on Python 3.13, adds tests for the largest untested module (`style.py`), and removes a dead stub file.

## Motivation

The architectural review found that while mypy strict mode provides excellent type safety, the linting story has gaps: only default ruff rules (E/F) are active, and ruff isn't run in CI at all. Test coverage is unmeasured. The project claims `>=3.12` support but only tests on 3.12. These are low-effort, high-value improvements.

Depends on PRD-600.1 because ruff CI (600.1.4) should land first, then rules can be expanded (600.2.1).

## Acceptance criteria

- [ ] Ruff configured with `I,UP,B,S,RUF,SIM,PT` rules
- [ ] `pytest-cov` reports coverage in CI with a floor
- [ ] `prd --version` prints the current version
- [ ] Version defined in one place only
- [ ] CI tests on Python 3.12 and 3.13
- [ ] `style.py` has test coverage
- [ ] Dead `cli.py` stub is removed
