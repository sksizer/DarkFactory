# Integrate analyze_transcript into workflows

## Summary

Add `BuiltIn("analyze_transcript")` to the `default`, `planning`, and
`extraction` workflows, positioned after `commit_transcript` and before
the final `commit` step so the analysis file lands in the same commit as
the transcript.

## Requirements

1. Add `BuiltIn("analyze_transcript")` to all three workflow task lists.
2. Position it after `commit_transcript` and before the final
   `commit("ready for review")` (or equivalent final commit step).
3. Do not change any other workflow steps or ordering.
4. Verify all three workflows still pass lint and typecheck.

## Technical Approach

### File: `src/darkfactory/workflows/default/workflow.py`

Locate the task list where `commit_transcript` appears. Insert
`BuiltIn("analyze_transcript")` immediately after it:

```python
# ... existing steps ...
BuiltIn("commit_transcript"),
BuiltIn("analyze_transcript"),   # <-- new
BuiltIn("commit", message="ready for review"),
# ... remaining steps ...
```

### File: `src/darkfactory/workflows/planning/workflow.py`

Same pattern — find `commit_transcript`, insert `analyze_transcript` after it,
before the next commit step.

### File: `src/darkfactory/workflows/extraction/workflow.py`

Same pattern — find `commit_transcript`, insert `analyze_transcript` after it,
before the next commit step.
