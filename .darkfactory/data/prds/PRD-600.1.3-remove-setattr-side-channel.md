# Remove setattr side channel in runner

## Summary

`runner.py` stores the last agent result via two parallel paths: (1) the typed field `ctx.last_invoke_result` at line ~418, and (2) an untyped `setattr(ctx, "_last_agent_result", result)` at line ~423. The getattr at line ~235 reads back the untyped version. This is redundant -- the typed field already exists on `ExecutionContext` at `workflow.py:238`.

## Technical approach

1. In `runner.py`, line ~235: change `getattr(ctx, "_last_agent_result", None)` to `ctx.last_invoke_result`
2. In `runner.py`, line ~423: delete the `setattr(ctx, "_last_agent_result", result)` line
3. Delete the comment about "side channel" at the same location

The fix is 3 lines changed. The existing `ctx.last_invoke_result = result` at line ~418 already stores the same value in a typed field.
