# Add --json support to validate command

## Summary

`prd validate` checks for cycles, missing dependencies, orphan references, and other DAG integrity issues but has no `--json` output mode. CI systems need structured error/warning lists for automated processing. The `--json` global flag is silently ignored by this command.

## Technical approach

`cmd_validate` (in `python/darkfactory/cli/validate.py`, after the PRD-556 CLI split) currently collects errors and warnings into lists and prints them with the Styler. Add a JSON branch:

1. Collect all findings into a list of dicts: `{"level": "error"|"warning", "check": "cycle"|"missing_dep"|..., "prd_id": "PRD-NNN", "message": "..."}`
2. When `args.json` is true, `json.dumps` the findings list and print to stdout
3. Exit with code 1 if any errors exist, 0 otherwise (same as current behavior)

The findings are already computed as strings -- the change is to capture them structurally before formatting.

## Acceptance criteria

- [ ] AC-1: `prd validate --json` emits a JSON array of finding objects
- [ ] AC-2: Each finding has `level`, `check`, `prd_id`, and `message` fields
- [ ] AC-3: Exit code behavior is unchanged (1 on errors, 0 on clean)
- [ ] AC-4: Human-readable output is unchanged when `--json` is not passed
- [ ] AC-5: Test covers JSON output with at least one error and one warning case
