# Shell-escape user-controlled values in `format_string`

## Summary

`ExecutionContext.format_string()` interpolates `{prd_title}`, `{prd_slug}`, and other user-controlled values from YAML frontmatter into strings that are subsequently passed to `subprocess.run(cmd, shell=True)` in ShellTask execution. A PRD title containing `$(command)`, backticks, or shell metacharacters would be executed by the shell. While this is a self-harm vector (users write their own PRDs), it should be defended against.

## Technical approach

1. Import `shlex.quote` in `workflow.py`
2. In `ExecutionContext.format_string()` (workflow.py, around line 269), shell-escape the values that could contain user-controlled content before interpolation:
   - `prd_title` -- from YAML frontmatter `title:` field
   - `prd_slug` -- derived from title
   - `branch` / `branch_name` -- derived from id + slug
3. Leave `prd_id` unescaped -- it's validated against `PRD_ID_RE` regex (`^PRD-\d+(?:\.\d+)*$`) and cannot contain shell metacharacters
4. Leave `worktree`, `repo_root`, `cwd` unescaped -- these are filesystem paths controlled by the system

### Code location

`workflow.py:269` -- `format_string` method on `ExecutionContext`. The method uses `template.format(prd_title=self.prd.title, ...)`.

The values flow from:
- `src/darkfactory/model/_persistence.py` -- `title=str(fm.get("title", ""))` (raw from YAML frontmatter, inside `parse_prd`)
- `workflow.py:272` -- interpolated into format string (`ExecutionContext.format_string`)
- `runner.py` -- `ctx.format_string(task.cmd)` for ShellTask, then `subprocess.run(cmd, shell=True)`

### Important: Only escape for ShellTask consumption

The `format_string` method is also used by BuiltIn kwargs (commit messages, etc.) where shell escaping would be wrong. Two approaches:

**Option A (preferred):** Add a `shell_escape=False` parameter to `format_string()`. Have `_run_shell` in runner.py call `ctx.format_string(task.cmd, shell_escape=True)`.

**Option B:** Shell-escape only in the ShellTask dispatch path in runner.py, between `format_string()` and `subprocess.run()`. This is messier because the format string has already been expanded.

## Acceptance criteria

- [ ] AC-1: PRD titles with shell metacharacters (`$()`, backticks, `;`, `|`) do not execute when used in ShellTask commands
- [ ] AC-2: BuiltIn kwargs (commit messages) are NOT shell-escaped
- [ ] AC-3: Test with a PRD title like `test $(echo pwned)` verifying it's treated as literal text
- [ ] AC-4: Existing ShellTask tests in `tests/test_runner.py` still pass

## Assessment (2026-04-11)

- **Value**: 2/5 — the vulnerability is real but the threat model is
  self-harm (users author their own PRDs). Defense in depth, not an
  active incident.
- **Effort**: s — the PRD's Option A (`shell_escape=False` parameter on
  `format_string`) is a focused 30–60 line change: add parameter, apply
  `shlex.quote` to `prd_title` / `prd_slug` / `branch_name`, have
  `_run_shell_once` (or its caller at `runner.py:472`) pass
  `shell_escape=True`, add the `$(echo pwned)` regression test.
- **Current state**: greenfield (confirmed directly). `src/darkfactory/runner.py:593`
  still uses `shell=True`, `runner.py:472` formats the shell command via
  `ctx.format_string(task.cmd)`, and `format_string` does not escape
  anything today. Note: an earlier state-survey pass claimed the count
  was zero; that was a grep miss on the research side — the surface is
  real.
- **Gaps to fully implement**:
  - Add `shell_escape` parameter to `ExecutionContext.format_string`.
  - Call it from the ShellTask dispatch path in `runner.py` and the
  duplicated dispatch in `system_runner.py:367`.
  - Regression test using a `test $(echo pwned)` title.
  - Leave BuiltIn kwargs (`runner.py:398`) unescaped.
- **Recommendation**: do-next — bundle with PRD-600.1.1 and PRD-600.1.3
  as a single "Phase 1 safety" PR. Low effort, closes a real injection
  surface, and the regression test is easy. Also worth adding a CI
  guard that fails if `shell=True` appears outside the two known
  dispatchers — that's the real deterrent against re-introducing the
  problem elsewhere.
