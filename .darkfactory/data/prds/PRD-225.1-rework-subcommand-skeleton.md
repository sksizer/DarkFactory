# prd rework subcommand skeleton

## Summary

Add a `prd rework <PRD-X>` subcommand that resumes the existing worktree for a PRD and validates that rework is possible. This task establishes the CLI entry point and the resume-worktree path without any comment processing or agent invocation — it just verifies the worktree exists, the PR is open, and prints what it would do.

## Requirements

1. Register a `rework` subcommand in `cli.py` that accepts a PRD ID as a positional argument.
2. Accept optional flags: `--execute` (actually run the rework vs dry-run), `--all`, `--since <commit>`, `--reviewer <user>`, `--from-pr-comment <id>`, `--reply-to-comments`.
3. Resolve the PRD from the `prds/` directory and validate it exists and has status `review`.
4. Locate the existing worktree for the PRD. Refuse with a clear error if no worktree exists (the user needs to run `prd run` first).
5. Locate the open PR for the PRD's branch. Refuse if the PR is already merged or closed.
6. In dry-run mode (no `--execute`), print what would happen: worktree path, PR number, branch name.
7. When `--execute` is passed, set up the execution context pointing at the existing worktree and exit (the actual workflow invocation comes in PRD-225.4).

## Technical Approach

### `python/darkfactory/cli.py`

Add `cmd_rework` following the same pattern as `cmd_run`:

```python
def cmd_rework(args, console):
    prd_id = args.prd_id
    prd_path = resolve_prd(prd_id)
    meta = load_prd(prd_path)

    if meta.status != "review":
        console.print(f"[red]ERROR:[/red] {prd_id} is in '{meta.status}', not 'review'")
        raise SystemExit(1)

    worktree_path = find_worktree(prd_id)
    if worktree_path is None:
        console.print(f"[red]ERROR:[/red] No worktree found for {prd_id}. Run 'prd run {prd_id}' first.")
        raise SystemExit(1)

    pr_number = find_open_pr(meta.branch_name)
    if pr_number is None:
        console.print(f"[red]ERROR:[/red] No open PR found for {prd_id}")
        raise SystemExit(1)

    if not args.execute:
        console.print(f"Would rework {prd_id}")
        console.print(f"  Worktree: {worktree_path}")
        console.print(f"  PR: #{pr_number}")
        console.print(f"  Branch: {meta.branch_name}")
        return

    # Set up execution context for the rework workflow (PRD-225.4)
    ...
```

Register the subcommand:

```python
p_rework = sub.add_parser("rework", help="Address PR review feedback for a PRD")
p_rework.add_argument("prd_id", help="PRD ID to rework")
p_rework.add_argument("--execute", action="store_true")
p_rework.add_argument("--all", action="store_true", help="Include resolved threads")
p_rework.add_argument("--since", help="Only comments after this commit")
p_rework.add_argument("--reviewer", help="Only comments from this reviewer")
p_rework.add_argument("--from-pr-comment", help="Address a single comment by ID")
p_rework.add_argument("--reply-to-comments", action="store_true", help="Post replies on addressed comments")
p_rework.set_defaults(func=cmd_rework)
```

### Helper functions

`find_worktree(prd_id)` — check if a git worktree exists for the PRD's branch. Use `git worktree list --porcelain` and match the branch name.

`find_open_pr(branch_name)` — run `gh pr view <branch> --json number,state --jq '.number'` and check the state is `OPEN`.
