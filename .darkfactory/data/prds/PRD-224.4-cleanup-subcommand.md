# Worktree lifecycle: prd cleanup subcommand + status hygiene line

## Summary

Three coordinated changes for worktree lifecycle management:

1. Confirm and document that successful runs do NOT remove the worktree (workflow-composition decision, no code change).
2. A new `prd cleanup [PRD-X | --merged | --all]` subcommand that removes worktree dirs and local branches.
3. A hygiene line at the bottom of `prd status` output when stale worktrees exist.

This is a `[CLI surface]` — user-initiated maintenance, not a workflow step. The underlying logic lives in `darkfactory.checks`.

## Requirements

1. **`prd cleanup PRD-X`**: removes the worktree directory and deletes the local branch for PRD-X. Refuses if the PR is still open (protects rework loops per PRD-225). Refuses if local has unpushed commits unless `--force`.
2. **`prd cleanup --merged`**: removes all worktrees for PRDs whose PR is merged. Same safety checks per worktree.
3. **`prd cleanup --all`**: removes all worktrees with confirmation prompt. Warns about any with open PRs.
4. **`prd status` hygiene line**: when `count > 0` stale worktrees exist, show `"N worktrees for merged PRDs (run 'prd cleanup --merged' to remove)"` at the bottom. Only shown when count > 0.
5. Add `find_stale_worktrees` and `is_safe_to_remove` functions to `darkfactory.checks` for reuse by other tools.
6. The cleanup must NOT remove a worktree whose PR is still open.

### `src/darkfactory/cli.py`

```python
@app.command()
def cleanup(
    prd_id: str | None = None,
    merged: bool = False,
    all_: bool = typer.Option(False, "--all"),
    force: bool = False,
):
    """Remove worktrees for completed PRDs."""
    if prd_id:
        _cleanup_single(prd_id, force)
    elif merged:
        _cleanup_merged(force)
    elif all_:
        _cleanup_all(force)
    else:
        console.print("Specify PRD-X, --merged, or --all")
```

In `cmd_status`, append hygiene line:

```python
stale = find_stale_worktrees(repo_root)
```

### Tests (`tests/test_cli_cleanup.py`)

- Cleanup single PRD with merged PR -> removes worktree and branch.
- Cleanup single PRD with open PR -> refuses.
- Cleanup with unpushed commits -> refuses without --force, succeeds with --force.
- `--merged` removes only merged-PR worktrees.
- Status hygiene line shows when stale worktrees exist, hidden when none.
