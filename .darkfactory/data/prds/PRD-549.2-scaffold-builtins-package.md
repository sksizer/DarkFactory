# Scaffold builtins/ package with registry and shared helpers

## Summary

Convert `python/darkfactory/builtins.py` from a single module into a Python package by creating the `python/darkfactory/builtins/` directory with `__init__.py`, `_registry.py`, and `_shared.py`. Move the `@builtin` decorator, `BUILTIN_REGISTRY` dict, `BuiltInFunc` type alias, and shared helpers into their respective new modules. `__init__.py` re-exports the public API so all existing import sites continue to work unchanged.

**No builtins are moved in this task.** The old `builtins.py` file is replaced by the package directory, but all builtin functions remain importable from their current location via `__init__.py` re-exports. This is an empty functional change — all tests pass before and after.

## Requirements

1. Create `python/darkfactory/builtins/` as a Python package.
2. Move `BUILTINS`, `BuiltInFunc`, and the `@builtin` decorator into `python/darkfactory/builtins/_registry.py`.
3. Create `python/darkfactory/builtins/_registry_test.py` with unit tests for the decorator (duplicate registration raises `ValueError`, successful registration adds to dict).
4. Move shared helpers (`_run`, `_scan_for_forbidden_attribution`, `_FORBIDDEN_ATTRIBUTION_PATTERNS`) into `python/darkfactory/builtins/_shared.py`.
5. Create `python/darkfactory/builtins/_shared_test.py` with unit tests for shared helpers.
6. `python/darkfactory/builtins/__init__.py` imports all builtin submodules (to trigger `@builtin` registration) and re-exports the public API (`BUILTINS`, `BuiltInFunc`, `builtin`, and all nine builtin functions).
7. Every existing `from darkfactory.builtins import X` and `from darkfactory import builtins` import site continues to work unchanged.
8. All existing tests pass. `just test`, `just lint`, `just typecheck`, `just format-check` all clean.

## Technical Approach

### `python/darkfactory/builtins/_registry.py`

```python
from __future__ import annotations
from typing import Callable

BuiltInFunc = Callable[..., None]

BUILTINS: dict[str, BuiltInFunc] = {}

def builtin(name: str) -> Callable[[BuiltInFunc], BuiltInFunc]:
    def decorator(func: BuiltInFunc) -> BuiltInFunc:
        if name in BUILTINS:
            raise ValueError(f"duplicate builtin registration for {name!r}")
        BUILTINS[name] = func
        return func
    return decorator
```

### `python/darkfactory/builtins/_shared.py`

Move from the old `builtins.py`:
- `_run(ctx, cmd, *, check, capture)` — subprocess helper with dry-run support
- `_FORBIDDEN_ATTRIBUTION_PATTERNS` — tuple of compiled regexes
- `_scan_for_forbidden_attribution(text, *, source)` — raises on forbidden patterns

### `python/darkfactory/builtins/__init__.py`

```python
"""Built-in task primitives — re-export hub.

Imports every submodule so @builtin-decorated functions register on
package import. Re-exports the public API for backwards compatibility.
"""
from darkfactory.builtins._registry import BUILTINS, BuiltInFunc, builtin

# Import submodules to trigger registration (added by later PRDs as builtins move)
# For now, the builtin functions still live in this file until migrated.

__all__ = ["BUILTINS", "BuiltInFunc", "builtin", ...]
```

Initially, `__init__.py` contains all the builtin function definitions (copied from the old `builtins.py`) so nothing breaks. Each subsequent PRD (549.3–549.11) extracts one builtin into its own submodule and removes it from `__init__.py`.

### Migration strategy

1. Rename `python/darkfactory/builtins.py` → delete it.
2. Create `python/darkfactory/builtins/` directory.
3. Create `_registry.py`, `_shared.py`, `__init__.py`.
4. `__init__.py` initially holds all builtin implementations (moved verbatim from old file) plus imports from `_registry` and `_shared`.
5. Verify all tests pass — this is a no-op refactor at this point.
