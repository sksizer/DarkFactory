# Extract safe-branch-cleanup helper into builtins/_shared.py

## Summary

Both `ensure_worktree` (auto-recovery) and graph execution (pre-run cleanup) need the same logic: given a branch name and repo root, check the PR state via `gh` and delete the local branch if the PR is merged or closed. This task extracts that logic into a reusable helper so both consumers depend on a single, tested API rather than duplicating the check-and-delete pattern.

## Requirements

1. Add a `delete_branch_if_safe(repo_root: Path, branch: str) -> BranchCleanupResult` function to `builtins/_shared.py`.
2. The function must:
   - Check PR state via `gh pr list --head <branch> --state all --json state` (reuse the pattern from `checks.py:_get_pr_state`).
   - If the PR is MERGED or CLOSED, delete the local branch with `git branch -D <branch>`.
   - If the PR is OPEN or UNKNOWN, refuse and return the reason.
   - Handle `gh` not being installed (graceful fallback — refuse with explanation).
3. Return a result dataclass, not a bool, so callers get the reason for refusal.
4. Unit tests covering: merged PR, closed PR, open PR, unknown (no gh), branch doesn't exist.

## Technical approach

### Result dataclass

```python
@dataclass
class BranchCleanupResult:
    cleaned: bool
    reason: str  # human-readable explanation
    pr_state: str  # "MERGED" | "CLOSED" | "OPEN" | "UNKNOWN"
```

### Implementation

```python
def delete_branch_if_safe(repo_root: Path, branch: str) -> BranchCleanupResult:
    pr_state = _get_pr_state(branch, repo_root)  # reuse from checks.py
    if pr_state in ("MERGED", "CLOSED"):
        subprocess.run(
            ["git", "-C", str(repo_root), "branch", "-D", branch],
            check=True, capture_output=True, text=True,
        )
        return BranchCleanupResult(cleaned=True, reason=f"PR {pr_state.lower()}", pr_state=pr_state)
    return BranchCleanupResult(
        cleaned=False,
        reason=f"PR state is {pr_state}; refusing to delete",
        pr_state=pr_state,
    )
```

### Reuse of `_get_pr_state`

`checks.py` already has `_get_pr_state(branch, repo_root) -> str`. Either:
- Import it directly (it's module-private with `_` prefix, so rename to `get_pr_state` and make it public), or
- Duplicate the 10-line function in `_shared.py` to avoid a cross-module dependency.

Prefer making it public in `checks.py` and importing, since it's already tested there.
