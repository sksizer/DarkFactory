# Unified dispatch engine

## Summary

Extract the duplicated dispatch loop from `runner.py` and `system_runner.py`
into a single parameterized `run_tasks()` function. Both `runner.run_workflow()`
and `system_runner.run_system_operation()` become thin callers that supply
their own builtin registry, prompt composer, and model picker. All runner
features (event logging, styler passthrough, timeout resolution, dry-run,
retry-on-failure) are implemented once and available to both paths.

## Requirements

1. Create `run_tasks()` function (in `runner.py` or a new `engine.py`)
   parameterized on: tasks list, context, builtin registry, prompt composer,
   model picker, and optional model override.
2. `run_tasks()` handles: task dispatch (builtin, agent, shell), event logging,
   styler passthrough, timeout resolution, dry-run, retry-on-failure, and
   worktree lock lifecycle.
3. `runner.run_workflow()` calls `run_tasks()` with workflow-specific
   callables.
4. `system_runner.run_system_operation()` calls `run_tasks()` with
   system-specific callables.
5. System operations gain event logging, styler passthrough, and timeout
   resolution (previously workflow-only features).
6. `cli/system.py` passes `styler` to the engine.
7. System agent invocations produce colorized streaming output and emit
   `task_start`/`task_finish` events.
8. All existing tests pass.

## Technical Approach

### `run_tasks()` signature

```python
def run_tasks(
    tasks: list[Task],
    ctx: C,
    builtins: dict[str, Callable],
    compose_prompt: Callable[[AgentTask, C], str],
    pick_model: Callable[[AgentTask, C, str | None], str],
    model_override: str | None = None,
    event_writer: EventWriter | None = None,
    styler: Styler | None = None,
    timeout_config: TimeoutConfig | None = None,
    dry_run: bool = False,
) -> RunResult:
```

### Dispatch logic

The core loop iterates tasks and dispatches by type:
- `BuiltIn` → look up in builtins dict, call
- `AgentTask` → compose prompt, pick model, invoke agent
- `ShellTask` → run shell command
- `InteractiveTask` → `spawn_claude()` (added in PRD-630.8)

Extract common patterns from both runners:
- `TaskStep` tracking
- Success/failure counting
- Event emission
- Timeout handling
- Retry logic

### Migration path

1. Extract `run_tasks()` with the unified logic.
2. Refactor `run_workflow()` to call `run_tasks()`.
3. Refactor `run_system_operation()` to call `run_tasks()`.
4. Verify both paths work identically.
