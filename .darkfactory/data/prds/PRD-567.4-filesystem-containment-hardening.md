# Filesystem containment hardening: block absolute-path escapes

## Summary

The current containment model uses `--disallowed-tools "Edit(../)"` to prevent agents from escaping their worktree. This blocks relative-path traversal but not absolute-path escapes — an agent can `Write("/Users/.../DarkFactory/.darkfactory/data/prds/...")` and bypass the `../` check entirely. This was observed during PRD-560 when a planning agent in a worktree wrote child PRDs to the main repo.

This feature adds absolute-path blocking to `invoke.py` and a post-invocation verification step in the runner to catch any escapes that slip through.

## Acceptance criteria

- [ ] AC-1: Agents cannot use absolute paths to write files outside their worktree.
- [ ] AC-2: A post-invocation check detects and reports any out-of-worktree modifications.
