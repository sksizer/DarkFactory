# Single-source the package version

## Summary

Version `"0.1.0"` is duplicated in `pyproject.toml:3` and `src/darkfactory/__init__.py:5`. These must be kept in sync manually, which will eventually drift. Use hatchling's regex version plugin to derive the version from a single source.

## Technical approach

**Option A (recommended): hatch-regex-version**

1. Add `hatch-regex-version` to build-system requires:
   ```toml
   [build-system]
   requires = ["hatchling", "hatch-regex-version"]
   ```
2. Replace the static version:
   ```toml
   [project]
   dynamic = ["version"]

   [tool.hatch.version]
   source = "regex"
   path = "src/darkfactory/__init__.py"
   ```
3. Keep `__version__ = "0.1.0"` in `__init__.py` as the single source of truth.

**Option B: Read from pyproject.toml at runtime**

Use `importlib.metadata.version("darkfactory")` in `__init__.py`. Simpler but requires the package to be installed (not just importable from source tree).

## Acceptance criteria

- [ ] AC-1: Version is defined in exactly one location
- [ ] AC-2: `pip show darkfactory` and `darkfactory.__version__` agree
- [ ] AC-3: `uv build` produces a wheel with the correct version
- [ ] AC-4: `prd --version` (from PRD-600.2.3) shows the correct version
