# CLI --all flag and queue filter arguments

## Summary

Extend the `prd run` subparser with the `--all` flag and the queue
filter arguments (`--priority`, `--tag`, `--exclude`). Make `prd_id`
optional (currently required) and enforce mutual exclusivity: passing
both `--all` and a positional `prd_id` is a hard error.

This is a parser-only change ŌĆö no execution logic. The wiring into
`cmd_run` happens in PRD-563.4.

## Requirements

1. Make the `prd_id` positional argument optional (`nargs="?"`) on the
   `run` subparser.
2. Add `--all` flag (`action="store_true"`) ŌĆö selects drain-ready-queue
   mode.
3. Add `--priority` argument ŌĆö accepts `critical`, `high`, `medium`,
   `low`. Filters to PRDs at or above this priority.
4. Add `--tag` argument ŌĆö repeatable (`action="append"`), OR semantics.
5. Add `--exclude` argument ŌĆö repeatable (`action="append"`), excludes
   specific PRD IDs.
6. Validate mutual exclusivity: if `--all` is set and `prd_id` is also
   provided, exit with a clear error message. If neither `--all` nor
   `prd_id` is provided, also error.

## Technical Approach

### File: `python/darkfactory/cli/_parser.py`

In the `sub_run` section:

```python
sub_run.add_argument("prd_id", nargs="?", default=None)
sub_run.add_argument(
    "--all",
    dest="run_all",
    action="store_true",
    help="Drain the ready queue: run all ready PRDs without a target",
)
sub_run.add_argument(
    "--priority",
    default=None,
    choices=["critical", "high", "medium", "low"],
    help="Only run PRDs at or above this priority",
)
sub_run.add_argument(
    "--tag",
    dest="tags",
    action="append",
    default=None,
    help="Only run PRDs with this tag (repeatable, OR semantics)",
)
sub_run.add_argument(
    "--exclude",
    dest="exclude_ids",
    action="append",
    default=None,
    help="Exclude specific PRD IDs (repeatable)",
)
```

The mutual-exclusivity validation belongs in `cmd_run` (PRD-563.4)
since `argparse` mutually-exclusive groups don't cleanly handle
positional-vs-flag conflicts. The parser just makes `prd_id` optional.
