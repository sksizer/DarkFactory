# Delete system_runner.py and final cleanup

## Summary

Delete `system_runner.py` entirely now that all system operation execution
flows through the unified `run_tasks()` engine. Update all callers that
imported from `system_runner` to use the unified engine. Verify that no
backward-compatibility shims are needed (alpha, single user). Run the full
test and lint suite to confirm clean state.

## Requirements

1. Delete `src/darkfactory/system_runner.py`.
2. Update `cli/system.py` to call the unified engine directly instead of
   `system_runner.run_system_operation()`.
3. Update `cli/discuss.py` if it imports from `system_runner`.
4. Update any other modules that import from `system_runner`.
5. Remove any re-exports of `system_runner` symbols from `__init__.py` files.
6. `just test && just lint && just typecheck && just format-check` all clean.

## Technical Approach

### Deletion

```bash
rm src/darkfactory/system_runner.py
```

### Caller updates

In `cli/system.py`, replace:
```python
from darkfactory.system_runner import run_system_operation
```
with a call to `run_tasks()` using system-specific callables (which should
already exist from PRD-630.5).

### Grep check

Search the entire codebase for `system_runner` imports and references.
Update or remove every occurrence.

### Validation

Run the full suite:
```bash
just test && just lint && just typecheck && just format-check
```

## Acceptance Criteria

- [ ] AC-1: `system_runner.py` is deleted.
- [ ] AC-2: No imports of `system_runner` remain in the codebase.
- [ ] AC-3: `cli/system.py` uses the unified engine.
- [ ] AC-4: `just test && just lint && just typecheck && just format-check` all clean.
