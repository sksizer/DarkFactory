# Local polling rework-watch daemon

## Summary

A long-running local process `prd rework-watch` that polls open PRs for new review comments and triggers `prd rework <PRD-X>` automatically when unaddressed feedback appears. Runs on the user's machine, polling at a configurable interval.

Phase-2 piece — depends on the manual `prd rework` command being solid first (PRD-225.4).

## Current State

No implementation exists. Blocked on PRD-225.4 (rework workflow).

## Note on Architecture

This PRD was written before the module-per-concern principle was established. The proposed `rework_watch.py` at the package root should instead follow established patterns — potentially as part of a `rework/` subpackage or within the CLI module structure. The daemon/polling pattern may also overlap with future scheduled/automated execution features.

## Requirements

1. Register `prd rework-watch` subcommand with modes: foreground (default), `--daemon`, `--status`, `--pause`, `--resume`, `--stop`.
2. Poll open PRs at configurable interval (default 60s) whose head branch matches `prd/PRD-*`.
3. For each PR, fetch comments and compare against last-seen comment ID stored in `.darkfactory/state/rework-watch.json`.
4. New unresolved comments trigger `prd rework PRD-X` as a subprocess.
5. Rate limiting: max N rework cycles per PR per hour (default 3) to prevent thrashing.
6. Honor the per-PRD process lock so manual `prd run` doesn't race with the watcher.
7. Pause file at `.darkfactory/state/rework-watch.pause` halts without stopping the process.
8. Refuse to start if worktree for any active PR is missing.

## Acceptance Criteria

- [ ] `prd rework-watch` polls open PRs at the configured interval
- [ ] New unresolved comments trigger `prd rework PRD-X` automatically
- [ ] Rate limiting caps rework cycles to 3 per PR per hour by default
- [ ] Pause file halts polling without stopping the process
- [ ] Watcher refuses to start if any active PR's worktree is missing
- [ ] `--daemon`, `--status`, `--pause`, `--resume`, `--stop` modes work
- [ ] Tests cover: polling, rate limiting, pause, missing worktree, state persistence
