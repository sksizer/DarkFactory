# Pre-run stale worktree cleanup in graph execution

## Summary

Before the DAG execution loop begins, proactively find and clean stale worktrees whose PRs are merged or closed. This prevents stale state from accumulating across batch runs and complements the reactive auto-recovery in `ensure_worktree` (PRD-567.1.2).

This task depends on PRD-567.1.1 (the shared cleanup helper) but NOT on PRD-567.1.2 — both are independent consumers of the same API.

## Requirements

1. At the start of `execute_graph` (or the CLI entry point that calls it), call `find_stale_worktrees` from `checks.py` to discover worktrees with merged/closed PRs.
2. For each stale worktree, call `delete_branch_if_safe` to remove the branch, then `git worktree remove` to clean up the worktree directory.
3. Log each cleanup action at INFO level.
4. If cleanup fails for a specific worktree (e.g. dirty working tree), log a warning and continue — do not block the entire execution run.
5. Only run cleanup when `--execute` is set (not during dry-run).
6. Unit tests covering: stale worktrees cleaned, cleanup failure is non-fatal, dry-run skips cleanup.

## Technical approach

```python
# In graph_execution.py, at the top of execute_graph():
def _cleanup_stale_worktrees(repo_root: Path, logger: logging.Logger) -> None:
    """Clean stale worktrees before DAG execution."""
    from darkfactory.builtins._shared import delete_branch_if_safe
    from darkfactory.checks import find_stale_worktrees, is_safe_to_remove

    stale = find_stale_worktrees(repo_root)
    for wt in stale:
        status = is_safe_to_remove(wt, force=False)
        if not status.safe:
            logger.warning("skipping stale worktree %s: %s", wt.branch, status.reason)
            continue
        result = delete_branch_if_safe(repo_root, wt.branch)
        if result.cleaned:
            try:
                subprocess.run(
                    ["git", "-C", str(repo_root), "worktree", "remove", str(wt.worktree_path)],
                    check=True, capture_output=True, text=True,
                )
                logger.info("cleaned stale worktree: %s (PR %s)", wt.branch, result.pr_state.lower())
            except subprocess.CalledProcessError as exc:
                logger.warning("worktree remove failed for %s: %s", wt.branch, exc.stderr)
        else:
            logger.debug("skipping %s: %s", wt.branch, result.reason)
```

Call `_cleanup_stale_worktrees(repo_root, logger)` at the top of the execution path, gated on `not dry_run`.

## Acceptance criteria

- [ ] AC-1: `prd run --all --execute` cleans stale worktrees (merged/closed PRs) before starting the DAG loop.
- [ ] AC-2: Cleanup failures for individual worktrees are logged as warnings and do not block execution.
- [ ] AC-3: Dry-run mode (`prd run --all` without `--execute`) does not perform cleanup.
- [ ] AC-4: Cleaned worktrees are logged at INFO level with branch name and PR state.
- [ ] AC-5: `just test`, `just lint`, `just typecheck` all pass.
