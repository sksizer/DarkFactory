# Add Python 3.13 to CI test matrix

## Summary

`pyproject.toml` claims `requires-python = ">=3.12"` but CI only tests on Python 3.12. Python 3.13 is now released and may have behavioral differences (e.g., faster `tomllib`, changes to some stdlib modules). Testing on 3.13 validates the version claim.

## Technical approach

Convert the CI workflow to use a matrix strategy:

```yaml
strategy:
  matrix:
    python-version: ["3.12", "3.13"]
steps:
  - uses: actions/checkout@v4
  - uses: astral-sh/setup-uv@v6
  - run: uv python install ${{ matrix.python-version }}
  - run: uv sync
  - run: uv run pytest
  - run: uv run mypy src tests
```

## Acceptance criteria

- [ ] AC-1: CI runs tests on both Python 3.12 and 3.13
- [ ] AC-2: All tests pass on both versions
- [ ] AC-3: mypy passes on both versions
