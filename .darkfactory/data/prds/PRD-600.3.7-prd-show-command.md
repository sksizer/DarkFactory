# Add `prd show` command for single-PRD inspection

## Summary

There is no way to display all metadata for a single PRD from the command line. Users must use `prd plan PRD-NNN` (shows workflow info, not just PRD data) or open the file manually. Every entity-management CLI needs a basic `show`/`info`/`describe` command.

## Technical approach

1. Add `show` subparser to `_parser.py`:
   ```python
   sub_show = sub.add_parser("show", help="Display metadata for a single PRD")
   sub_show.add_argument("prd_id", help="PRD ID (e.g. PRD-042)")
   sub_show.set_defaults(func=cmd_show)
   ```

2. Create `src/darkfactory/cli/show.py` with `cmd_show(args)` (following the per-subcommand module pattern in `cli/`) and a peer `cli/show_test.py`. Use `darkfactory.model.load_one(args.data_dir, args.prd_id)` to load the PRD. `cmd_show` should display:
   - All frontmatter fields (id, title, kind, status, priority, effort, capability, etc.)
   - Parent and children
   - Dependencies (depends_on, blocks)
   - Assigned workflow
   - File path

3. With `--json`, emit the PRD as a JSON object with all frontmatter fields.

4. Human-readable output uses Styler for formatting, similar to `prd plan` output but focused on metadata, not workflow tasks.

## Acceptance criteria

- [ ] AC-1: `prd show PRD-042` displays all frontmatter metadata
- [ ] AC-2: Output includes parent, children, dependencies, and assigned workflow
- [ ] AC-3: `prd show PRD-042 --json` emits a JSON object
- [ ] AC-4: Invalid PRD ID gives an actionable error message
- [ ] AC-5: Test covers both human-readable and JSON output
