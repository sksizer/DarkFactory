# Delete dead cli.py stub file

## Summary

`src/darkfactory/cli.py` is a 9-line stub that raises `ImportError` unconditionally. Its docstring says it exists "pending deletion once git rm can be run." Python's import system prefers the `cli/` package directory, so this file is never loaded. It should be deleted.

## Technical approach

```bash
git rm src/darkfactory/cli.py
```

Verify no imports reference `darkfactory.cli` in a way that would hit the file instead of the package (they can't -- Python resolves packages over modules).

## Acceptance criteria

- [ ] AC-1: `src/darkfactory/cli.py` no longer exists
- [ ] AC-2: All tests pass
- [ ] AC-3: `from darkfactory.cli import main` still works (resolves to `cli/__init__.py`)
