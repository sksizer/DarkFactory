# Typed data bundles for inter-task state

## Summary

Define the frozen dataclass bundles — `PrdContext`, `ReworkContext`, and
`AgentResult` — that will serve as typed keys into `PhaseState`. Each bundle
is declared near its producing module. This task creates the types only; wiring
them into the contexts and builtins is handled by subsequent tasks.

## Requirements

1. Create `PrdContext` frozen dataclass near `gather_prd_context.py` (either
   in the same file or a sibling `_types.py`).
2. Create `ReworkContext` frozen dataclass near `resolve_rework_context.py`.
3. Create `AgentResult` frozen dataclass near `invoke.py` (it is produced by
   the agent invocation layer).
4. All bundles use only stdlib types and existing project types.
5. mypy strict passes on all new and modified files.

## Technical Approach

### `PrdContext` (near `src/darkfactory/builtins/gather_prd_context.py`)

```python
@dataclass(frozen=True)
class PrdContext:
    summary: str
    body: str
    parent_ref: str | None = None
    dependency_refs: tuple[str, ...] = ()
```

### `ReworkContext` (near `src/darkfactory/builtins/resolve_rework_context.py`)

```python
@dataclass(frozen=True)
class ReworkContext:
    pr_number: int
    review_threads: list[ReviewThread]
    reply_to_comments: bool = False
    comment_filters: CommentFilters | None = None
```

The `ReviewThread` and `CommentFilters` types should already exist in the
rework module. If they are currently defined inline or as dicts, define
proper types as part of this task.

### `AgentResult` (near `src/darkfactory/invoke.py`)

```python
@dataclass(frozen=True)
class AgentResult:
    output: str
    success: bool
    model: str
    cost_usd: float | None = None
```
