# Define SYSTEM_OPERATION_TEMPLATE

## Summary

Define `SYSTEM_OPERATION_TEMPLATE` for non-PRD-bound system operations (PRD-223). System operations have a different lifecycle: they acquire a global lock on open, and release it + write a report file on close. No PR creation, no status transitions — system ops aren't tied to PRD state.

## Requirements

1. Define `SYSTEM_OPERATION_TEMPLATE` in `src/darkfactory/templates_builtin.py`.
2. Opening tasks:
   - `acquire_global_lock` — prevent concurrent system operations
   - `log_operation_start` — record the operation start time and context
3. Closing tasks:
   - `write_report` — write an operation report file
   - `log_operation_end` — record completion
   - `release_global_lock` — release the lock acquired on open
4. Middle allows `ShellTask` and `AgentTask`; no minimum count constraints (some system ops may be pure shell).
5. Add tests for the system operation template's compose/validation behavior.

## Technical Approach

### `src/darkfactory/templates_builtin.py`

```python
SYSTEM_OPERATION_TEMPLATE = WorkflowTemplate(
    name="system-operation",
    description="System operation lifecycle: lock, execute, report, unlock.",
    open=[
        BuiltIn("acquire_global_lock"),
        BuiltIn("log_operation_start"),
    ],
    middle_kinds=[AgentTask, ShellTask],
    middle_required={},
    close=[
        BuiltIn("write_report"),
        BuiltIn("log_operation_end"),
        BuiltIn("release_global_lock"),
    ],
)
```

### Tests

- Valid composition produces `[lock, log_start, *middle, report, log_end, unlock]`
- Empty middle is allowed (middle_required is empty)
- Disallowed task types in middle raise `TemplateViolation`

## Acceptance Criteria

- [ ] AC-1: `SYSTEM_OPERATION_TEMPLATE` is importable from `darkfactory.templates_builtin`.
- [ ] AC-2: The template's close list includes `release_global_lock` as the final task.
- [ ] AC-3: The template's open list includes `acquire_global_lock` as the first task.
- [ ] AC-4: Composing with an empty middle succeeds (no minimum count constraints).
- [ ] AC-5: Test coverage for valid composition and kind violations.
