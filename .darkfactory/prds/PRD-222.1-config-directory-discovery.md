# Config directory discovery and --directory flag

## Summary

Replace the current `_find_repo_root` / `_default_prd_dir` / `_default_workflows_dir` helpers in `cli.py` with a new `discovery.py` module that locates a project's `.darkfactory/` directory. Add a `--directory` / `-C` global CLI flag and `DARKFACTORY_DIR` env var to override the default. All existing subcommands resolve paths through the discovered `.darkfactory/` root instead of hardcoding `prds/` and `workflows/` relative to the git root.

## Requirements

1. Create `src/darkfactory/discovery.py` with a `find_darkfactory_dir(start: Path) -> Path | None` function that walks up from `start` looking for a `.darkfactory/` directory.
2. Add a `resolve_project_root(cli_dir: Path | None, env_var: str | None, cwd: Path) -> Path` function that applies the resolution order: CLI flag > env var > walk-up discovery from cwd.
3. If no `.darkfactory/` is found in any parent, the function returns `None`. The CLI should print a clear error: "No `.darkfactory/` directory found. Run `prd init` to set up this project." and exit 1.
4. Add `--directory` / `-C` as a global argument in `build_parser()` (line ~1499 in `cli.py`).
5. Read `DARKFACTORY_DIR` env var as fallback when `--directory` is not provided.
6. Replace `_find_repo_root`, `_default_prd_dir`, and `_default_workflows_dir` in `cli.py` with calls to the new discovery module. The `prd_dir` becomes `<discovered>/.darkfactory/prds/` and workflows dir becomes `<discovered>/.darkfactory/workflows/`.
7. No special-case detection for "am I in darkfactory's own repo" — the tool always looks for `.darkfactory/`.

## Technical Approach

### `src/darkfactory/discovery.py`

```python
from pathlib import Path
import os

def find_darkfactory_dir(start: Path) -> Path | None:
    """Walk up from start looking for a .darkfactory/ directory."""
    current = start.resolve()
    while True:
        candidate = current / ".darkfactory"
        if candidate.is_dir():
            return candidate
        parent = current.parent
        if parent == current:
            return None
        current = parent

def resolve_project_root(
    cli_dir: Path | None = None,
    cwd: Path | None = None,
) -> Path | None:
    """Resolve the .darkfactory directory.

    Priority: --directory CLI flag > DARKFACTORY_DIR env > walk-up from cwd.
    """
    if cli_dir is not None:
        df = cli_dir / ".darkfactory"
        return df if df.is_dir() else None

    env_dir = os.environ.get("DARKFACTORY_DIR")
    if env_dir:
        df = Path(env_dir) / ".darkfactory"
        return df if df.is_dir() else None

    return find_darkfactory_dir(cwd or Path.cwd())
```

### Changes to `src/darkfactory/cli.py`

- Remove `_find_repo_root` (line 58), `_default_prd_dir` (line 68), `_default_workflows_dir` (line 80).
- In `build_parser()`, add `--directory` / `-C` as a top-level argument before subparsers.
- In `main()`, call `resolve_project_root(args.directory)` and derive `prd_dir` and `workflows_dir` from the result.
- Subcommands that currently accept `--prd-dir` / `--workflows-dir` keep working as explicit overrides; the discovery path is only the default.
