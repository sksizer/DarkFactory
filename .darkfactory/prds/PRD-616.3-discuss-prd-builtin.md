# Implement discuss_prd system builtin

## Summary

Create the `discuss_prd` system builtin that loads a prompt file, substitutes context placeholders, prints a phase banner, and spawns an interactive Claude Code subprocess. The subprocess call is wrapped in a monkeypatchable function for test isolation. Non-zero exits log a warning but do not abort the chain.

## Requirements

1. New file `src/darkfactory/builtins/discuss_prd.py` with the builtin function.
2. Register `discuss_prd` in `SYSTEM_BUILTINS` via `@_register`.
3. Accept kwargs: `phase: str`, `prompt_file: str`, `instructions: str | None`.
4. Load the prompt file relative to the operation's directory (`commands/discuss/`).
5. Substitute `{PRD_CONTEXT}` with `ctx._shared_state["prd_context"]` and `{PHASE}` with `phase`.
6. Print a phase banner to stderr with the phase name and abort instructions.
7. Pause ~1 second after the banner for the user to abort.
8. Spawn `subprocess.run(["claude", composed_prompt], cwd=ctx.cwd, check=False)` via a `_spawn_discuss_prd` wrapper function.
9. On non-zero exit from the subprocess, log a warning but proceed (do not raise).
10. Peer test file at `src/darkfactory/builtins/discuss_prd_test.py`.

## Technical Approach

### discuss_prd.py

```python
def _spawn_discuss_prd(prompt: str, cwd: Path) -> int:
    """Monkeypatchable wrapper around subprocess.run for Claude Code."""
    result = subprocess.run(["claude", prompt], cwd=cwd, check=False)
    return result.returncode

def discuss_prd(
```

- Resolve prompt file path from `ctx.operation.operation_dir / prompt_file` (or from the commands/discuss package directory).
- Read the prompt template.
- Substitute placeholders.
- Print banner to stderr.
- Call `_spawn_discuss_prd(composed_prompt, ctx.repo_root)`.
- Log warning on non-zero return.

### Phase banner format

```
─────────────────────────────────────
 Phase: {phase}
 Press Ctrl-C now to abort the chain.
─────────────────────────────────────
```
