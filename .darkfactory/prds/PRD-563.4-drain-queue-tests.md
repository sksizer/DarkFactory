# Comprehensive tests for drain-ready-queue execution mode

## Summary

Write the test suite covering all queue-mode scenarios: discovery, filtering, ordering, execution, mid-run unlocking, failure isolation, and CLI argument validation. Tests should cover both the pure functions (unit tests) and the end-to-end CLI flow (integration tests using the injectable `run_workflow_fn`).

## Requirements

1. Test `deps_satisfied` with: all deps done, one dep not done, dep in review, missing dep, no deps.
2. Test `matches_filters` with: priority filtering, tag OR-matching, exclusion, combined filters.
3. Test `discover_ready_queue` with: empty PRD set, no ready PRDs, multiple ready PRDs sorted by priority then number, dependency chain ordering.
4. Test queue-mode execution (via `execute_graph` with `queue_candidates`):
   - Empty queue: no runs, exit 0.
   - Single ready PRD: runs it, exit 0.
   - Multiple independent PRDs ordered by priority.
   - Dependency chain unlocked mid-run (PRD becomes ready after its dep completes).
   - `--max-runs` cutoff.
   - Failure isolation: failed PRD is blocked, dependents skipped, unrelated PRDs continue, exit non-zero.
5. Test filter combinations: priority + tag, priority + exclude, all three.
6. Test CLI argument validation:
   - `--all` with positional PRD-ID is an error.
   - `--priority` / `--tag` / `--exclude` without `--all` is an error.
7. Test exclusion of transitive dependents when an excluded PRD breaks their deps.

## Technical Approach

### File: `tests/test_drain_ready_queue.py`

Use the same test patterns as the existing `tests/test_graph_execution.py`: build minimal PRD dicts in memory, inject a fake `run_workflow_fn`, and assert on the `ExecutionReport`.

#### Unit tests for pure functions

```python

    def test_all_deps_done(self): ...
    def test_dep_not_done(self): ...
    def test_dep_in_review(self): ...

    def test_no_deps(self): ...

class TestMatchesFilters:
    def test_priority_high_only(self): ...
    def test_tag_or_matching(self): ...
    def test_exclusion(self): ...
    def test_combined_filters(self): ...

class TestDiscoverReadyQueue:
    def test_empty_set(self): ...
    def test_no_ready_prds(self): ...
    def test_priority_ordering(self): ...
    def test_topo_ordering(self): ...
    def test_tiebreak_by_number(self): ...
```

#### Integration tests for queue execution

```python

    def test_empty_queue(self): ...
```

### Test fixtures

Create helper functions to build minimal PRD objects with specified `id`, `status`, `priority`, `depends_on`, and `tags` fields, avoiding the need to write YAML files to disk for unit tests.

## Acceptance Criteria

- [ ] AC-1: Unit tests for `deps_satisfied`, `matches_filters`, and `discover_ready_queue` cover all edge cases listed in Requirements.
- [ ] AC-2: Integration tests exercise queue execution end-to-end using the injectable `run_workflow_fn`.
- [ ] AC-3: Mid-run unlock scenario (dependency completes, dependent becomes ready) is tested.
- [ ] AC-4: Failure isolation test verifies failed PRD is blocked, dependents skipped, unrelated PRDs continue.
- [ ] AC-5: CLI validation tests confirm mutual exclusivity and filter-requires-all errors.
- [ ] AC-6: All tests pass with `uv run pytest tests/test_drain_ready_queue.py`.
