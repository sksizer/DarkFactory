# Add ruff check and format check to CI

## Summary

The CI workflow (`.github/workflows/ci.yml`) runs `pytest` and `mypy` but does not run `ruff check` or `ruff format --check`. Lint violations and formatting drift can merge undetected. This is the single highest-impact DevOps improvement identified by the review.

## Technical approach

Add two steps to the CI workflow after the existing `uv sync` step:

```yaml
- name: Lint
  run: uv run ruff check src tests

- name: Format check
  run: uv run ruff format --check src tests
```

Also add a wheel build verification step:

```yaml
- name: Build wheel
  run: uv build
```

### Code location

`.github/workflows/ci.yml:22-24` currently runs only:
```yaml
- run: uv run pytest
- run: uv run mypy src tests
```

### Note

This task uses the current ruff defaults (E/F rules). PRD-600.2.1 expands the rule set afterward. Landing CI enforcement first means any new rules added by 600.2.1 are immediately enforced.

## Acceptance criteria

- [ ] AC-1: CI runs `ruff check src tests` and fails on violations
- [ ] AC-2: CI runs `ruff format --check src tests` and fails on formatting drift
- [ ] AC-3: CI runs `uv build` and fails if the wheel can't be built
- [ ] AC-4: Current codebase passes all new CI checks (fix any existing violations first)
