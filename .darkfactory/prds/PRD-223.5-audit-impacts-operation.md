# audit-impacts system operation

## Summary

Implement the `audit-impacts` system operation — a read-only operation that walks all PRDs, reads their `impacts` field, and checks whether each declared file path actually exists on disk. Produces a report listing declared impacts that don't correspond to existing files, catching the kind of PRD-paths drift that has caused issues before.

This is a pure read-only operation (`creates_pr=False`) invoked via `prd system run audit-impacts`.

## Requirements

1. The operation must load all PRDs and iterate their `impacts` lists.
2. For each impact path, check existence relative to `ctx.repo_root`.
3. Report lines should include the PRD id, the declared path, and whether it's missing or present.
4. Output a summary: total impacts checked, total missing, list of missing paths grouped by PRD.
5. The operation is read-only — it never modifies PRD files or creates PRs.
6. Exit with non-zero status if any impacts are missing (useful for CI).

## Technical Approach

**New file**: `.darkfactory/operations/audit-impacts/operation.py`

```python
from darkfactory.system import SystemOperation
from darkfactory.workflow import BuiltIn

operation = SystemOperation(
    name="audit-impacts",
    description=(
        "Walk all PRDs, check that declared impact paths exist on disk. "
        "Reports missing paths — catches PRD-vs-repo drift."
    ),
    creates_pr=False,
    requires_clean_main=False,
    tasks=[
        BuiltIn("audit_impacts_check"),
    ],
)
```

**New builtin**: `audit_impacts_check` in `src/darkfactory/builtins/system_builtins.py` — iterates `ctx.prds`, checks each `impacts` path against `ctx.repo_root`, populates `ctx.report`.

**New file**: `tests/test_audit_impacts.py` — tests covering:
- Operation discovers and loads correctly
- Reports missing paths
- Reports clean when all paths exist
- Handles PRDs with empty impacts lists
