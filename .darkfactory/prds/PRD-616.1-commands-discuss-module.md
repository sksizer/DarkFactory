# Define commands/discuss module with SystemOperation and prompt files

## Summary

Create the `src/darkfactory/commands/discuss/` subpackage that defines the discuss `SystemOperation` and its two prompt files. This establishes the `commands/` organizational pattern for rich CLI commands and provides the operation definition that the CLI wiring (PRD-616.5) will import.

## Requirements

1. Create `src/darkfactory/commands/__init__.py` as an empty package marker.
2. Create `src/darkfactory/commands/discuss/__init__.py` exporting `discuss_operation: SystemOperation`.
3. Create `src/darkfactory/commands/discuss/operation.py` defining the `SystemOperation` with four tasks in order: `gather_prd_context`, `discuss_prd(phase="discuss")`, `discuss_prd(phase="critique")`, `commit_prd_changes`.
4. Create `src/darkfactory/commands/discuss/prompts/discuss.md` with collaborator framing and `{PRD_CONTEXT}` / `{PHASE}` placeholders.
5. Create `src/darkfactory/commands/discuss/prompts/critique.md` with critic framing and the same placeholders.

## Technical Approach

### operation.py

```python
from darkfactory.system import BuiltIn, SystemOperation

discuss_operation = SystemOperation(
    name="discuss",
    description="Interactive PRD discussion chain — gather, discuss, critique, commit.",
    requires_clean_main=False,
    creates_pr=False,
    accepts_target=True,
    tasks=[
        BuiltIn("gather_prd_context"),
        BuiltIn("discuss_prd", kwargs={
            "phase": "discuss",
            "prompt_file": "prompts/discuss.md",
        }),
        BuiltIn("discuss_prd", kwargs={
            "phase": "critique",
            "prompt_file": "prompts/critique.md",
        }),
        BuiltIn("commit_prd_changes", kwargs={
            "message": "chore(prd): {target_prd} discuss session refinements",
        }),
    ],
)
```

### prompts/discuss.md

Collaborator framing: help develop the PRD, ask clarifying questions, suggest missing sections, surface decisions. Includes `{PRD_CONTEXT}` placeholder at the top and explicit instruction to exit when finished.

### prompts/critique.md

Critic framing: find ambiguity, hidden assumptions, scope creep, missing acceptance criteria, risks. Includes `{PRD_CONTEXT}` placeholder and exit instruction.
