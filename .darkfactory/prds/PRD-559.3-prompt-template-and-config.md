# Create Stage 2 prompt template and analysis config keys

## Summary

Create the markdown prompt template used by the Stage 2 LLM narrative call
and add the analysis configuration keys to `.darkfactory/config.toml`. These
are independent of the detector implementation and can be done in parallel.

## Requirements

1. Create `analyze_transcript_prompt.md` adjacent to the builtin code with
   `{{FINDINGS}}` and `{{TRANSCRIPT_EXCERPT}}` placeholders.
2. The prompt must instruct the LLM to produce markdown with a fixed shape:
   Summary / Findings / Recommendations / Suggested PRD.
3. The prompt should specify that output is read-only analysis — no file
   edits.
4. Add three config keys under `[analysis]` in `.darkfactory/config.toml`:
   - `min_severity` — minimum finding severity to trigger Stage 2
     (default: `"warning"`).
   - `model_default` — model for warning-severity runs (default: `"haiku"`).
   - `model_severe` — model for error-severity runs (default: `"sonnet"`).

## Technical Approach

### File: `src/darkfactory/builtins/analyze_transcript_prompt.md`

Write a prompt that:

1. Opens with a system-level instruction: "You are analyzing a transcript
   from an automated agent run. Your job is to identify what went wrong
   and recommend fixes."
2. Includes `{{FINDINGS}}` placeholder — will be replaced with the
   formatted Stage 1 findings.
3. Includes `{{TRANSCRIPT_EXCERPT}}` placeholder — will be replaced with
   the relevant transcript lines (events tagged by findings ± context).
4. Specifies the output format:
   ```
   ## Summary
   <1-2 sentence overview>

   ## Findings
   <bullet list of each finding with context>

   ## Recommendations
   <actionable suggestions>

   ## Suggested PRD
   <optional: a draft PRD block if the fix is a process/tooling change>
   ```
5. Instructs the LLM to omit the "Suggested PRD" section if no tooling
   change is warranted.

### File: `.darkfactory/config.toml`

Add an `[analysis]` section with commented defaults:

```toml
[analysis]
# min_severity = "warning"
# model_default = "haiku"
# model_severe = "sonnet"
```

## Acceptance Criteria

- [ ] AC-1: `analyze_transcript_prompt.md` exists at the correct path with
      both placeholders present.
- [ ] AC-2: The prompt specifies the four-section output shape.
- [ ] AC-3: Config keys are present in `.darkfactory/config.toml` under
      `[analysis]` with documented defaults.
- [ ] AC-4: Existing config.toml content is preserved — only the new section
      is appended.
