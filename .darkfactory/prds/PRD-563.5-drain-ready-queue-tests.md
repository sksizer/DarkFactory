# Comprehensive tests for drain-ready-queue execution mode

## Summary

Write the test suite for the drain-ready-queue feature, covering all
scenarios listed in PRD-563 AC-11: empty queue, single ready PRD,
multiple independent PRDs ordered by priority, dependency chain
unlocked mid-run, filter combinations, exclusion, max-runs cutoff,
and failure isolation.

Tests should cover both the pure functions (discovery, filtering,
sorting) and the integrated execution loop (queue strategy through
`execute_graph`).

## Requirements

1. Test `discover_ready_queue` with:
   - Empty PRD set → empty result.
   - Single ready PRD with satisfied deps → returned.
   - Multiple independent PRDs → ordered by priority then number.
   - PRD with unsatisfied dep → excluded.
   - PRD with dep in `review` status → included.
2. Test `matches_filters` with:
   - `min_priority="high"` excludes medium/low PRDs.
   - `tags=["harness"]` includes PRDs with that tag, excludes others.
   - Multiple tags → OR semantics.
   - `exclude_ids=["PRD-5"]` excludes that PRD.
   - Empty filters → all pass.
3. Test `QueueStrategy` through `execute_graph`:
   - Single ready PRD runs successfully.
   - Multiple ready PRDs run in priority order.
   - A dependency chain: A depends on B, both ready. B runs first,
     then A is picked up after re-load.
   - Mid-run unlock: A depends on B (not ready initially). B completes
     and A transitions to ready — next iteration picks up A.
   - `--max-runs` cutoff stops execution.
   - Failure isolation: failed PRD is blocked, its dependents skipped,
     unrelated PRDs continue.
4. Test CLI integration:
   - `--all` and `prd_id` together → error.
   - Neither `--all` nor `prd_id` → error.
   - `--all --priority high --tag harness --exclude PRD-99` parses
     correctly.
