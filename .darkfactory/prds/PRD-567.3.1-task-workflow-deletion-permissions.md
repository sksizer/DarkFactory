# Add file deletion permissions to task workflow

## Summary

The task workflow includes `Bash(git rm:*)` but PRD-549.12 still failed to delete a file. The agent may have used direct `rm` (not `git rm`), which is not in the allowlist. Add `Bash(rm:*)` to the task workflow's tool list so both deletion methods work.

## Requirements

1. Add `"Bash(rm:*)"` to the `tools` list in `task/workflow.py`, after `"Bash(git rm:*)"`.
2. Verify the pattern matches common deletion commands: `rm file.py`, `rm -f file.py`, `rm -rf dir/`.
3. No prompt changes needed — the task workflow role prompt already permits destructive operations scoped to the PRD.

## Technical approach

```python
# In task/workflow.py, tools list (currently line 64):
"Bash(git rm:*)",
"Bash(rm:*)",      # ADD: direct file deletion
```

Single-line change.
