# Post-invocation containment verification in runner

## Summary

Even with `--disallowed-tools` rules in place, containment could be bypassed through `Bash` commands (e.g. `echo > /path/outside/worktree`). As a defense-in-depth measure, the runner should check after each agent invocation whether any files outside the worktree were modified.

## Requirements

1. After an `AgentTask` completes successfully, run `git status --porcelain` in the main repo (not the worktree) to detect unexpected modifications.
2. If modifications are found in the main repo that weren't there before the agent ran, log a warning with the affected file paths.
3. Optionally (configurable), treat out-of-worktree modifications as a task failure rather than a warning.
4. The check must compare main repo state before and after the agent runs, not just check for dirty state (since the main repo may already have untracked files).
5. The check must not add significant latency — `git status` in the main repo is fast.

## Technical approach

```python
# In runner.py, around the AgentTask dispatch:
def _check_containment(repo_root: Path, before_status: str, logger: logging.Logger) -> list[str]:
    """Return list of files modified in the main repo since before_status snapshot."""
    after = subprocess.run(
        ["git", "-C", str(repo_root), "status", "--porcelain"],
        capture_output=True, text=True,
    ).stdout
    before_files = {line.strip() for line in before_status.splitlines() if line.strip()}
    after_files = {line.strip() for line in after.splitlines() if line.strip()}
    new_modifications = after_files - before_files
    if new_modifications:
        escaped = [f.lstrip("MADRCU?! ") for f in new_modifications]
        logger.warning(
            "containment violation: agent modified files in main repo: %s",
            ", ".join(escaped),
        )
        return escaped
    return []

# Before agent invocation:
before_status = subprocess.run(
    ["git", "-C", str(ctx.repo_root), "status", "--porcelain"],
    capture_output=True, text=True,
).stdout

# After agent invocation:
escaped_files = _check_containment(ctx.repo_root, before_status, ctx.logger)
if escaped_files:
    # Log or fail depending on configuration
    ...
```

## Acceptance criteria

- [ ] AC-1: The runner captures main repo `git status` before each agent invocation.
- [ ] AC-2: After agent invocation, new modifications in the main repo are detected and logged as warnings.
- [ ] AC-3: The check does not flag pre-existing dirty state in the main repo.
- [ ] AC-4: The check adds less than 500ms of latency per agent invocation.
- [ ] AC-5: `just test`, `just lint`, `just typecheck` all pass.
