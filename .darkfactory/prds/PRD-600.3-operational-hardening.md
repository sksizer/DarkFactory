# Operational hardening and CLI quality improvements

## Summary

A collection of medium-priority improvements to CLI quality, scripting friendliness, and operational safety. Addresses: runner code deduplication, `--json` gaps in CI-critical commands, honest `--json` flag behavior, scriptable cleanup, help text examples, a missing inspection command, silent workflow failure prevention, and reconcile scalability.

## Motivation

These findings came from the CLI/UX reviewer, tooling reviewer, and adversarial reviewer. Each individually is small but collectively they represent the difference between a "works for the author" tool and a "works for anyone" tool. Priority items are `--json` for `validate` (CI needs structured errors) and `prd show` (basic inspection primitive every entity-management CLI needs).

All children are independent and can be parallelized.

## Acceptance criteria

- [ ] `_run_shell_once` exists in exactly one location
- [ ] `prd validate --json` emits structured error/warning objects
- [ ] `prd tree --json` emits structured hierarchy
- [ ] `prd --json <unsupported-cmd>` warns instead of silently ignoring
- [ ] `prd cleanup --all --yes` skips interactive confirmation
- [ ] `prd run --help` includes usage examples
- [ ] `prd show PRD-NNN` displays PRD metadata
- [ ] Broken project-level workflow raises error, not silent fallback
- [ ] Reconcile paginates or warns when at PR fetch limit
