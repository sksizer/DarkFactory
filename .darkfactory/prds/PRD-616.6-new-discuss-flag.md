# Add --discuss flag to prd new command

## Summary

Add a `--discuss` flag to `prd new` that chains into a discuss session against the just-created PRD after successful creation. Composes with `--open`: if both are set, the editor opens first, then the discuss session starts after the editor exits.

## Requirements

1. Add `--discuss` argument to the `new` subparser in `cli/new.py` or `cli/_parser.py`.
2. After successful PRD creation in `cmd_new`, if `--discuss` is set, call `launch_discuss_for_prd(new_id, args)` from `cli/discuss.py`.
3. Compatible with `--open`: if both flags are set, `--open` runs first (editor exits), then `--discuss` launches.
4. `prd new --help` documents the `--discuss` flag.
5. Integration test verifying the PRD is created AND the discuss chain is invoked against the new PRD's id.

## Technical Approach

### cli/new.py modification

After the existing `--open` block in `cmd_new`:

```python
if args.discuss:
    from darkfactory.cli.discuss import launch_discuss_for_prd
    launch_discuss_for_prd(new_id, args)
```

The lazy import avoids circular dependencies and keeps the discuss command's code out of the new command's import path.

### Test

Mock `launch_discuss_for_prd` and verify it is called with the correct new PRD id when `--discuss` is passed. Verify ordering with `--open`: editor subprocess exits before `launch_discuss_for_prd` is called.

## Acceptance Criteria

- [ ] AC-1: `prd new --discuss --title "Some title"` creates the PRD and launches discuss chain against the new id
- [ ] AC-2: Composes correctly with `--open` (editor first, then discuss)
- [ ] AC-3: `prd new --help` documents the `--discuss` flag
- [ ] AC-4: Integration test verifies the flag triggers the discuss chain with the correct PRD id
- [ ] AC-5: Passes mypy strict
