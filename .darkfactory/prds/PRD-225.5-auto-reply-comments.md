# Auto-post replies to addressed PR comments

## Summary

After the rework agent finishes, optionally post replies on each PR comment that was addressed. The agent's per-comment reply notes are extracted from its output and posted via `gh` with a `[harness]` prefix so reviewers can distinguish bot replies from human ones. This is opt-in via the `--reply-to-comments` flag.

## Requirements

1. Extract per-comment reply notes from the agent's output. The agent is instructed to produce lines like `Addressed: <description>` or `Disagree: <reason>` for each comment.
2. Parse the agent output to match reply notes to their corresponding review threads (by thread ID or positional ordering).
3. Post replies using `gh api` to comment on the specific review thread.
4. Prefix each reply with `[harness] addressed in {commit_sha}: ` so it's clearly identifiable as a bot reply.
5. Only execute when `--reply-to-comments` is passed — this is opt-in.
6. Handle failures gracefully: if posting a reply fails (rate limit, permission), log a warning but don't fail the rework run.

## Technical Approach

### `src/darkfactory/pr_comments.py` — reply posting

Add reply functions to the existing module:

```python
@dataclass
class CommentReply:
    thread_id: str
    body: str

def post_comment_replies(
    pr_number: int,
    replies: list[CommentReply],
    commit_sha: str,
) -> list[tuple[str, bool]]:
    """Post replies to PR comment threads. Returns (thread_id, success) pairs."""
    results = []
    for reply in replies:
        prefixed = f"[harness] addressed in {commit_sha[:8]}: {reply.body}"
        success = _post_reply(pr_number, reply.thread_id, prefixed)
        results.append((reply.thread_id, success))
    return results

def _post_reply(pr_number: int, thread_id: str, body: str) -> bool:
    """Post a single reply via gh api."""
    try:
        subprocess.run(
            ["gh", "api", f"repos/{{owner}}/{{repo}}/pulls/{pr_number}/comments/{thread_id}/replies",
             "-f", f"body={body}"],
            capture_output=True, text=True, check=True,
        )
        return True
    except subprocess.CalledProcessError:
        return False
```

### Integration in workflow

The reply-posting step runs after commit and push, only if `ctx.reply_to_comments` is True:

```python
if ctx.reply_to_comments:
    replies = parse_agent_reply_notes(ctx.agent_output, ctx.review_threads)
    results = post_comment_replies(ctx.pr_number, replies, ctx.commit_sha)
    for thread_id, success in results:
        if not success:
            console.print(f"[yellow]WARNING: Failed to post reply to thread {thread_id}[/yellow]")
```
