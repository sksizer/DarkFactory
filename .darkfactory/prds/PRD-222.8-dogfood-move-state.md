# Dogfood: move darkfactory's own state to .darkfactory/

## Summary

Apply the `.darkfactory/` convention to darkfactory's own repository. Run `prd init` against the repo root, move all existing PRDs from `prds/` to `.darkfactory/prds/`, remove the old top-level `prds/` directory, and verify that every harness flow works against darkfactory's own clone using the same code path as any other project. No dev-mode fallback, no special-case detection.

## Requirements

1. Run `prd init` in darkfactory's own repo to create the `.darkfactory/` structure.
2. Move all files from `prds/` to `.darkfactory/prds/` preserving filenames and content.
3. Delete the old top-level `prds/` directory after the move.
4. Update `.gitignore` to include `.darkfactory/worktrees/` and `.darkfactory/transcripts/` (should be handled by `prd init`).
5. Update any code, scripts, CI config, or documentation that references the old `prds/` path to use `.darkfactory/prds/`.
6. Verify all harness flows work: `prd status`, `prd validate`, `prd tree`, `prd list-workflows`, `prd run`, workflow execution.
7. Confirm there is no dev-mode fallback anywhere — grep the codebase for any `prds/` hardcoded paths or `pyproject.toml` sniffing and remove them.
8. The CLAUDE.md or any harness configuration referencing `prds/` must be updated.

## Technical Approach

### Step 1: Initialize

```bash
prd init  # creates .darkfactory/ at repo root
```

### Step 2: Move PRDs

```bash
mv prds/*.md .darkfactory/prds/
rmdir prds  # or rm -r if it has subdirectories
```

### Step 3: Update references

Search for hardcoded `prds/` references:
- `src/darkfactory/cli.py` — `_default_prd_dir` should already be gone (PRD-222.1), but verify.
- `CLAUDE.md` — update any instructions referencing `prds/`.
- `.github/workflows/` — update CI if it references `prds/`.
- `tests/` — update test fixtures that assume `prds/` at repo root.
- `README.md` — should already be updated by PRD-222.7.

### Step 4: Verify

Run the full suite:
```bash
prd validate
prd status
prd tree
prd list-workflows
```

All should work against `.darkfactory/prds/` without any special-case logic.

### Step 5: Grep for remnants

```bash
grep -r '"prds/"' src/ tests/
grep -r "'prds/'" src/ tests/
grep -r 'prds/' .github/ CLAUDE.md
```

Remove or update any remaining references.

## Acceptance Criteria

- [ ] AC-1: `.darkfactory/` directory exists at the repo root with the standard layout.
- [ ] AC-2: All existing PRD files are under `.darkfactory/prds/`.
- [ ] AC-3: The old top-level `prds/` directory no longer exists.
- [ ] AC-4: `prd validate` passes against `.darkfactory/prds/`.
- [ ] AC-5: `prd status`, `prd tree`, and `prd list-workflows` work correctly.
- [ ] AC-6: No hardcoded `prds/` paths remain in `src/` or `tests/`.
- [ ] AC-7: No dev-mode fallback or pyproject.toml sniffing anywhere in the codebase.
- [ ] AC-8: `.gitignore` excludes `.darkfactory/worktrees/` and `.darkfactory/transcripts/`.
