# CLI --all flag, filter flags, and queue-mode wiring

## Summary

Add the `--all`, `--priority`, `--tag`, and `--exclude` CLI flags to the `prd run` subcommand. Wire them into the queue-mode candidate strategy so `prd run --all` discovers and executes all ready PRDs. Ensure `--all` and a positional `PRD-ID` are mutually exclusive.

## Requirements

1. `prd run --all` enters queue mode (no positional PRD-ID required).
2. `prd run --all <PRD-ID>` is a hard error with a clear message.
3. `--priority P` accepts `high`, `medium`, or `low` and filters to PRDs at or above that priority.
4. `--tag T` is repeatable and filters to PRDs with any matching tag (OR semantics).
5. `--exclude PRD-ID` is repeatable and excludes specific PRDs.
6. `--priority`, `--tag`, and `--exclude` are only valid with `--all`; passing them without `--all` is an error.
7. Dry-run (default) prints the ordered queue with reasoning (deps satisfied, priority, filters).
8. `--execute` with `--all` runs the queue through the refactored `execute_graph` using the queue candidate strategy.
9. `--max-runs` works with `--all` just as it does with rooted mode.

## Technical Approach

### File: `src/darkfactory/cli/_parser.py`

Make the `prd_id` positional argument optional (with `nargs="?"` and `default=None`). Add:

```python
run_parser.add_argument("--all", action="store_true", help="Run all ready PRDs (queue mode)")
run_parser.add_argument("--priority", choices=["high", "medium", "low"], help="Filter to PRDs at or above this priority")
run_parser.add_argument("--tag", action="append", dest="tags", default=[], help="Filter to PRDs with this tag (repeatable, OR semantics)")
run_parser.add_argument("--exclude", action="append", dest="exclude_ids", default=[], help="Exclude specific PRD IDs (repeatable)")
```

### File: `src/darkfactory/cli/__init__.py`

In `cmd_run`:

1. Validate mutual exclusivity: if `args.all` and `args.prd_id` are both set, print error and return 1.
2. Validate filter flags: if any of `--priority`, `--tag`, `--exclude` are set without `--all`, print error and return 1.
3. If `args.all`:
   - Build `QueueFilters` from `args.priority`, `args.tags`, `args.exclude_ids`.
   - Build the queue candidate strategy via `queue_candidates(filters)`.
   - Pass it to `execute_graph(root_id=None, candidate_fn=..., ...)` or `plan_execution` for dry-run.
4. If `args.prd_id` (existing path): unchanged behavior.
5. If neither `--all` nor `prd_id`: print error and return 1.

### Dry-run output for queue mode

For the dry-run case, call `plan_execution` with the queue candidate strategy and print:

```
Queue mode: N ready PRDs found (filters: priority>=high, tags=[harness])

Execution order:
  1. PRD-101  (priority=high, deps satisfied)
  2. PRD-205  (priority=high, deps satisfied, depends on PRD-101)
  3. PRD-310  (priority=medium, deps satisfied)

Skipped:
  - PRD-999  (excluded by --exclude)
```

## Acceptance Criteria
