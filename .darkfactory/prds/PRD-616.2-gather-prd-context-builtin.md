# Implement gather_prd_context system builtin

## Summary

Create the `gather_prd_context` system builtin that reads the target PRD from disk, walks its parent and direct dependencies one level deep, and composes a markdown context block stored in `ctx._shared_state["prd_context"]`. This is a pure data-gathering step with no agent invocation or subprocess spawning.

## Requirements

1. New file `src/darkfactory/builtins/gather_prd_context.py` with the builtin function.
2. Register `gather_prd_context` in `SYSTEM_BUILTINS` via the `@_register` decorator in `system_builtins.py`.
3. Read `ctx.target_prd` to resolve the PRD file from the loaded PRDs in `ctx.prds`.
4. Walk parent (`prd.parent`) and direct dependencies (`prd.depends_on`) one level deep.
5. Compose markdown context block matching the format specified in the parent PRD.
6. Store the block at `ctx._shared_state["prd_context"]`.
7. Peer test file at `src/darkfactory/builtins/gather_prd_context_test.py`.

## Technical Approach

### gather_prd_context.py

```python
def gather_prd_context(ctx: SystemContext) -> None:
```

- Resolve target PRD from `ctx.prds` using `ctx.targets[0]` (the target PRD id).
- Read the PRD file content from disk (the file path is derivable from the PRD's id and the configured PRD directory).
- For parent and each dependency, look them up in `ctx.prds` and extract id, title, status, and first line of body as a summary.
- Compose the markdown block:
  ```
  ## Target PRD
  - id: PRD-616
  - title: ...
  - status: draft
  - kind: feature

  ### Body
  <full PRD body>

  ## Parent
  - PRD-XXX: ... (status: ...) — <one-line summary>

  ## Dependencies
  - PRD-YYY: ... (status: ...) — <one-line summary>
  ```
- Store in `ctx._shared_state["prd_context"]`.

### Peer test

Test against a temporary PRD directory with a target PRD, a parent PRD, and two dependency PRDs. Verify the context block contains all expected sections and data.
