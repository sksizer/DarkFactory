# Package metadata and installability

## Summary

Verify and complete the package metadata so that `uv tool install darkfactory` (or `pipx install`, `pip install`) produces working `prd` and `darkfactory` binaries on the user's PATH. Add `darkfactory` as a secondary entry-point alias. Update the README quickstart to reflect the new installation and project setup flow using `.darkfactory/`.

## Requirements

1. Verify the existing `[project.scripts]` entry `prd = "darkfactory.cli:main"` works via `uv tool install .` from a local checkout.
2. Add `darkfactory = "darkfactory.cli:main"` as a secondary entry point in `[project.scripts]`.
3. Ensure `src/darkfactory/workflows/` and its subdirectories are included in the built wheel (verify `[tool.setuptools.packages]` or equivalent includes them).
4. Update README.md quickstart section to show:
   - `uv tool install darkfactory` (or `pipx install darkfactory`)
   - `cd ~/my-project && prd init`
   - `prd status`
5. README must explicitly document that `.darkfactory/` is the target-project convention.
6. README must note that darkfactory's own repo uses the same `.darkfactory/` convention — no special cases.

## Technical Approach

### `pyproject.toml` changes

```toml
[project.scripts]
prd = "darkfactory.cli:main"
darkfactory = "darkfactory.cli:main"
```

Verify that `[tool.setuptools.package-data]` or `[tool.setuptools.packages.find]` includes `darkfactory.workflows` and its subpackages. Since workflows are Python modules (not data files), they should be discovered automatically by `find_packages`, but confirm.

### Verification steps

1. `uv tool install .` from repo root — both `prd` and `darkfactory` commands should be on PATH.
2. `prd --help` and `darkfactory --help` both work.
3. `prd list-workflows` shows built-in workflows (post-222.3).

### README.md update

Rewrite the quickstart section. Keep the rest of the README intact. The new quickstart:

```markdown
## Quickstart

### Install

```bash
uv tool install darkfactory
# or: pipx install darkfactory
```

### Set up a project

```bash
cd ~/my-project
prd init        # creates .darkfactory/ directory
prd status      # view PRD status
```

### Project layout

darkfactory stores all project state under `.darkfactory/` at your repo root:
- `.darkfactory/prds/` — PRD files (tracked in git)
- `.darkfactory/workflows/` — custom workflows (tracked; optional)
- `.darkfactory/config.toml` — project config (tracked; optional)
- `.darkfactory/worktrees/` — runtime state (git-ignored)
```

## Acceptance Criteria

- [ ] AC-1: `uv tool install .` produces a working `prd` binary on PATH.
- [ ] AC-2: `uv tool install .` also produces a `darkfactory` alias binary on PATH.
- [ ] AC-3: Built-in workflows are included in the installed package.
- [ ] AC-4: README quickstart shows the `prd init` flow with `.darkfactory/`.
- [ ] AC-5: README documents `.darkfactory/` as the universal project convention.
