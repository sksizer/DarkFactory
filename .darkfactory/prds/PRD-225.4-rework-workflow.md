# Rework workflow definition

## Summary

Create the `rework` workflow that the `prd rework --execute` command invokes. This workflow reuses the existing worktree (skipping creation), fetches PR comments, composes the rework prompt, invokes the agent, runs validation (test/format/lint/typecheck), commits with a rework-specific message, and pushes to the existing branch so the PR auto-updates. It does NOT create a new PR.

## Requirements

1. Define a `workflows/rework/workflow.yaml` workflow file following the existing workflow format.
2. The workflow must skip `ensure_worktree` creation — the worktree must already exist (validated by PRD-225.1).
3. The workflow must skip `set_status` — the PRD stays in `review`.
4. The workflow must skip `create_pr` — the PR already exists.
5. The workflow must include a step to fetch PR comments (using `pr_comments.fetch_pr_comments` from PRD-225.2).
6. The workflow must compose the rework prompt (using templates from PRD-225.3) and invoke the agent.
7. After the agent runs, execute the standard validation steps: format, lint, typecheck, test.
8. Commit with message `chore(prd): {prd_id} address review feedback`.
9. Push to the existing branch so the PR auto-updates.
10. Wire `cmd_rework --execute` to invoke this workflow via the runner.

## Technical Approach

### `workflows/rework/workflow.yaml`

```yaml
name: rework
description: Address PR review feedback for an existing PRD

tasks:
  - name: fetch_comments
    type: builtin
    builtin: fetch_pr_comments
    description: "Pull unresolved PR review comments"

  - name: agent
    type: agent
    prompt_dir: prompts
    description: "Address review feedback"

  - name: format
    type: shell
    command: "uv run ruff format ."
    description: "Format code"

  - name: lint
    type: shell
    command: "uv run ruff check --fix ."
    description: "Lint code"

  - name: typecheck
    type: shell
    command: "uv run pyright"
    description: "Type check"
    optional: true

  - name: test
    type: shell
    command: "uv run pytest"
    description: "Run tests"

  - name: commit
    type: builtin
    builtin: commit
    message: "chore(prd): {prd_id} address review feedback"
    description: "Commit rework changes"

  - name: push
    type: builtin
    builtin: push_branch
    description: "Push to existing branch (PR auto-updates)"
```

### `src/darkfactory/cli.py` — `cmd_rework` completion

Complete the `--execute` path in `cmd_rework`:

```python
if args.execute:
    # Build comment filters from CLI args
    filters = CommentFilters(
        include_resolved=args.all,
        since_commit=args.since,
        reviewer=args.reviewer,
        single_comment_id=args.from_pr_comment,
    )

    # Fetch comments and store on context
    threads = fetch_pr_comments(pr_number, filters=filters)
    if not threads:
        console.print(f"[yellow]No unaddressed comments found for {prd_id}[/yellow]")
        return

    # Load and run the rework workflow
    workflow = load_workflow("rework")
    ctx = ExecutionContext(
        prd=meta,
        worktree_path=worktree_path,
        pr_number=pr_number,
        review_threads=threads,
        reply_to_comments=args.reply_to_comments,
    )
    runner.execute(workflow, ctx)
```

### `src/darkfactory/runner.py`

The runner may need a small extension to pass `review_threads` to the prompt composition step. The `ExecutionContext` dataclass gets a new optional field `review_threads: list[ReviewThread] | None = None`.
