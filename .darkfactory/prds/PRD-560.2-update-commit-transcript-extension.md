# Update commit_transcript.py for .jsonl extension

## Summary

Update `commit_transcript.py` to look for `.jsonl` transcript files instead of `.log` files when copying from the transient location to the committed location. Change both the source glob/path and the destination filename to use the `.jsonl` extension.

## Requirements

1. Change the transient transcript source path from `.harness-transcripts/{prd_id}.log` to `.harness-transcripts/{prd_id}.jsonl`.
2. Change the committed transcript destination path from `.darkfactory/transcripts/{prd_id}-{timestamp}.log` to `.darkfactory/transcripts/{prd_id}-{timestamp}.jsonl`.
3. Ensure any glob patterns or file-existence checks in the commit logic reference `.jsonl`.

## Technical Approach

### File: `src/darkfactory/builtins/commit_transcript.py` (~lines 36-58)

Find all references to `.log` in this file and replace with `.jsonl`. This includes:

- The source path construction for finding the transient transcript.
- The destination path construction for the committed transcript.
- Any string literals or glob patterns that reference the `.log` extension.

This is a straightforward find-and-replace scoped to this single file.

## Acceptance Criteria

- [ ] AC-1: `commit_transcript` looks for `.jsonl` files in `.harness-transcripts/`.
- [ ] AC-2: Committed transcripts are written to `.darkfactory/transcripts/` with `.jsonl` extension.
- [ ] AC-3: No references to `.log` transcripts remain in `commit_transcript.py`.
