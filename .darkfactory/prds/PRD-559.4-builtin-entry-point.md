# Implement analyze_transcript builtin entry point

## Summary

Implement the main `analyze_transcript` builtin function that orchestrates
Stage 1 (deterministic scan) and Stage 2 (LLM narrative), writes the analysis
file, augments `ctx.run_summary`, and handles all failure modes gracefully.
Register the builtin in `__init__.py`.

## Requirements

1. Register `analyze_transcript` as a builtin via `@builtin("analyze_transcript")`.
2. Stage 1: load the JSONL transcript, parse events, run all registered
   detectors, collect findings.
3. Stage 2: if any finding meets the severity threshold (from config), invoke
   `claude --print` with the prompt template, selected model (severity-tiered),
   and filtered transcript excerpt.
4. Write output to `.darkfactory/transcripts/{prd_id}-{ts}.analysis.md` and
   stage it with `git add`.
5. Append a short analysis section to `ctx.run_summary` (title + finding
   category bullets + pointer to analysis file).
6. Respect dry-run mode: log actions without writing files or invoking LLM.
7. Handle all failure modes (missing transcript, scanner crash, LLM failure)
   by logging a warning and returning cleanly.
8. Add the import to `src/darkfactory/builtins/__init__.py` so the builtin
   is registered at import time.

## Technical Approach

### File: `src/darkfactory/builtins/analyze_transcript.py`

```python
@builtin("analyze_transcript")
def analyze_transcript(ctx: ExecutionContext, **kwargs) -> None:
    # 1. Locate transcript file
    transcript_path = find_transcript(ctx)
    if not transcript_path:
        logger.info("No transcript found, skipping analysis")
        return

    # 2. Parse JSONL events
    events = parse_transcript(transcript_path)

    # 3. Stage 1 — run detectors
    findings = run_detectors(events)

    # 4. Check severity threshold
    if not exceeds_threshold(findings, ctx.config):
        logger.info("No findings above threshold, skipping Stage 2")
        append_summary(ctx, findings, analysis_path=None)
        return

    # 5. Dry-run check
    if ctx.dry_run:
        logger.info("Dry run: would invoke LLM and write analysis")
        return

    # 6. Stage 2 — LLM narrative
    try:
        narrative = invoke_llm(findings, events, ctx.config)
    except Exception:
        logger.warning("LLM call failed, skipping narrative", exc_info=True)
        narrative = None

    # 7. Write analysis file
    analysis_path = write_analysis(ctx, findings, narrative)

    # 8. Stage with git add
    subprocess.run(["git", "add", str(analysis_path)], check=True)

    # 9. Augment run summary
    append_summary(ctx, findings, analysis_path)
```

Helper functions:

- `find_transcript(ctx)` — look for `.harness-transcripts/{prd_id}.log`,
  return Path or None. Mirror `commit_transcript` pattern.
- `parse_transcript(path)` — read JSONL, return `list[dict]`.
- `run_detectors(events)` — iterate `DETECTORS.values()`, collect findings,
  wrap each call in try/except to isolate detector failures.
- `exceeds_threshold(findings, config)` — check if any finding severity
  meets `analysis.min_severity` from config.
- `invoke_llm(findings, events, config)` — load prompt template, substitute
  placeholders, select model based on max severity, call `claude --print`.
  Excerpt: events tagged by findings ± 3 lines, capped at ~4k tokens.
- `write_analysis(ctx, findings, narrative)` — format and write the
  `.analysis.md` file.
- `append_summary(ctx, findings, analysis_path)` — append short section to
  `ctx.run_summary`.

### File: `src/darkfactory/builtins/analyze_transcript_test.py`

Test cases:

1. Missing transcript — returns cleanly, no error.
2. Scanner returns only info findings — Stage 2 is skipped.
3. Scanner returns warning findings — Stage 2 fires (mock subprocess).
4. Model selection: warning-only uses `model_default`, error-severity uses
   `model_severe`.
5. Dry-run mode — no files written, no subprocess calls.
6. LLM failure — logs warning, returns cleanly.
7. `ctx.run_summary` is augmented with analysis section.
