# Integrate effort-based timeout into harness runner

## Summary

Wire the `resolve_timeout()` function from `darkfactory/timeouts.py` into the harness runner so that agent task invocations use effort-based timeouts instead of the hardcoded 600s default. Add a `--timeout` CLI flag to `prd run` that overrides all other timeout sources. Log the resolved timeout and its source when each task starts.

## Requirements

1. In `runner.py`'s `_run_agent()` function, call `resolve_timeout()` with the PRD's effort, capability, `timeout_minutes` frontmatter field, config timeouts, and any CLI override.
2. Pass the resolved timeout (in seconds) to `invoke_claude()` as `timeout_seconds`.
3. Add `--timeout <minutes>` flag to the `prd run` CLI command. This value is threaded through to `resolve_timeout()` as the CLI override.
4. When a task starts, log a message like: `Timeout: 1200s (source: default, effort=m, capability×1.0)` so failures are diagnosable.
5. Read timeout config from `.darkfactory/config.toml` `[timeouts]` section if present.

## Technical Approach

### `src/darkfactory/runner.py`

In the `_run_agent()` method (around line 293-302), before calling `invoke_claude()`:

```python
from darkfactory.timeouts import resolve_timeout

timeout_seconds, timeout_source = resolve_timeout(
    effort=prd.frontmatter.get("effort"),
    capability=prd.frontmatter.get("capability"),
    timeout_minutes_frontmatter=prd.frontmatter.get("timeout_minutes"),
    config_timeouts=self.config.get("timeouts"),
    cli_override=self.cli_timeout_minutes,
)
logger.info("Timeout: %ds (source: %s)", timeout_seconds, timeout_source)
```

Then pass `timeout_seconds=timeout_seconds` to `invoke_claude()`.

### `src/darkfactory/cli.py`

Add `--timeout` option to the `prd run` command:

```python
@click.option("--timeout", type=int, default=None, help="Override timeout in minutes")
```

Thread the value through to the runner.

### Config reading

If `.darkfactory/config.toml` has a `[timeouts]` section, parse it and pass to `resolve_timeout()`. If the section doesn't exist, pass `None` (built-in defaults apply).

## Acceptance Criteria

- [ ] AC-1: Agent tasks use effort-based timeouts from `resolve_timeout()` instead of hardcoded 600s.
- [ ] AC-2: `prd run --timeout 30` sets a 30-minute timeout that overrides all other sources.
- [ ] AC-3: When a task starts, the log shows the resolved timeout value and source.
- [ ] AC-4: Config timeouts from `.darkfactory/config.toml` are read and passed to `resolve_timeout()`.
- [ ] AC-5: When no config or CLI override is present, the default effort-based mapping is used.
