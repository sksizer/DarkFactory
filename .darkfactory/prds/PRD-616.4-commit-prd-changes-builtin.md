# Implement commit_prd_changes system builtin

## Summary

Create the `commit_prd_changes` system builtin that checks for working-tree changes to the target PRD file, shows a diff, prompts the user for confirmation, and optionally commits. This is the end-of-chain step that makes PRD edits from a discussion durable in git history.

## Requirements

1. New file `src/darkfactory/builtins/commit_prd_changes.py` with the builtin function.
2. Register `commit_prd_changes` in `SYSTEM_BUILTINS` via `@_register`.
3. Signature: `commit_prd_changes(ctx: SystemContext, *, message: str | None = None, paths: list[str] | None = None) -> None`.
4. Resolve `paths` default to the target PRD file path.
5. Resolve `message` default; substitute `{target_prd}` via `ctx.format_string` if present in template.
6. Run `git diff --quiet -- <paths>` to detect changes. If clean, print `No PRD changes to commit.` and return.
7. If dirty:
   - Print commit phase banner.
   - Show colored `git diff -- <paths>` output.
   - If other files are dirty, print a note listing them (up to 5).
   - Print the suggested commit message.
   - Prompt `Commit these changes? [y/N/e(dit message)]` with default N.
   - On `y`: `git add -- <paths>` then `git commit -m <message>`.
   - On `e`: prompt for new message, then commit with that message.
   - On `n` or empty: log `Skipped commit. Changes left in working tree.` and return.
8. Never push, create a PR, mutate frontmatter, or modify files outside the git commands.
9. Git subprocess calls via a `_run_git_commit` wrapper for test isolation.
10. Peer test file at `src/darkfactory/builtins/commit_prd_changes_test.py`.

```python
def _run_git_commit(args: list[str], cwd: Path) -> subprocess.CompletedProcess[str]:
    """Monkeypatchable wrapper for git subprocess calls."""
    return subprocess.run(args, cwd=cwd, check=False, capture_output=True, text=True)

```

- Resolve target PRD file path from `ctx.targets[0]` and the PRD directory.
- Check `git diff --quiet` for the target paths.
- Check `git status --porcelain` for other dirty files to surface as a note.
- Use `input()` for the interactive prompt (monkeypatchable in tests).

### Peer test

Test against a real-but-temporary git repo:
- Init a tmp repo, write a PRD file, commit it, modify the PRD.
- Test `y` path: verify `git add` and `git commit` are called with correct args.
- Test `n` path: verify no commit happens.
- Test `e` path: verify new message is used.
- Test no-changes path: verify early return with message, no prompt.
- Test unrelated dirty files: verify the note is printed and they are NOT included in the commit.
