# Provide PRD_IMPLEMENTATION_TEMPLATE

## Summary

Bundle a built-in `PRD_IMPLEMENTATION_TEMPLATE` that captures the standard PRD implementation lifecycle with PRD-224 invariants enforced. This is the template that most workflows will compose from. It is a declarative definition — the BuiltIns it references either already exist or are introduced by PRD-224.

## Requirements

1. Create `src/darkfactory/templates_builtin.py` containing `PRD_IMPLEMENTATION_TEMPLATE`.
2. The template's opening tasks enforce:
   - `ensure_worktree` (PRD-217 lock + PRD-224.2 stale check)
   - `set_status(to="in-progress")`
   - `commit(message="chore(prd): {prd_id} start work")`
3. The template's closing tasks enforce:
   - `summarize_agent_run` (PRD-224.3)
   - `commit_transcript` (PRD-224.6)
   - `set_status(to="review")`
   - `commit(message="chore(prd): {prd_id} ready for review")`
   - `push_branch`
   - `create_pr`
4. The middle requires at least 1 `AgentTask` and at least 1 `ShellTask`.
5. The template is importable: `from darkfactory.templates_builtin import PRD_IMPLEMENTATION_TEMPLATE`.

## Technical Approach

### `src/darkfactory/templates_builtin.py`

```python
from .templates import WorkflowTemplate
from .workflow import BuiltIn, AgentTask, ShellTask

PRD_IMPLEMENTATION_TEMPLATE = WorkflowTemplate(
    name="prd-implementation",
    description="Standard PRD implementation lifecycle with enforced invariants.",
    open=[
        BuiltIn("ensure_worktree"),
        BuiltIn("set_status", kwargs={"to": "in-progress"}),
        BuiltIn("commit", kwargs={"message": "chore(prd): {prd_id} start work"}),
    ],
    middle_kinds=[AgentTask, ShellTask],
    middle_required={
        AgentTask: (1, None),
        ShellTask: (1, None),
    },
    close=[
        BuiltIn("summarize_agent_run"),
        BuiltIn("commit_transcript"),
        BuiltIn("set_status", kwargs={"to": "review"}),
        BuiltIn("commit", kwargs={"message": "chore(prd): {prd_id} ready for review"}),
        BuiltIn("push_branch"),
        BuiltIn("create_pr"),
    ],
)
```

## Acceptance Criteria

- [ ] AC-1: `PRD_IMPLEMENTATION_TEMPLATE` is importable from `darkfactory.templates_builtin`.
- [ ] AC-2: The template's `open` list contains `ensure_worktree`, `set_status(in-progress)`, and `commit(start work)` in that order.
- [ ] AC-3: The template's `close` list contains all PRD-224 invariant tasks (`summarize_agent_run`, `commit_transcript`, `set_status(review)`, `commit(ready for review)`, `push_branch`, `create_pr`) in that order.
- [ ] AC-4: Composing with no `AgentTask` in the middle raises `TemplateViolation`.
- [ ] AC-5: Composing with no `ShellTask` in the middle raises `TemplateViolation`.
