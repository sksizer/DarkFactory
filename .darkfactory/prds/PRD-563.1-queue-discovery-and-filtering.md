# Queue discovery and filtering pure functions

## Summary

Implement the pure functions that discover all `ready` PRDs with satisfied dependencies, apply user-specified filters (priority, tag, exclude), and return them in topologically-sorted order with a priority-then-number tiebreak. These functions are the foundation of the queue execution mode and have no side effects.

## Requirements

1. Define a `QueueFilters` dataclass with fields: `min_priority` (optional priority threshold), `tags` (list of tags for OR-match filtering), and `exclude_ids` (set of PRD IDs to exclude).
2. Implement `deps_satisfied(prd, prds)` — returns `True` when every entry in `prd.depends_on` resolves to a PRD with status `done` or `review`.
3. Implement `matches_filters(prd, filters)` — returns `True` when the PRD passes priority, tag, and exclusion filters.
4. Implement `discover_ready_queue(prds, filters)` — collects all PRDs with `status == "ready"` whose deps are satisfied and that match filters, then topologically sorts with priority-rank then PRD-number tiebreak.
5. Exclusion via `--exclude` must also exclude transitive dependents whose deps become unsatisfied by the exclusion.

## Technical Approach

### File: `src/darkfactory/graph_execution.py`

Add the following near the top of the file, after the existing imports and before the executor section:

```python
from dataclasses import dataclass, field

@dataclass
class QueueFilters:
    min_priority: str | None = None   # "high", "medium", "low" or None
    tags: list[str] = field(default_factory=list)
    exclude_ids: set[str] = field(default_factory=set)

PRIORITY_RANK = {"critical": 0, "high": 1, "medium": 2, "low": 3}

def deps_satisfied(prd: PRD, prds: dict[str, PRD]) -> bool:
    for dep_id in prd.depends_on:
        dep = prds.get(dep_id)
        if dep is None or dep.status not in ("done", "review"):
            return False
    return True

def matches_filters(prd: PRD, filters: QueueFilters) -> bool:
    if filters.min_priority:
        threshold = PRIORITY_RANK.get(filters.min_priority, 2)
        if PRIORITY_RANK.get(prd.priority, 2) > threshold:
            return False
    if filters.tags:
        if not any(t in prd.tags for t in filters.tags):
            return False
    if prd.id in filters.exclude_ids:
        return False
    return True

def discover_ready_queue(
    prds: dict[str, PRD], filters: QueueFilters
) -> list[PRD]:
    ready = [
        p for p in prds.values()
        if p.status == "ready"
        and deps_satisfied(p, prds)
        and matches_filters(p, filters)
        and containment.is_runnable(p, prds)
    ]
    return topo_sort_with_tiebreak(ready, prds, key=lambda p: (PRIORITY_RANK.get(p.priority, 2), p.number))
```

The `topo_sort_with_tiebreak` function builds a restricted `depends_on` subgraph over the ready set, then performs a topological sort using the provided key as tiebreaker when multiple nodes have zero in-degree simultaneously.
