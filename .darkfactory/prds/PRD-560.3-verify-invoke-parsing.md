# Verify and update invoke.py transcript parsing

## Summary

Verify that `_find_terminal_result` and `_parse_sentinels` in `invoke.py` continue to work correctly with the new JSONL transcript format. The new `darkfactory_metadata`, `darkfactory_section`, and `darkfactory_stderr` envelope lines are valid JSON, so these functions should parse them without error — but they must not be confused for actual sentinel or result data. Update if necessary.

## Requirements

1. Verify that `_find_terminal_result` (~lines 104-343) correctly skips or ignores the new `darkfactory_metadata`, `darkfactory_section`, and `darkfactory_stderr` JSONL lines when searching for the terminal result.
2. Verify that `_parse_sentinels` correctly skips or ignores the new envelope lines when scanning for sentinel patterns.
3. If either function would incorrectly match on the new envelope lines, add a filter to exclude lines with `"type"` starting with `"darkfactory_"`.
4. Update any `.log` file path references in `invoke.py` to `.jsonl` if present.

## Technical Approach

### File: `src/darkfactory/invoke.py` (~lines 104-343)

1. Read `_find_terminal_result` and trace its logic for each new envelope line type:
   - `{"type": "darkfactory_metadata", ...}` — does the function's JSON parsing pick this up? Does it match any result-detection heuristic?
   - `{"type": "darkfactory_section", ...}` — same analysis.
   - `{"type": "darkfactory_stderr", ...}` — same analysis.

2. Read `_parse_sentinels` and do the same analysis.

3. If either function could misinterpret an envelope line as a meaningful result or sentinel, add an early-continue check:
   ```python
   parsed = json.loads(line)
   if parsed.get("type", "").startswith("darkfactory_"):
       continue
   ```

4. Search for any `.log` path references and update to `.jsonl`.

## Acceptance Criteria

- [ ] AC-1: `_find_terminal_result` does not misinterpret `darkfactory_*` envelope lines as terminal results.
- [ ] AC-2: `_parse_sentinels` does not misinterpret `darkfactory_*` envelope lines as sentinels.
- [ ] AC-3: Existing transcript parsing continues to function correctly with the new JSONL format.
- [ ] AC-4: Any `.log` path references in `invoke.py` are updated to `.jsonl`.
