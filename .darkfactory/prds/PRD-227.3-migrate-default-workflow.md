# Migrate the default workflow to use PRD_IMPLEMENTATION_TEMPLATE

## Summary

Rewrite `workflows/default/workflow.py` to call `PRD_IMPLEMENTATION_TEMPLATE.compose(...)` instead of declaring a flat `Workflow(tasks=[...])`. The end behavior is identical — the task list is the same — but the workflow is now provably template-conformant. The runner can check `workflow.template_name == "prd-implementation"` and surface it in `prd plan` output.

## Requirements

1. Replace the flat `Workflow(tasks=[...])` construction in `workflows/default/workflow.py` with a call to `PRD_IMPLEMENTATION_TEMPLATE.compose(...)`.
2. The middle should contain exactly the tasks that currently sit between the opening and closing builtins.
3. The resulting task list must be identical to the current one — this is a pure refactor with no behavioral change.
4. Update `prd plan` output to display `workflow: default (template: prd-implementation)` when the workflow has a `template_name`.
5. All existing workflow tests must continue to pass.

## Technical Approach

### `workflows/default/workflow.py`

Before:
```python
workflow = Workflow(
    name="default",
    tasks=[
        BuiltIn("ensure_worktree"),
        BuiltIn("set_status", kwargs={"to": "in-progress"}),
        BuiltIn("commit", kwargs={"message": "chore(prd): {prd_id} start work"}),
        AgentTask(...),
        ShellTask("test", ...),
        ShellTask("format", ...),
        ShellTask("lint", ...),
        ShellTask("typecheck", ...),
        BuiltIn("summarize_agent_run"),
        BuiltIn("commit_transcript"),
        BuiltIn("set_status", kwargs={"to": "review"}),
        BuiltIn("commit", kwargs={"message": "chore(prd): {prd_id} ready for review"}),
        BuiltIn("push_branch"),
        BuiltIn("create_pr"),
    ],
)
```

After:
```python
from darkfactory.templates_builtin import PRD_IMPLEMENTATION_TEMPLATE

workflow = PRD_IMPLEMENTATION_TEMPLATE.compose(
    name="default",
    description="General-purpose PRD implementation workflow.",
    applies_to=lambda prd, prds: True,
    priority=0,
    middle=[
        AgentTask(...),
        ShellTask("test", ...),
        ShellTask("format", ...),
        ShellTask("lint", ...),
        ShellTask("typecheck", ...),
    ],
)
```

### `prd plan` output update

In the CLI code that displays workflow assignment, check for `workflow.template_name` and append it:
```
workflow: default (template: prd-implementation)
```
If `template_name` is `None`, show `(no template)`.
