# Migrate the extraction workflow to use a template

## Summary

Migrate `workflows/extraction/workflow.py` to use `PRD_IMPLEMENTATION_TEMPLATE.compose(...)`, verifying that the template works for non-default workflows. If the extraction workflow's open/close tasks differ enough from the standard template, define a variant template and document why. This task surfaces template flexibility questions early.

## Requirements

1. Analyze the extraction workflow's current task list to determine if it matches the `PRD_IMPLEMENTATION_TEMPLATE`'s open/close sequence.
2. If it matches: refactor to use `PRD_IMPLEMENTATION_TEMPLATE.compose(...)` with the extraction-specific middle.
3. If it does not match: either define a new template (e.g. `EXTRACTION_TEMPLATE`) or document why the extraction workflow cannot use the standard template and what invariants it needs instead.
4. Document the decision in a code comment or the workflow file itself.

## Technical Approach

### Step 1: Compare task lists

Read the current `workflows/extraction/workflow.py` and compare its opening and closing builtins against `PRD_IMPLEMENTATION_TEMPLATE.open` and `.close`.

### Step 2: Migrate or document

If the open/close match:
```python
from darkfactory.templates_builtin import PRD_IMPLEMENTATION_TEMPLATE

workflow = PRD_IMPLEMENTATION_TEMPLATE.compose(
    name="extraction",
    description="Workflow for extraction-type PRD tasks.",
    applies_to=lambda prd, prds: ...,
    priority=...,
    middle=[...extraction-specific tasks...],
)
```

If they don't match, create a new template in `templates_builtin.py` or a local template, and document the differences.

## Acceptance Criteria

- [ ] AC-1: The extraction workflow either uses `PRD_IMPLEMENTATION_TEMPLATE` or its own template — it is not a flat `Workflow(tasks=[...])` anymore.
- [ ] AC-2: If a new template was needed, it is defined and documented with rationale.
- [ ] AC-3: All existing tests for the extraction workflow pass.
- [ ] AC-4: The decision (same template vs different template) is documented in code or comments.
