# Cascade resolver for workflows

## Summary

Build a unified workflow registry that discovers workflows from all three layers (built-in, user, project), validates each against the workflow API contract, and enforces strict name uniqueness across layers. Any name collision is a hard error at startup. Any workflow failing API validation is a hard error with a clear diagnostic message naming the layer, file path, and specific violation.

## Requirements

1. Create `src/darkfactory/registry.py` with a `build_workflow_registry(project_dir: Path | None) -> dict[str, Workflow]` function.
2. The registry collects workflows from three layers in order:
   - **Built-in**: from `darkfactory.workflows.get_builtin_workflows()` (PRD-222.3).
   - **User**: from `~/.config/darkfactory/workflows/` via `load_workflows()` (PRD-222.4 paths).
   - **Project**: from `<project>/.darkfactory/workflows/` via `load_workflows()` (PRD-222.1 discovery).
3. **Collision detection**: if any workflow name appears in more than one layer, raise `WorkflowNameCollision` with a message naming all conflicting paths. This is fatal — no subcommand runs.
4. **API validation**: every discovered workflow is validated at load time against the expected contract (must be a `Workflow` instance with required attributes: `name`, `steps`, `applies_to`). Failures raise `InvalidWorkflow(layer, path, reason)`. Fatal.
5. Both error types print actionable remediation hints:
   - Collision: "Workflow 'X' found in multiple layers: [paths]. Rename or delete one to resolve."
   - Invalid: "Workflow at [path] (layer: [layer]) failed validation: [reason]."
6. Replace the current `_load_workflows_or_fail` in `cli.py` with a call to `build_workflow_registry()`.

## Technical Approach

### `src/darkfactory/registry.py`

```python
from pathlib import Path
from darkfactory.models import Workflow
from darkfactory.loader import load_workflows
from darkfactory.workflows import get_builtin_workflows
from darkfactory.paths import user_workflows_dir

class WorkflowNameCollision(Exception):
    """Raised when the same workflow name appears in multiple layers."""
    ...

class InvalidWorkflow(Exception):
    """Raised when a workflow fails API contract validation."""
    ...

def _validate_workflow(wf: Workflow, layer: str, path: Path) -> None:
    """Validate a workflow against the expected API contract."""
    if not hasattr(wf, 'name') or not wf.name:
        raise InvalidWorkflow(layer, path, "'name' attribute is missing or empty")
    if not hasattr(wf, 'steps') or not isinstance(wf.steps, list):
        raise InvalidWorkflow(layer, path, f"'steps' must be a list; got {type(wf.steps).__name__}")
    if not hasattr(wf, 'applies_to') or not callable(wf.applies_to):
        raise InvalidWorkflow(layer, path, "'applies_to' must be callable")

def build_workflow_registry(project_dir: Path | None = None) -> dict[str, Workflow]:
    """Discover, validate, and return all workflows from all layers."""
    registry: dict[str, tuple[str, Path, Workflow]] = {}

    # Layer 1: built-in
    for name, wf in get_builtin_workflows().items():
        _validate_workflow(wf, "built-in", Path("<package>") / name)
        registry[name] = ("built-in", Path("<package>") / name, wf)

    # Layer 2: user
    user_dir = user_workflows_dir()
    if user_dir.is_dir():
        for name, wf in load_workflows(user_dir).items():
            _validate_workflow(wf, "user", user_dir / name)
            if name in registry:
                existing_layer, existing_path, _ = registry[name]
                raise WorkflowNameCollision(name, [
                    (existing_layer, existing_path),
                    ("user", user_dir / name),
                ])
            registry[name] = ("user", user_dir / name, wf)

    # Layer 3: project
    if project_dir:
        proj_wf_dir = project_dir / "workflows"
        if proj_wf_dir.is_dir():
            for name, wf in load_workflows(proj_wf_dir).items():
                _validate_workflow(wf, "project", proj_wf_dir / name)
                if name in registry:
                    existing_layer, existing_path, _ = registry[name]
                    raise WorkflowNameCollision(name, [
                        (existing_layer, existing_path),
                        ("project", proj_wf_dir / name),
                    ])
                registry[name] = ("project", proj_wf_dir / name, wf)

    return {name: wf for name, (_, _, wf) in registry.items()}
```

### Changes to `src/darkfactory/cli.py`

- Replace `_load_workflows_or_fail()` (line 90) with a call to `build_workflow_registry(darkfactory_dir)`.
- Error handling: catch `WorkflowNameCollision` and `InvalidWorkflow` in `main()`, print the message, and exit 1.

### `tests/test_registry.py`

Table-driven tests covering:
- Built-in only (no user/project dirs).
- User-only workflows discovered correctly.
- Project-only workflows discovered correctly.
- Built-in + project with no collision — all merged.
- Collision between built-in and user.
- Collision between built-in and project.
- Collision between user and project.
- Invalid workflow in each layer (missing name, bad steps type, non-callable applies_to).
- Each error message includes layer, path, and specific violation.

## Acceptance Criteria

- [ ] AC-1: Workflows from all three layers are discovered and merged into a single registry.
- [ ] AC-2: A workflow name appearing in two or more layers raises `WorkflowNameCollision` with all conflicting paths.
- [ ] AC-3: A workflow failing API validation raises `InvalidWorkflow` naming layer, path, and violation.
- [ ] AC-4: Both errors are fatal — no subcommand runs until resolved.
- [ ] AC-5: Error messages include actionable remediation hints.
- [ ] AC-6: Unit tests cover all layer combinations, collision pairs, and validation failure modes.
