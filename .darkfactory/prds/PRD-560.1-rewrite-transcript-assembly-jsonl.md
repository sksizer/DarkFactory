# Rewrite runner.py transcript assembly to emit JSONL

## Summary

Replace the `#`-prefixed metadata and separator lines in `runner.py:_run_agent` (~lines 356-377) with proper JSONL lines, and change the transcript file extension from `.log` to `.jsonl`. This is the core change that makes transcripts valid JSONL from top to bottom.

## Requirements

1. Emit a `{"type": "darkfactory_metadata", "task": "...", "model": "...", "success": ..., "exit_code": ..., "failure_reason": ...}` JSONL line at the top of the transcript instead of the `#`-prefixed metadata comment lines.
2. Replace the `# ---- stdout ----` separator with a `{"type": "darkfactory_section", "section": "stdout"}` JSONL line.
3. Replace the `# ---- stderr ----` separator with a `{"type": "darkfactory_section", "section": "stderr"}` JSONL line.
4. Wrap stderr content (plain text) in a single `{"type": "darkfactory_stderr", "text": "..."}` JSONL line containing the full stderr block.
5. Write the transcript to `.jsonl` extension instead of `.log` — both the transient path (`.harness-transcripts/{prd_id}.jsonl`) and any path variable used for the committed location.

## Technical Approach

### File: `src/darkfactory/runner.py` (~lines 356-377)

Locate the transcript assembly block in `_run_agent`. Currently it writes lines like:

```python
f"# task: {task_name}\n"
f"# model: {model}\n"
f"# success: {success}\n"
f"# exit_code: {exit_code}\n"
f"# failure_reason: {failure_reason}\n"
"# ---- stdout ----\n"
result.stdout
"# ---- stderr ----\n"
result.stderr
```

Replace with:

```python
import json

metadata_line = json.dumps({
    "type": "darkfactory_metadata",
    "task": task_name,
    "model": model,
    "success": success,
    "exit_code": exit_code,
    "failure_reason": failure_reason,
}) + "\n"

stdout_section = json.dumps({"type": "darkfactory_section", "section": "stdout"}) + "\n"
stderr_section = json.dumps({"type": "darkfactory_section", "section": "stderr"}) + "\n"
stderr_line = json.dumps({"type": "darkfactory_stderr", "text": result.stderr}) + "\n"
```

Then change the file extension from `.log` to `.jsonl` wherever the transient transcript path is constructed.

## Acceptance Criteria

- [ ] AC-1: Transcripts written by `runner.py` are valid JSONL (every line parses as JSON).
- [ ] AC-2: Transcript files use `.jsonl` extension in the transient location (`.harness-transcripts/{prd_id}.jsonl`).
- [ ] AC-3: Metadata (task, model, success, exit_code, failure_reason) is present as a `darkfactory_metadata` JSONL line at the top of the file.
- [ ] AC-4: Stdout content is preceded by a `darkfactory_section` line with `"section": "stdout"`.
- [ ] AC-5: Stderr content is wrapped in a single `darkfactory_stderr` JSONL line.
- [ ] AC-6: No `#`-prefixed comment lines remain in the transcript output.
