# Configure branch protection for path-filtered and PRD validation checks

## Summary

Update the GitHub branch protection rules for `main` so that `PRD Validation / validate` is a strictly required status check on all PRs, while `CI / test` uses the "require when applicable" setting that GitHub supports for path-filtered workflows. This ensures PRD-only PRs can merge without waiting for the skipped CI check.

## Requirements

1. Add `PRD Validation / validate` as a required status check (always required).
2. Configure `CI / test` as a required check with "require when applicable" behavior so that path-filtered skips don't block merge.
3. Both status checks must appear on PRs.
4. PRD-only PRs must be mergeable without waiting for the skipped CI check.

## Technical Approach

This is a GitHub repository settings change, not a code change. The steps are:

1. Navigate to **Settings > Branches > Branch protection rules** for `main`.
2. Under **Require status checks to pass before merging**:
   - Add `PRD Validation / validate` as a required check.
   - Ensure `CI / test` is listed as a required check. GitHub automatically handles path-filtered workflows — when `ci.yml` is skipped due to path filters, the check is not required for merge.
3. Verify by opening a PRD-only PR and confirming it can merge with only the PRD validation check passing.

### Note on GitHub behavior

GitHub's branch protection handles path-filtered workflows natively. When a workflow has `paths:` filters and doesn't run for a PR, the corresponding status check is automatically marked as "Expected — Waiting for status to be reported" but does not block merge if the workflow was skipped due to path filters. No special configuration beyond adding both checks as required is needed.

## Acceptance Criteria

- [ ] AC-1: `PRD Validation / validate` appears as a required status check on all PRs.
- [ ] AC-2: `CI / test` appears as a required status check on PRs that trigger it.
- [ ] AC-3: PRD-only PRs can merge without waiting for the skipped CI check.
- [ ] AC-4: PRs touching source code require both checks to pass before merge.
