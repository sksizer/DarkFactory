# Configure branch protection for CI and PRD validation checks

## Summary

Update the GitHub branch protection rules for `main` so that both `CI / test` and `PRD Validation / validate` are required status checks on all PRs. CI runs on every PR without path filtering, so the required check is always satisfied.

## Requirements

1. Add `PRD Validation / validate` as a required status check (always required).
2. Add `CI / test` as a required status check (always required; CI runs on every PR).
3. Both status checks must pass before a PR can merge.

## Technical Approach

This is a ruleset configuration change captured in `.github/rulesets/main-protection.json`. The CI workflow (`ci.yml`) runs on all pull requests without path filtering, so requiring `CI / test` as a status check is safe — there will always be a check run to satisfy the requirement.

The ruleset enables `do_not_enforce_on_create: true` so that the first push to a new branch is not blocked before any checks have had a chance to run.

## Acceptance Criteria

- [ ] AC-1: `PRD Validation / validate` appears as a required status check on all PRs.
- [ ] AC-2: `CI / test` appears as a required status check on all PRs.
- [ ] AC-3: PRs cannot merge until both checks pass.
