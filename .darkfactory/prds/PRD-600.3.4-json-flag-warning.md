# Warn when --json is passed to unsupported command

## Summary

`--json` is a global flag that is silently ignored by 9 of 20 commands. Users passing `prd tree --json` or `prd validate --json` get human output with no indication their flag was ignored. Either warn on stderr or move `--json` to per-command flags.

## Technical approach

**Preferred: Warn on stderr.** After command dispatch in `main.py`, check if `args.json` was True and the command handler did not consume it. Add a lightweight mechanism:

1. Add a `_json_consumed` flag on the args namespace, defaulting to `False`
2. Each command that supports `--json` sets `args._json_consumed = True` at entry
3. After `func(args)` returns in `main.py`, if `args.json and not getattr(args, '_json_consumed', False)`: print a warning to stderr

Alternative: Define a set of commands that support JSON in `_parser.py` and check before dispatch.

Note: As `--json` support is added to more commands (PRD-600.3.2, 600.3.3, 600.4.3), this set shrinks. This task is best done after those.

## Acceptance criteria

- [ ] AC-1: `prd init --json` prints a warning to stderr
- [ ] AC-2: `prd status --json` does NOT print a warning (it supports JSON)
- [ ] AC-3: Warning text includes which command was run and that it doesn't support JSON
- [ ] AC-4: Warning goes to stderr, not stdout (so it doesn't corrupt piped output)
