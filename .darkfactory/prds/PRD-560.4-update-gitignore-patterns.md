# Update .gitignore patterns for .jsonl transcripts

## Summary

Update `.gitignore` to ignore `.jsonl` transcript files in the transient `.harness-transcripts/` directory, matching the new file extension introduced by PRD-560.1. Remove or update any existing `.log`-specific ignore patterns for transcripts.

## Requirements

1. Add or update `.gitignore` entries so that `.harness-transcripts/*.jsonl` files are ignored.
2. If there are existing ignore patterns for `.harness-transcripts/*.log`, update them to `.jsonl` or add the `.jsonl` pattern alongside (to cover both old and new files).
3. Ensure committed transcripts in `.darkfactory/transcripts/` are NOT ignored (they should be tracked by git).

## Technical Approach

### File: `.gitignore`

1. Search for existing `.harness-transcripts` patterns in `.gitignore`.
2. If a pattern like `.harness-transcripts/*.log` or `.harness-transcripts/` exists, ensure `.jsonl` files are also covered. The simplest approach is to ignore the entire `.harness-transcripts/` directory if not already done.
3. Verify that `.darkfactory/transcripts/` is not ignored.

## Acceptance Criteria

- [ ] AC-1: `.harness-transcripts/*.jsonl` files are ignored by git.
- [ ] AC-2: Committed transcripts in `.darkfactory/transcripts/` remain tracked.
- [ ] AC-3: No transient transcript files (old `.log` or new `.jsonl`) appear in `git status` after a run.
