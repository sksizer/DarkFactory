# Add tests for style.py

## Summary

`style.py` is the largest untested module in the codebase at 499 LOC. It contains the `Styler` class, theme resolution, `NO_COLOR`/TTY detection, icon set selection, and all Rich-based rendering. No tests exist anywhere for this module.

## Technical approach

Create `tests/test_style.py` covering:

1. **Styler construction**: Test that `Styler` accepts dark/light themes, no-color mode, and icon set overrides
2. **Element rendering**: Test `styler.render(Element.X, text)` returns expected styled output for key elements
3. **NO_COLOR compliance**: Test that `NO_COLOR=1` suppresses all color codes
4. **TTY detection**: Test that non-TTY output suppresses color (mock `sys.stdout.isatty()`)
5. **Icon set fallback**: Test nerdfont -> emoji -> ascii cascade
6. **Theme auto-detection**: Test `COLORFGBG` env var parsing for dark/light detection
7. **Config resolution**: Test that CLI flag > env var > config file > auto-detect

Focus on behavioral tests (what the user sees) rather than implementation details (which Rich primitives are used).

## Acceptance criteria

- [ ] AC-1: `tests/test_style.py` exists with tests for each concern above
- [ ] AC-2: `NO_COLOR` and `--no-color` properly suppress output styling
- [ ] AC-3: Theme and icon-set cascading works as documented
- [ ] AC-4: All tests pass, no new dependencies needed
