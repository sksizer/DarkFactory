# QueueFilters dataclass and discover_ready_queue function

## Summary

Add the pure-function core of the drain-ready-queue feature: a
`QueueFilters` dataclass that captures `--priority`, `--tag`, and
`--exclude` filters, a `matches_filters` predicate, and a
`discover_ready_queue` function that finds all `ready` PRDs with
satisfied dependencies, applies filters, and returns them in
topological order with a priority-then-number tiebreak.

These are standalone functions with no CLI or execution-loop coupling,
making them independently testable and easy to integrate later.

## Requirements

1. Define a `QueueFilters` dataclass with fields:
   - `min_priority: str | None` — minimum priority level (inclusive).
   - `tags: list[str]` — tag filter with OR semantics.
   - `exclude_ids: list[str]` — PRD IDs to exclude.
2. Implement `matches_filters(prd: PRD, filters: QueueFilters) -> bool`
   that returns `True` if the PRD passes all active filters.
3. Implement `deps_satisfied(prd: PRD, prds: dict[str, PRD]) -> bool`
   that returns `True` when all `depends_on` entries are `done` or
   `review`.
4. Implement `discover_ready_queue(prds: dict[str, PRD], filters: QueueFilters) -> list[PRD]`
   that returns all `ready` PRDs with satisfied deps, filtered, and
   topologically sorted with priority-then-PRD-number tiebreak.
5. Priority ordering: `critical` > `high` > `medium` > `low`.
   PRDs at or above `min_priority` pass; PRDs below are excluded.
6. Excluded PRDs' dependents should also be excluded when the exclusion
   breaks their dependency chain.

## Technical Approach

### File: `src/darkfactory/graph_execution.py`

Add the following at module level, after the existing traversal helpers:

```python
@dataclass
class QueueFilters:
    min_priority: str | None = None
    tags: list[str] = field(default_factory=list)
    exclude_ids: list[str] = field(default_factory=list)

PRIORITY_RANK = {"critical": 0, "high": 1, "medium": 2, "low": 3}

def matches_filters(prd: PRD, filters: QueueFilters) -> bool:
    if filters.min_priority:
        threshold = PRIORITY_RANK.get(filters.min_priority, 2)
        if PRIORITY_RANK.get(prd.priority, 2) > threshold:
            return False
    if filters.tags and not any(t in prd.tags for t in filters.tags):
        return False
    if prd.id in filters.exclude_ids:
        return False
    return True

def deps_satisfied(prd: PRD, prds: dict[str, PRD]) -> bool:
    for dep_id in prd.depends_on:
        dep = prds.get(dep_id)
        if dep is None or dep.status not in ("done", "review"):
            return False
    return True

def discover_ready_queue(
    prds: dict[str, PRD], filters: QueueFilters
) -> list[PRD]:
    ready = [
        p for p in prds.values()
        if p.status == "ready"
        and deps_satisfied(p, prds)
        and matches_filters(p, filters)
    ]
    return topo_sort_with_tiebreak(ready, prds)
```

The `topo_sort_with_tiebreak` function performs a topological sort on
the `depends_on` sub-graph restricted to the ready set, breaking ties
by `(PRIORITY_RANK[priority], prd_number)` so high-priority and older
PRDs come first.

      and `exclude_ids` fields.
