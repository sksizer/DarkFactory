# Wire up prd discuss CLI command with fail-fast binary checks

## Summary

Register the `prd discuss <prd-id>` subcommand in the CLI parser and implement the handler in `cli/discuss.py`. The handler probes for `claude` and `git` on PATH, resolves the PRD, builds a `SystemContext`, and dispatches to `system_runner.run_system_operation` with the discuss operation. Also write integration tests verifying the full chain dispatch order and the missing-binary fail-fast behavior.

## Requirements

1. New file `src/darkfactory/cli/discuss.py` with `cmd_discuss` entry point and `launch_discuss_for_prd` helper.
2. Register `discuss` subparser in `cli/_parser.py` with `prd_id` argument.
3. Import and wire `cmd_discuss` in `cli/main.py`.
4. `cmd_discuss` probes `shutil.which("claude")` and `shutil.which("git")` before any chain work. Exits with a clear error if either is missing.
5. Resolves the PRD by id (use `_resolve_prd_or_exit` from PRD-615 if available, otherwise inline).
6. Builds `SystemContext` with `target_prd` set and dispatches to `run_system_operation(discuss_operation, ctx)`.
7. `launch_discuss_for_prd(prd_id, args)` is a shared helper that PRD-616.6 will call from `prd new --discuss`.
8. `prd discuss --help` describes the command, its phases, and the manual-exit interaction model.
9. Integration test in `tests/test_cli_discuss.py` verifying CLI dispatch.
10. Integration test in `tests/test_commands_discuss.py` verifying operation shape and chain order (mocked interactive launches).
